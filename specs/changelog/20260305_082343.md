# Changelog — 2026-03-05T08:23:43Z

## Update

- `server/extensions/plugins/sdk/jh-instance.ts` — Fixed `CAPABILITY_INVOKE` response to send a properly typed `RESOLVE_CAPABILITY_RESPONSE` message (with `payload: { requestId, result }`) instead of a plain `PostMessageResponse` without a `type` field, so the host bridge can route the response to the correct `pendingCapabilityRef` entry. Without this fix, all `bridge.resolve()` calls timed out after 3 s and returned empty arrays, meaning no card-buttons or card-badges ever rendered.

- `src/extensions/Plugins/uiInjections/CardPluginButtons.tsx` — Updated `PluginButton` interface to include `callback?: { __callbackId: string }`. Updated `handleButtonClick` to broadcast `BUTTON_CLICKED` postMessage (with `callbackId` + card/list/board `args`) to all plugin iframes instead of sending a `buttonText`-based message. Only the plugin iframe whose `callbackRegistry` contains the matching ID will invoke the callback; others ignore it silently.

- `public/sdk/jh-instance.js` — Rebuilt from updated `jh-instance.ts` source.

## New

- `callbackRegistry` Map in the SDK — stores callback functions replaced by opaque `{ __callbackId }` tokens during postMessage serialization. Keys are `jh-${timestamp}-${counter}` strings.

- `serializeResult()` helper in the SDK — recursively walks a capability result and replaces any `function` value with `{ __callbackId: id }`, storing the original function in `callbackRegistry`.

- `BUTTON_CLICKED` message listener in the SDK — when received, looks up the `callbackId` in `callbackRegistry` and invokes the stored callback with a fresh `FrameContext` constructed from the `args` payload (card, list, board context). Silently ignores the message if the `callbackId` is not in its registry, allowing safe broadcast to all plugin iframes.

## Technical Debt

- Callback entries in `callbackRegistry` are never removed. For long-running board sessions with many card renders this could accumulate stale entries. A TTL or WeakRef approach should be added in a future cleanup iteration.
