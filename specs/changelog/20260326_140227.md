## Update
- Replaced the empty `colors` placeholder in `tailwind.config.ts` with a full set of semantic colour aliases (`primary`, `primary-hover`, `secondary`, `accent`, `bg-base`, `bg-surface`, `bg-overlay`, `bg-sunken`, `border`, `border-strong`, `danger`, `success`, `warning`, `info`) pointing to CSS custom properties, and added a `textColor` extension for `base`, `muted`, `subtle`, `inverse`, and `link`.
- Replaced the `cva`-based `Button.tsx` implementation with a new one built on `buttonVariants` from `src/config/theme.ts`, eliminating the `class-variance-authority` dependency from this component and aligning it with the design token system.

## New
- `src/config/theme.ts` — new module exporting `buttonVariants` (primary, secondary, ghost, danger, success, lg, md, sm, icon), `ButtonVariant`, and `ButtonSize` types. This is the single source of truth for button appearance across the app.
- `specs/tests/sprint-118-button-variants.md` — Playwright MCP test spec covering 8 scenarios: primary/secondary/ghost/danger variants in dark mode, and colour changes when switching to light mode.

## Technical Debt
- `src/common/components/Button.tsx` previously depended on `class-variance-authority` (`cva`). The package is still listed in `package.json` and may still be used elsewhere; it has not been removed. If no other file uses it, it should be pruned.
- Existing callers of `Button` that pass `variant="ghost"` with dark-mode-specific ghost styling (`text-gray-300 hover:bg-white/10 hover:text-white`) will now receive semantic token classes instead. Visual regression check is recommended when Sprint 119 migration runs.
