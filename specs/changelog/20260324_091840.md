## New
- `specs/tests/health-check-board.md`: 26-test MCP scenario spec covering all Sprint 115 acceptance criteria:
  - AC-1: GET `/api/v1/health-check/presets` returns preset list
  - AC-2: Add custom URL health check → 201, `latestResult: null`
  - AC-3: Add preset health check → 201, `type: 'preset'`, `presetKey` set
  - AC-4: Duplicate URL per board → 409 `health-check-url-already-monitored`
  - AC-5: On-demand probe of green endpoint → `status: 'green'`
  - AC-6: Slow endpoint → amber (noted as integration-only; requires mock)
  - AC-7: Unreachable endpoint probe → `status: 'red'`, `errorMessage` set
  - AC-8: SSRF rejection for private IPs (`192.168.x`, `10.x`, `127.x`), localhost, `.local`
  - AC-9: GET list after probe embeds `latestResult`
  - AC-10: `HEALTH_CHECK_ENABLED=false` → all routes return 404
  - Additional: URL validation (invalid scheme, credentials, relative URL, name > 120 chars), rate limiting (429), DELETE lifecycle, unauthenticated access.
