# Changelog — Sprint 80, Iteration 7: Guest Invite Flow

## Update
- `src/extensions/Board/components/BoardMembersPanel/index.tsx`: Added tab navigation (Members / Guests) to the Board Members Panel. The panel now renders `GuestsTab` when the Guests tab is active and the existing member list/add UI when the Members tab is active.
- `server/extensions/board/api/guests/index.ts`: Refactored to re-export `handleInviteGuest` from the new `create.ts` module (email-based invite) and demoted the legacy userId handler to an unexported internal function.

## New
- `src/extensions/Board/components/BoardMembersPanel/GuestsTab.tsx`: New Guests tab UI. Board admins see an email invite input with success/error feedback; all members see the current guest list with name/email display. Admins can remove guests via a "Remove" button.
- `src/extensions/Board/slices/boardGuestsSlice.ts`: RTK Query slice for board guests (`getBoardGuests`, `inviteBoardGuest`, `revokeBoardGuest`). Registered in `src/reducers.ts` and `src/store/index.ts`.
- `server/extensions/board/api/guests/create.ts`: `handleInviteGuestByEmail` — accepts `{ email }` in the POST body, looks up the user by email, creates a minimal stub account if the user doesn't exist, validates the user is not already a non-guest workspace member, prevents duplicate invites, and returns the `board_guest_access` row.
- `specs/tests/sprint-80-iteration-7-guest-invite.md`: Playwright MCP test covering: tab visibility, invite new guest, workspace-member error, duplicate invite error, guest removal, and read-only view for non-admin members.

## Technical Debt
- Stub accounts created for non-existent email invitees have no password. A future "claim account" / invite-acceptance flow should be added so these users can set a password and log in.
- The `handleInviteGuestById` function in `guests/index.ts` is now dead code; it should be removed when the userId-based path is confirmed unused.

- Iteration 9: Guest Client Permission Guards + Auth-Aware Route Guards — hide board settings and member management from GUEST users, redirect from workspace members route.
