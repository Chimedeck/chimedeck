# Changelog — 20260311_021901

## Update

- `server/mods/observability/metrics.ts`: added `realtimeConflicts` (counter `realtime.conflicts`) and `realtimePropagationDelayMs` (histogram `realtime.propagation_delay_ms`) instruments to the `Metrics` interface; both are no-ops when `OTEL_ENABLED=false`.
- `server/extensions/card/api/move.ts`: imported `recordConflict` from the new `conflictHandler` module and calls it after a card move when a position collision is detected (concurrent move race).
- `server/mods/events/write.ts`: added `emittedAt: Date.now()` field to the WebSocket message payload so clients can compute propagation delay.
- `src/extensions/Realtime/client/socket.ts`: added optional `emittedAt?: number` field to `RealtimeEvent` type.
- `src/extensions/Realtime/hooks/useWebSocket.ts`: added `pingPropagationDelay()` helper that fires `POST /api/v1/metrics/propagation` (fire-and-forget, never throws) for every incoming WS event that carries an `emittedAt` timestamp.
- `server/index.ts`: registered `POST /api/v1/metrics/propagation` route via `handlePropagationPing`.

## New

- `server/extensions/realtime/mods/conflictHandler.ts`: exports `recordConflict({ boardId, entityType })` — increments both `realtime.conflicts` and the legacy `conflict_total` counters; safe no-op when OTel is disabled.
- `server/extensions/realtime/api/metrics.ts`: `POST /api/v1/metrics/propagation` endpoint; validates `delayMs` ≥ 0, records it in `realtime.propagation_delay_ms` and `syncDelayMs` histograms when `OTEL_ENABLED=true`, always responds 204 to avoid client errors when OTel is off.
- `tests/e2e/otel-metrics.spec.ts`: Playwright E2E tests covering the propagation endpoint (valid/invalid payloads) and a smoke test for card move conflict detection.

## Technical Debt

- The conflict counter is currently triggered by a post-update DB query to detect position collisions; a more efficient approach would be to detect the collision before the `UPDATE` and handle it transactionally.

- Add Prometheus scrape config to `docker-compose.yml` for the OTel Prometheus exporter so metrics are visible in local development.
