## New
- Added `db/migrations/0034_automation.ts` migration script, creating four new tables for the automation feature:
  - `automations`: Stores automation definitions, including type, status, icon, run count, and timestamps.
  - `automation_triggers`: Stores trigger configuration for each automation, with unique constraint on `automation_id`.
  - `automation_actions`: Stores ordered actions for each automation, including type and config.
  - `automation_run_log`: Stores audit log of automation runs, including status, context, error messages, and timestamps.
- All ID columns use `string` (varchar) type to match existing schema conventions.
- Enum and JSONB fields implemented per architecture spec.
