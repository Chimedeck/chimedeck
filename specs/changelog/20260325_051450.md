# Changelog — 20260325_051450

## Update

- `server/extensions/users/api/me.ts`: `handleGetMe` and `handlePatchMe` now return `avatar_url` as the stable proxy path `/api/v1/users/:id/avatar` instead of a raw S3 presigned URL.
- `server/extensions/users/api/profile/get.ts`: `handleGetProfile` now returns proxy path for `avatar_url`.
- `server/extensions/comment/api/list.ts`: `handleListComments` now returns proxied `author_avatar_url` (proxy path per comment author) instead of a raw presigned URL.
- `server/common/avatar/resolveAvatarUrl.ts`: Added `buildAvatarProxyUrl` helper that returns the stable `/api/v1/users/:id/avatar` path for a given user+avatar_url (null when no avatar).

## New

- `server/extensions/users/api/avatar/proxy.ts`: New `GET /api/v1/users/:id/avatar` endpoint — authenticates caller, resolves user's avatar S3 key or external URL, and issues a 302 redirect to a fresh short-lived (60 s) presigned URL. Returns 401 if unauthenticated, 404 if user/avatar not found.
- `server/extensions/users/api/index.ts`: Registered the new `GET /api/v1/users/:id/avatar` route.
- `tests/e2e/sprint-117-avatar-proxy.md`: Playwright MCP markdown tests covering avatar proxy endpoint (auth checks, proxy path in me/comments responses, 404 for missing avatar).

## Technical Debt

- `resolveAvatarUrl` and `resolveAvatarUrlsInCollection` in `server/common/avatar/resolveAvatarUrl.ts` are no longer used by the main me/profile/comments endpoints but are kept for any remaining callers (e.g. upload confirmation). Review and remove when all callers are migrated.

## What Should Be Done Next

- Iteration 6: Update client-side avatar rendering (ProfileSettings, ActivityFeed comment avatars) to consume the proxied `avatar_url` from the API instead of generating presigned URLs client-side.
