# Changelog — 2026-03-20T02:18:47Z

## Update
- `server/extensions/notifications/api/index.ts` — registered `boardPreferenceRouter` as the first route handler so board-scoped notification-preference requests are resolved before generic notification routes.

## New
- `server/extensions/notifications/api/boardPreference/get.ts` — `GET /api/v1/boards/:boardId/notification-preference`: returns `{ data: { notifications_enabled, updated_at } }` for the authenticated user. Enforces auth + board membership (403 for non-members). Missing row defaults to `notifications_enabled: true` (opt-out model).
- `server/extensions/notifications/api/boardPreference/update.ts` — `PATCH /api/v1/boards/:boardId/notification-preference`: upserts the preference row. Enforces auth + board membership. Accepts `{ notifications_enabled: boolean }`. Returns the persisted row.
- `server/extensions/notifications/api/boardPreference/index.ts` — thin router that matches the `notification-preference` sub-path and delegates to get/update handlers.
