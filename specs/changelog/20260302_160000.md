# Changelog — 20260302_160000 — Sprint 16: Authentication UI

## Update
- Replaced stub `LoginPage` and `SignupPage` with full, accessible Tailwind-styled implementations.
- Expanded `authDuck.ts`: replaced direct `axios` calls with typed `authApi` wrappers; added `signupThunk` and `refreshTokenThunk`.
- `App.tsx` now dispatches `refreshTokenThunk` on mount so an authenticated session survives a browser page reload (httpOnly cookie sent automatically).

- `src/extensions/Auth/components/PasswordInput.tsx` — password field with show/hide toggle (lucide `Eye`/`EyeOff`).
- `src/extensions/Auth/components/AuthDivider.tsx` — "or" divider between form and OAuth buttons.
- `src/extensions/Auth/components/OAuthButton.tsx` — Google / GitHub OAuth button (rendered but disabled pending feature flags).
- `src/extensions/Auth/components/LoginForm.tsx` — login form with blur + submit validation, field-level errors, and API error mapping.
- `src/extensions/Auth/components/SignupForm.tsx` — sign-up form with name / email / password / confirm-password validation.
- `src/extensions/Auth/translations/en.json` — English strings for labels, actions, validation messages, and API errors.
- `specs/tests/auth-login.md` — Playwright test spec for login happy path, error path, toggle, and keyboard navigation.
- `specs/tests/auth-signup.md` — Playwright test spec for sign-up happy path and validation edge cases.
