# Changelog — 2026-03-05T08:10:36Z

## Update
- `src/extensions/Plugins/iframeHost/usePluginBridge.ts`: Added four `CTX_*` case handlers (`CTX_CARD`, `CTX_LIST`, `CTX_BOARD`, `CTX_MEMBER`) in the global postMessage switch so that plugin context-read calls (`t.card()`, `t.list()`, `t.board()`, `t.member()`) resolve correctly instead of timing out.

## New
- `pluginContextRef` (`Map<string, CapabilityContext>`) in `usePluginBridge` tracks the last context sent to each plugin via `CAPABILITY_INVOKE`. This is populated each time `resolve()` dispatches a capability invocation and consumed by the `CTX_*` handlers.
- `extractContextFields` helper — filters a context object down to only the requested fields (or returns the full object when no field list is given).
- `handleCtxQuery` — shared handler that looks up the stored context for the requesting plugin, extracts requested fields, and posts the result back as an SDK response.
