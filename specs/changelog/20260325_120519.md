# Changelog — 2026-03-25 12:05

## Update

- `ActivityFeed.tsx` — replaced raw `attachment.url` with `attachment.view_url` in `attachmentMap` and all thumbnail `<a href>` / `<img src>` references, eliminating the last raw S3 URL exposure in the activity timeline.

## New

- **`AttachmentItem` Comment action** — new `onInsertComment: (markdown: string) => void` prop. When the `ChatBubbleLeftIcon` button is clicked, builds `[alias ?? name](view_url)` and calls the prop. Button is hidden when `onInsertComment` is absent or the attachment is not READY/has no `view_url`.

- **`CommentEditor` external insert API** — new optional `insertMarkdownRef` prop (`MutableRefObject<((md: string) => void) | null>`). When provided, the component registers a cursor-aware insert function on mount and clears it on unmount.

- **`ActivityFeed` insert bridge** — new optional `insertMarkdownRef` prop forwarded to the embedded `CommentEditor`, allowing the parent (`CardModal`) to hold a shared ref that links attachment actions to the comment input.

- **`AttachmentPanel` Comment wiring** — new optional `insertMarkdownRef` prop; when provided, each READY file attachment row shows the Comment button; clicking it calls `insertMarkdownRef.current(md)` with the formatted link (no network call).

- **`CardModal`** — creates `insertMarkdownRef = useRef(null)` and passes it to both `AttachmentPanel` and `ActivityFeed`, completing the prop chain.

- **Translation key** `attachment.comment.insert` and `attachment.item.action.comment.ariaLabel` added to `en.json`.

- **Test file** `tests/e2e/sprint-117-attachment-comment-action.md` — Playwright MCP scenarios covering button visibility, markdown insertion, alias fallback, URL proxy check, and no-network assertion.

## Technical Debt

- The `insertMarkdownRef` pattern works for the primary add-comment editor in `ActivityFeed`. Edit-comment editors (inline per `CommentItem`) are not yet wired — a future iteration could expose the same ref from each `CommentItem` editor and allow the user to target whichever editor has focus.
