# Sprint 60 — Attachment UI Foundation

## Update
- No existing files were modified in this iteration.

## New
- `src/extensions/Attachments/types.ts` — TypeScript types/interfaces for `Attachment`, `AttachmentStatus`, `AttachmentType`, upload URL/confirm/multipart request+response shapes, `UploadEntry` (client-only in-flight state).
- `src/extensions/Attachments/api.ts` — Plain async functions (axios `apiClient`) covering all 8 attachment endpoints: `listAttachments`, `requestUploadUrl`, `confirmUpload`, `startMultipart`, `getPartUrl`, `completeMultipart`, `abortMultipart`, `createUrlAttachment`, `deleteAttachment`.
- `src/extensions/Attachments/utils/mimeIcon.ts` — `getMimeIcon(mimeType)` maps MIME strings to Heroicons (Photo, DocumentText, TableCells, ArchiveBox, Film, MusicalNote, Document, Link). Falls back to `DocumentIcon` for unknown types.
- `src/extensions/Attachments/utils/formatBytes.ts` — `formatBytes(bytes)` decimal formatter (e.g. `48200 → "48.2 KB"`, `12500000 → "12.5 MB"`).
- `src/extensions/Attachments/components/AttachmentItem.tsx` — Single attachment row with: MIME/URL icon badge, truncated filename, size, status chip (Uploading / Scanning / Ready / Rejected), open/download button, inline delete confirmation ("Delete? Yes / No"), `UploadProgressBar` while uploading.
- `src/extensions/Attachments/components/AttachmentThumbnail.tsx` — Clickable 96×64 image preview card using `thumbnailUrl` (falling back to `url`). Placeholder `PhotoIcon` when no URL yet.
- `src/extensions/Attachments/components/UploadProgressBar.tsx` — Accessible `role="progressbar"` element with determinate fill and indeterminate shimmer animation.

## Technical Debt
- The `UploadProgressBar` indeterminate animation uses a custom keyframe (`animate-[shimmer_...]`) not yet defined in Tailwind config; add `shimmer` to `tailwind.config.ts` if the indeterminate state is needed.
