# Changelog — 2026-03-30T11:30:00Z

## Update

- **server/index.ts**: The Bun `fetch` handler now wraps the `router(req)` call in a try-catch. Unhandled exceptions are passed to `handleUnhandledError` instead of propagating and crashing the request.

## New

- **server/common/middlewares/errorHandler.ts**: Introduced `handleUnhandledError(err)` — captures the error in Sentry (no-op when disabled), returns a JSON 500 response in the standard API envelope `{ error: { code, message } }`, with a plain-text fallback if serialisation itself fails.
- **tests/unit/server/common/middlewares/errorHandler.test.ts**: Unit tests covering: 500 envelope shape, Sentry `captureError` call, non-Error thrown values, and message extraction from `Error` instances (5 passing cases).

## Technical Debt

- `sendDefaultPii: false` is already set in `sentry.ts`; full PII redaction (scrubbing headers/query strings in `beforeSend`) is deferred to Iteration 7.
- `tracesSampleRate` is hardcoded to `1.0` — should become a configurable env var once traffic patterns are understood.

## What Should Be Done Next

- **Iteration 6**: Release tagging and Vite source-map upload pipeline so errors in production link back to readable source.
- **Iteration 7**: Privacy hardening (`beforeSend` redaction), documentation of PII policy, and final end-to-end manual verification checklist.
