# Changelog — 20260305_021501

## Update
- Wired plugin registry sub-routers into `server/extensions/plugins/api/index.ts` (replaced placeholder stub).

## New
- `server/extensions/plugins/api/registry/list.ts` — GET `/api/v1/plugins`: lists all plugins for workspace OWNER/ADMIN, public+active only for other authenticated users.
- `server/extensions/plugins/api/registry/get.ts` — GET `/api/v1/plugins/:pluginId`: returns plugin detail; non-admins get 404 for non-public/inactive plugins.
- `server/extensions/plugins/api/registry/create.ts` — POST `/api/v1/plugins`: registers a new plugin (admin only), generates `api_key` via `randomUUID`, fetches and caches `capabilities` from `manifestUrl` (fail-open on network error). Returns `api_key` only on creation.
- `server/extensions/plugins/api/registry/update.ts` — PATCH `/api/v1/plugins/:pluginId`: partially updates plugin metadata (admin only). Returns the updated record without `api_key`.
- `server/extensions/plugins/api/registry/delete.ts` — DELETE `/api/v1/plugins/:pluginId`: soft-deletes by setting `is_active=false` (admin only). Row is never hard-deleted.

## Technical Debt
- "Platform admin" is proxied as any user with OWNER or ADMIN role in any workspace. A dedicated `platform_admin` flag on the `users` table would be cleaner once the user management UI exists.
- `isRegistryAdmin` helper is duplicated across all five registry files; extract to a shared utility when needed.
