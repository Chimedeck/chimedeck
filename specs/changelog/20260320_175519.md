## Update
- Added card cover support with persisted cover metadata on cards: image attachment reference, color, and display size.
- Extended card APIs (`GET /cards/:id`, `PATCH /cards/:id`, list cards, and board fetch cards) to include cover fields and resolved `cover_image_url` for rendering.
- Updated card modal and board card tile UI to display and edit covers (size, color, upload image, remove cover).

## New
- New DB migration: `0090_card_cover.ts` adds `cover_attachment_id`, `cover_color`, and `cover_size` columns to `cards`.
- New shared server helper: `server/common/cards/cover.ts` to resolve signed cover image URLs from attachment references.

## Technical Debt
- `server/extensions/card/api/update.ts` remains high-complexity and should be split into smaller validation/update helpers.
- Existing accessibility lint warnings for role usage in card/list components remain pre-existing and were not refactored in this iteration.

## What Should Be Done Next

- Add API tests for `PATCH /cards/:id` cover validation paths (invalid attachment, non-image attachment, invalid color, invalid size).
- Consider extracting cover controls from `CardModal.tsx` into a dedicated component for maintainability.
