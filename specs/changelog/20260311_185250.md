## Update
- Refactored automation config to use central `env` pattern for feature gating and limits.
- No changes to existing API, UI, or authentication flows.

## New
- Created automation engine core modules in `server/extensions/automation/engine/`:
  - `index.ts`: Exports `evaluate()` to orchestrate registry, matcher, executor, and logger.
  - `registry.ts`: Implements trigger/action registries for handler extensibility.
  - `matcher.ts`: Matches events to triggers using registered handlers.
  - `executor.ts`: Executes actions transactionally, handles per-action errors, returns run status.
  - `logger.ts`: Writes automation run logs, enforces 1000-row cap, broadcasts run events.
- Added `common/types.ts` (shared interfaces) and `config/index.ts` (feature gate, concurrency, log cap).

## Technical Debt

- Registry, matcher, and executor extensibility patterns are minimal and will require further hardening as new trigger/action types are added.
- Logger eviction policy is FIFO (oldest purged); may need review for edge cases.
