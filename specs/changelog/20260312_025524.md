# Changelog — 2026-03-12T02:55:24Z

## Update
- `src/extensions/Automation/types.ts`: Added `icon` field to `CreateAutomationPayload` and `UpdateAutomationPayload`; added `CardButtonRunResult` type.

- `server/extensions/automation/api/index.ts`: Registered `POST /api/v1/cards/:cardId/automation-buttons/:automationId/run` route.

## New
- `server/extensions/automation/api/runCardButton.ts`: Server handler for `POST /api/v1/cards/:cardId/automation-buttons/:automationId/run`. Authenticates caller, verifies board membership, looks up enabled CARD_BUTTON automation, runs executor, writes run log, returns `{ data: { runLogId, status } }`.
- `src/extensions/Automation/components/shared/IconPicker.tsx`: Reusable icon picker grid — 24 selectable Heroicons, `ButtonIcon` renderer, `BUTTON_ICONS` const array.
- `src/extensions/Automation/components/CardButtons/CardButtonItem.tsx`: Single card button with icon, name, and loading/success/error run states.
- `src/extensions/Automation/components/CardButtons/AddCardButtonButton.tsx`: "Add button" CTA that opens the builder.
- `src/extensions/Automation/components/CardButtons/CardButtonBuilder.tsx`: Modal builder for creating/editing CARD_BUTTON automations (name + icon picker + action list).
- `src/extensions/Automation/components/CardButtons/CardButtonsSection.tsx`: Full "Automation" section rendered in the card detail modal — lists buttons, handles runs, opens builder.
- `tests/integration/automation/cardButtons.test.ts`: Integration tests for the run endpoint (auth guard, router registration, response shape).
- `tests/e2e/automation/cardButtons.md`: Playwright MCP e2e tests covering create, run (success + error), icon picker, validation, cross-card sharing, and archived card disabled state.

## Technical Debt
- `CreateAutomationPayload.trigger` is typed as `... | null` to support CARD_BUTTON/BOARD_BUTTON automations which have no trigger; the server `create.ts` already handles null triggers correctly.
