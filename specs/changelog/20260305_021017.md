# Changelog — 20260305_021017

## Update
- `server/extensions/plugins/api/index.ts` — wired GET and PUT handlers for `/api/v1/plugins/data`

## New
- `server/extensions/plugins/api/plugin-data/get.ts` — GET /api/v1/plugins/data: reads plugin-scoped key/value storage. Auth via `Authorization: ApiKey <key>`. Supports `scope`, `resourceId`, `key`, `visibility` (private/shared) and `userId` (required for private). Returns `{ data: value }` or `{ data: null }` when not found.
- `server/extensions/plugins/api/plugin-data/set.ts` — PUT /api/v1/plugins/data: upserts plugin-scoped key/value storage. Same auth. Body: `{ scope, resourceId, key, visibility, value, userId? }`. Uses select-then-insert/update to handle nullable `user_id` in the UNIQUE constraint correctly.

## Technical Debt
- `resolvePlugin` helper is duplicated in `get.ts` and `set.ts`. Should be extracted to a shared util if more plugin-data endpoints are added.
