# Changelog — 2026-03-10T09:29:00Z

## Update

- `server/extensions/customFields/api/index.ts`: Extended router to handle card-scoped custom field value routes (`GET/PUT/DELETE /api/v1/cards/:id/custom-field-values/:fieldId`) in addition to the existing board-scoped field definition routes.
- `server/extensions/card/api/get.ts`: Updated `GET /api/v1/cards/:id` to always include `customFieldValues` under the `includes` key in the response.

## New

- `server/extensions/customFields/api/cardValues.ts`: Implements `handleGetCardFieldValue`, `handleUpsertCardFieldValue`, and `handleDeleteCardFieldValue` for all 5 field types (TEXT, NUMBER, DATE, CHECKBOX, DROPDOWN). Upsert uses PUT semantics (201 on create, 200 on update). Includes board-membership auth guard and cross-board field mismatch validation (422 `custom-field-board-mismatch`).
- `tests/e2e/custom-field-values-api.md`: Playwright MCP markdown e2e test covering upsert, retrieve, delete of all 5 field types, card GET includes, type mismatch errors, cross-board rejection, and unauthenticated rejection.
