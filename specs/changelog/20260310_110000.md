# Changelog — 2026-03-10T11:00:00Z

## Update

- **board/api/patch.ts**: `PATCH /api/v1/boards/:id` now accepts `visibility` (enum: PUBLIC/PRIVATE/WORKSPACE), `description` (string|null), and `background` (string|null). Enum validation returns 400 with `bad-request` error name on invalid value.
- **board/api/create.ts**: `POST /api/v1/workspaces/:id/boards` now accepts `visibility`, `description`, and `background`. Defaults `visibility` to `PRIVATE` if not provided.
- **board/api/get.ts**: Cards fetched inside the board response now include `start_date` in the explicit column select.
- **card/api/update.ts**: `PATCH /api/v1/cards/:id` now accepts `start_date` (ISO 8601 string or null) with date validation.
- **card/api/create.ts**: `POST /api/v1/lists/:id/cards` now accepts `start_date` (ISO 8601 string or null) with date validation and persists it on insert.

## New

- Board creation supports initial `visibility`, `description`, and `background` fields.
- Cards support `start_date` field on create and update.

## Technical Debt

- `board/api/update.ts` (`handleUpdateBoard`) is imported in `board/api/index.ts` but is not wired to any route — it is dead code. Should be removed or merged into `patch.ts`.
