# Changelog — 2026-03-11T11:49:15Z

## Update

- Created `server/extensions/automation/common/types.ts` with shared TypeScript types for the automation system.
- Created `server/extensions/automation/config/index.ts` centralising `AUTOMATION_ENABLED` and engine constants.

## New
- **`registry.ts`**: `TRIGGER_REGISTRY` and `ACTION_REGISTRY` Maps with `registerTrigger`, `registerAction`, `getTriggerHandler`, `getActionHandler`, `getAllTriggerTypes`, `getAllActionTypes` exports. New handlers can be added without touching engine code.
- **`matcher.ts`**: `matchesTrigger()` delegates to the registered `TriggerHandler.matches()`, returning `false` (not throwing) for unknown types or handler bugs.
- **`executor.ts`**: `executeAutomation()` iterates ordered actions in one DB transaction, catches per-action errors individually, and returns `SUCCESS | PARTIAL | FAILED` with aggregated error message.
- **`logger.ts`**: `writeRunLog()` inserts a run log row, purges oldest rows beyond the 1000-row cap per automation (single `DELETE … NOT IN` query), increments `run_count + last_run_at` on the automation, and broadcasts the `automation_ran` WS event fire-and-forget.
- **`engine/index.ts`**: `evaluate({ boardId, event, context })` — loads enabled RULE automations, runs matcher, executes matching automations in batches of 5, writes logs. Never throws; fully fire-and-forget safe.
