# Changelog — 2026-03-10T04:52:41Z

## Sprint 51 — Client Forced Logout + HTTP Polling Fallback

### Update

- **`src/common/api/interceptors.ts`**: After a failed token refresh on a 401 response, the interceptor now redirects to `/login?reason=session_expired` in addition to clearing Redux auth state. This ensures the user is returned to the login page with context when their session expires.

- **`src/extensions/Realtime/client/socket.ts`**: Added WS close code `4001` handling — when the server closes a WebSocket with code 4001 (session revoked), the socket calls a `forcedLogoutCallback` and stops all reconnection attempts. Added `failedAttempts` counter: after 3 consecutive unexpected closes, subscribers are notified via `onPollingActive` callbacks to activate HTTP polling. When WS reconnects successfully after polling mode was active, `onPollingInactive` callbacks fire to stop polling. Added `setForcedLogoutCallback` and `usingPollingFallback` getter to the public API.

- **`src/extensions/Realtime/hooks/useWebSocket.ts`**: Extended `UseWebSocketResult` with `pollingActive: boolean`. Subscribes to `onPollingActive`/`onPollingInactive` socket callbacks and reflects them as React state.

- **`src/extensions/Board/components/BoardHeader.tsx`**: Added optional `pollingActive` prop; renders `PollingIndicator` alongside `ConnectionBadge`.

- **`src/extensions/Board/containers/BoardPage/BoardPage.tsx`**: Destructures `pollingActive` from `useWebSocket`; adds `usePollingFallback` hook call; passes `pollingActive` to `BoardHeader`.

- **`src/main.tsx`**: Wires `socket.setForcedLogoutCallback` to dispatch `clearAuth()` and redirect to `/login?reason=session_expired`, ensuring WS-initiated forced logouts propagate to Redux and the browser URL.

### New

- **`src/extensions/Realtime/PollingFallback.tsx`**: `usePollingFallback` hook — polls `GET /api/v1/boards/:id/events?since=<lastSequence>` every 5 s when `active=true`. Delivers new events to `onEvents` callback with deduplication (skips sequences already seen, guards against WS reconnect race conditions). Stops automatically when `active` flips to `false`.

- **`src/extensions/Realtime/PollingIndicator.tsx`**: Pill badge displayed in `BoardHeader` when HTTP polling fallback is active. Styled to match `ConnectionBadge` (blue palette, animated sync icon).

### Technical Debt

- `POLLING_FALLBACK_THRESHOLD` (3 failures) is a compile-time constant; consider making it configurable via feature flags if the number needs tuning in production.
- `failedAttempts` is in-memory on the socket singleton — it resets on page reload. This is acceptable since polling activates quickly anyway.

### What Should Be Done Next

- Playwright tests: verify 401 mid-session redirects to `/login?reason=session_expired`, polling indicator appears when WS is blocked, and disappears when WS recovers.
- Confirm no event duplication or reordering during fallback transitions under load.
