# Sprint 117 Iteration 1 — DB Migration + Secure Attachment Proxy

## Update
- `server/extensions/attachment/api/list.ts`: Removed raw S3 presigned URL generation from the list endpoint. Now returns `view_url` (proxy path) and `thumbnail_url` (proxy path) instead. Added `alias` field (null by default).

## New
- `db/migrations/0099_attachment_alias.ts`: Adds nullable `alias` column (varchar 255) to `attachments` table.
- `server/extensions/attachment/api/view.ts`: `GET /api/v1/attachments/:id/view` — authenticated proxy endpoint that 302-redirects to a 60 s short-lived presigned S3 URL. Enforces board membership; handles URL-type, PENDING, REJECTED, and missing-key edge cases.
- `server/extensions/attachment/api/thumbnail.ts`: `GET /api/v1/attachments/:id/thumbnail` — same proxy pattern for the thumbnail S3 key.
- `server/extensions/attachment/api/index.ts`: Registers the two new GET proxy routes before the existing `:id/url` route.
- `tests/e2e/sprint-117-attachment-proxy.md`: Playwright MCP markdown tests covering 401 for unauthenticated, 403 for non-member, 200/302 for board member, proxy path shape in list response, alias field presence, and thumbnail edge cases.

## Technical Debt
- The legacy `GET /api/v1/attachments/:id/url` endpoint still exists and still returns raw presigned URLs. It should be deprecated and removed once all clients migrate to the proxy endpoints.
