# Changelog — 20260302_052312

## Update
- `server/extensions/card/api/index.ts`: Added routes for labels (attach/detach), members (assign/remove), checklist items (create/update/delete via `/checklist-items/:id`), and due-date query (`/workspaces/:id/cards/due`).
- `server/extensions/card/api/get.ts`: Populated `includes.labels`, `includes.members`, and `includes.checklistItems` from the DB instead of returning empty arrays.
- `server/index.ts`: Mounted `labelRouter` for workspace-scoped label management.
- `src/extensions/Card/api.ts`: Added `Label`, `CardMember`, `ChecklistItem` types; added API functions for labels CRUD, card label attach/detach, member assign/remove, checklist item CRUD, and due-date query.
- `src/extensions/Card/translations/en.json`: Added i18n strings for labels, members, checklist, and due date sections.

## New
- `db/migrations/0007_card_extended.ts`: Migration adding `labels`, `card_labels`, `card_members`, and `checklist_items` tables with FK cascades.
- `server/extensions/card/mods/labels/validate.ts`: `validateCardLabelLimit()` — enforces max 20 labels per card (returns 400 if limit reached).
- `server/extensions/label/api/create.ts`: `POST /api/v1/workspaces/:id/labels` — create workspace-scoped label; min role ADMIN.
- `server/extensions/label/api/list.ts`: `GET /api/v1/workspaces/:id/labels` — list workspace labels; min role VIEWER.
- `server/extensions/label/api/update.ts`: `PATCH /api/v1/labels/:id` — update label name/color; min role ADMIN.
- `server/extensions/label/api/delete.ts`: `DELETE /api/v1/labels/:id` — delete label (cascades card_labels); min role ADMIN.
- `server/extensions/label/api/index.ts`: Label router wiring all four label endpoints.
- `server/extensions/card/api/labels.ts`: `POST /api/v1/cards/:id/labels` (attach) and `DELETE /api/v1/cards/:id/labels/:labelId` (detach); validates workspace match and max-20 limit.
- `server/extensions/card/api/members.ts`: `POST /api/v1/cards/:id/members` (assign) and `DELETE /api/v1/cards/:id/members/:userId` (remove); validates target user is a workspace member.
- `server/extensions/card/api/checklist.ts`: `POST /api/v1/cards/:id/checklist` (create), `PATCH /api/v1/checklist-items/:id` (update title/checked/position), `DELETE /api/v1/checklist-items/:id`; uses fractional indexing for ordering.
- `server/extensions/card/api/dueDate.ts`: `GET /api/v1/workspaces/:id/cards/due?before=<ISO8601>` — returns assigned cards with due_date < before for the current user.
- `src/extensions/Card/components/LabelChip.tsx`: Colored label badge with optional remove button.
- `src/extensions/Card/components/LabelPicker.tsx`: Multi-select dropdown for attaching/detaching workspace labels.
- `src/extensions/Card/components/CardMemberAvatars.tsx`: Avatar stack for assigned card members.
- `src/extensions/Card/components/MemberAssignModal.tsx`: Modal for assigning/removing workspace members from a card.
- `src/extensions/Card/components/ChecklistProgress.tsx`: Progress bar showing checked/total count.
- `src/extensions/Card/components/ChecklistItem.tsx`: Single checklist row (toggle, inline rename, delete).
- `src/extensions/Card/components/ChecklistSection.tsx`: Full checklist widget with progress, items, and add form.
- `src/extensions/Card/containers/CardModal/sections/LabelsSection.tsx`: Label chips + picker section for CardModal.
- `src/extensions/Card/containers/CardModal/sections/MembersSection.tsx`: Member avatars + assign modal trigger for CardModal.
- `src/extensions/Card/containers/CardModal/sections/ChecklistSection.tsx`: Thin wrapper delegating to ChecklistSection component.
- `src/extensions/Card/containers/CardModal/sections/DueDateSection.tsx`: Date input section for setting/clearing card due date.
- `specs/tests/card-extended-fields.test.ts`: 15 unit tests covering label limit, member-in-workspace check, label workspace match, checklist ordering, due date filtering, includes shape, and hex color validation.
