## Technical Debt

- `RunLogRow.tsx` module-level constant objects (`STATUS_ICON`, `TYPE_ICON`) look up translation values at module initialisation time.  This is safe with a static JSON import but would need refactoring if runtime locale switching were ever introduced.
- Dynamic strings that contain runtime values (e.g. quota counts, overflow badge counts) are kept as JS template literals; only the static surrounding text was moved into translations.
- Several pre-existing TypeScript errors in Automation (`AutomationQuota` export mismatch, `boardId` prop in `ActionList`, `configFieldRenderer` exactOptionalPropertyTypes) were not introduced by this iteration and remain unaddressed to stay within scope.
