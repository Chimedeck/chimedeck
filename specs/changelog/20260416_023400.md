# Changelog — 2026-04-16T02:34:00Z

## Update

- `server/extensions/attachment/workers/thumbnail.ts`: Added GIF early-return path. When `mime_type` is `image/gif`, the worker now persists `width` and `height` from Sharp metadata then returns immediately — no WebP thumbnail is generated and no `thumbnail_key` is written to the DB. This preserves GIF animation when the file is later served via the `/view` endpoint.

## New

_Nothing new added._

## Technical Debt

- Existing GIF attachments uploaded before this change will have `thumbnail_key = NULL` and `width = NULL` / `height = NULL`. Re-processing them requires re-queuing the thumbnail worker (out of scope for this sprint).
