# Changelog — 2026-03-10T03:56:50Z

## Update

- `server/middlewares/permissionManager.ts` — extended `Role` type to include `GUEST` (rank 0, below all workspace roles) and updated `ROLE_RANK` map accordingly.
- `server/extensions/board/api/patch.ts`, `delete.ts`, `archive.ts`, `duplicate.ts`, `labels.ts` — integrated `guestGuard` call after `requireWorkspaceMembership` on all mutation handlers so GUEST-role callers receive `403 { name: "guest-role-insufficient-permissions" }` with a clear message.
- `server/extensions/board/api/index.ts` — added three new guest routes: `POST /api/v1/boards/:id/guests`, `GET /api/v1/boards/:id/guests`, `DELETE /api/v1/boards/:id/guests/:userId`.

## New

- `db/migrations/0030_guest_role.ts` — migration that extends the `memberships.role` CHECK constraint to include `GUEST` and creates the `board_guest_access` join table (columns: `id`, `user_id`, `board_id`, `granted_by`, `granted_at`; unique constraint on `(user_id, board_id)`).
- `server/extensions/board/api/guests/index.ts` — implements guest invite (`handleInviteGuest`), revoke (`handleRevokeGuest`), and list (`handleListGuests`) handlers. Invite is idempotent; revoke also cleans up the GUEST workspace membership when no other board grants remain.
- `server/middlewares/guestGuard.ts` — lightweight synchronous middleware that returns `403 { name: "guest-role-insufficient-permissions" }` when `req.callerRole === 'GUEST'`. Must be called after `requireWorkspaceMembership`.
- `specs/tests/guest-role-access-control.md` — 9-scenario Playwright test spec covering invite, revoke, list, read-only access validation, idempotent duplicate invite, and edge cases (existing member conflict, non-ADMIN invite attempt, non-existent user).

## Technical Debt

- The `board_guest_access.id` is generated server-side via `randomUUID()`. A future migration could use a PostgreSQL `uuid_generate_v4()` default to make inserts DB-native.
- `guestGuard` is added individually to each mutation handler. If more mutation handlers are added in future, the guard must be added manually. A centralised middleware chain in the router would be more robust.
