# Changelog — 20260302_042516

## Update
- Created `src/extensions/Workspace/` frontend extension following the sample-project component/container/duck pattern.

## New
- **`api.ts`** — Typed API client functions for all workspace endpoints (CRUD, invite creation/inspection/acceptance, member list/role-update/removal).
- **`routes.ts`** — Route configuration for `WorkspacePage` (`/workspace`, `/workspace/:workspaceId`) and `AcceptInvitePage` (`/invites/:token/accept`).
- **`translations/en.json`** — English translation strings for all Workspace UI copy.
- **`components/RoleBadge.tsx`** — Badge component that renders a styled label for each role (`OWNER`, `ADMIN`, `MEMBER`, `VIEWER`).
- **`components/WorkspaceSwitcher.tsx`** — Dropdown for switching between the current user's workspaces; handles loading and empty states.
- **`components/InviteMemberModal.tsx`** — Modal form for inviting a new member; surfaces invite errors (insufficient role, generic).
- **`components/MemberList.tsx`** — Table of workspace members with inline role-change selects and remove actions; guards against last-owner removal.
- **`containers/WorkspacePage/WorkspacePage.duck.ts`** — Redux Toolkit slice with thunks for all workspace API actions and selectors for each state field.
- **`containers/WorkspacePage/WorkspacePage.tsx`** — Main workspace management page wiring together the switcher, member list, and invite modal.
- **`containers/AcceptInvitePage/AcceptInvitePage.tsx`** — Page for accepting an invite token; handles expired (410) and already-used (409) error states.

## Technical Debt
- `AcceptInvitePage` uses a `declare const api` placeholder; it should receive the `api` instance from the Redux thunk extras or a React context once the full frontend app is wired.
- `WorkspacePage` uses a `PLACEHOLDER_CURRENT_USER_ID` constant; replace with the real authenticated user selector from the auth duck.
- Route config references `~/config/routeTypes` which does not exist yet in this project — define `RouteConfig` type when the router is set up.
