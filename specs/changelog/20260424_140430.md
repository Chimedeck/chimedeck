# Update
- Added a board-level notification preference flag `only_related_to_me` and wired it into notification fanout filters.
- Updated board notification preference API (`GET`/`PATCH /api/v1/boards/:boardId/notification-preference`) to return and persist `only_related_to_me`.
- Updated board settings UI to include a new toggle: "Only notify events related to me".

# New
- Added migration `0117_board_notification_related_only.ts` to extend `board_notification_preferences` with `only_related_to_me` (default `false`).
- Added `relatedCardRecipients` helper to resolve whether a recipient is related to a card by:
  - card assignment (`card_members`), or
  - checklist assignment (`checklist_items.assigned_member_id`).
- Added related-only delivery rules to activity/direct dispatch paths:
  - `card_created` blocked when related-only mode is enabled.
  - `card_moved`, `card_member_assigned`, `card_member_unassigned`, `checklist_item_assigned`, `checklist_item_unassigned`, `checklist_item_due_date_updated`, `card_updated`, `card_deleted`, `card_archived` require card/checklist relation.
  - `card_commented` allows card/checklist relation OR reply target (`replyToUserId`).
- Added reply-target propagation from comment create API to direct comment notification dispatch payload.

# Technical Debt
- `boardActivityDispatch.ts` still reports broad pre-existing no-unsafe lint debt unrelated to this behavior change.
- No dedicated integration test exists yet for the new related-only matrix across all notification types.
