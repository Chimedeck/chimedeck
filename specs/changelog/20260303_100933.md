# Changelog — 20260303_100933

## Update

### Search 500 Error Fixed
- **Root cause**: `server/extensions/search/api/query.ts` was referencing a non-existent `archived` boolean column on the `boards` table. Boards use a `state` enum (`'ACTIVE'` | `'ARCHIVED'`) instead.
- **Fix**: Changed board search query to select `state` instead of `archived`, and filter with `WHERE state = 'ACTIVE'` instead of `WHERE archived = false`.
- **Files**: `server/extensions/search/api/query.ts`

### Board Settings Click-Outside Close
- **Problem**: Opening the ··· board settings dropdown had no way to dismiss it except clicking the button again.
- **Fix**: Added an invisible `role="presentation"` full-screen backdrop div (`fixed inset-0 z-10`) behind the dropdown. Clicking anywhere outside the menu triggers `setMenuOpen(false)`.
- **Files**: `src/extensions/Board/components/BoardHeader.tsx`

## Technical Debt

- The `connected` prop on `BoardHeader` is marked deprecated but not yet removed — will need a follow-up cleanup sprint.
- Search does not yet support trigram (fuzzy) matching — only exact `to_tsquery` full-text. Cards/boards need the `pg_trgm` extension and `GIN` index for fuzzy prefix search.
- `SearchModal` uses a polling/debounce model; could be moved to a React Query or SWR cache to reduce redundant fetches.
