## New
- `server/extensions/search/common/searchLogger.ts` — structured observability logger for the search extension. Emits JSON log lines to stdout for: `search.request`, `search.results`, `search.permission_filter_applied`, `search.permission_denied`, `search.feature_disabled`, `board_search.request`, `board_search.results`, `board_search.access_denied`, `board.access_checked`. Never logs sensitive data (no `q`, no titles).
- Logging integrated into `server/extensions/search/api/query.ts`: logs `search.permission_denied` on auth/membership failures, `search.request` on valid requests, `search.results` with result count and hasMore on success.
- Logging integrated into `server/extensions/search/mods/queryWorkspaceSearch.ts`: logs `search.permission_filter_applied` before executing board-access-filtered SQL queries.
- Logging integrated into `server/extensions/search/api/getBoardSearch.ts`: logs `board_search.access_denied` on access errors, `board_search.request` and `board_search.results` on successful queries.
- Logging integrated into `server/extensions/board/api/get.ts`: logs `board.access_checked` with `result: "denied"` or `"allowed"` for every board GET, enabling access regression detection in production logs.

## Technical Debt
- Log lines are written to stdout via `console.info` — suitable for container log collection. A future iteration could route them to a dedicated metrics sink (Prometheus counters, OpenTelemetry spans) for alerting.
- The `callerRole` field is `undefined` in the first `applyBoardVisibility` error branch of `get.ts` because the middleware failed before populating the role; this is a known gap and is acceptable for logging purposes.
