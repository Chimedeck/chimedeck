## New

- `package.json` — Bun runtime, scripts (`dev`, `lint`, `typecheck`, `db:migrate`, `db:studio`, `test`), dependencies (`@prisma/client`, `node-cache`), devDependencies (`eslint`, `typescript`, `prettier`, `prisma`, `bun-types`).

- `.env.example` — all required and optional env vars documented, including Redis and non-Redis modes and feature flag sources.
- `Dockerfile` — Bun image, installs deps, runs Prisma generate, starts server.
- `server/index.ts` — `Bun.serve` entry point; loads feature flags, routes `/health` and `/api/v1/flags`, applies security headers and request logging.
- `server/config/env.ts` — single file centralising all `Bun.env` access.
- `server/config/app.ts` — app config (port, isDev).
- `server/config/flags.ts` — wires env vars into `flagsConfig` consumed by flags module.
- `server/mods/logger.ts` — request logging middleware.
- `server/mods/helmet.ts` — security headers middleware.
- `server/middlewares/parser/index.ts` — JSON body parser.
- `server/mods/openapi/index.ts` — route registry stub for documentation.
- `server/mods/flags/types.ts` — `FlagProvider` interface and `FlagContext` type.
- `server/mods/flags/defaults.ts` — hardcoded fallback values for all known flags.
- `server/mods/flags/index.ts` — singleton `flags` object backed by `CompositeFlagProvider`.
- `server/mods/flags/providers/env.ts` — reads `FLAG_<KEY>=true|false` from `Bun.env`.
- `server/mods/flags/providers/json.ts` — parses JSON file at `FEATURE_FLAGS_JSON_PATH`, hot-reloads every 30 s.
- `server/mods/flags/providers/flagsmith.ts` — Flagsmith SDK stub (no-op until SDK installed).
- `server/mods/flags/providers/featbit.ts` — FeatBit SDK stub (no-op until SDK installed).
- `server/mods/flags/providers/composite.ts` — merges sources: defaults → json → env → remote.
- `server/mods/flags/flags.test.ts` — unit tests for flag defaults.
- `prisma/schema.prisma` — baseline schema with `datasource` + `generator` (no models yet).

## Technical Debt

- `flagsmith.ts` and `featbit.ts` are stubs; actual SDK calls are commented out with `TODO`. SDKs will be wired in when feature flag providers are needed.
- `server/middlewares/parser/index.ts` uses a loose type signature to avoid a Bun/undici type conflict; should be tightened once `bun-types` exposes a proper `Request.clone()` return type.
- Prisma client is generated but not used yet (no models in schema). `db:migrate` requires a running Postgres instance.
