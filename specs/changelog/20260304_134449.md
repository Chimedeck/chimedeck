# Changelog — 2026-03-04

## Update
- Added architecture documentation for the Plugin System.

## New
- `specs/architecture/plugins.md` — full architecture spec covering:
  - Plugin data model (`plugins`, `board_plugins`, `plugin_auth_tokens` tables)
  - Plugin manifest (`manifest.json`) format and capability contract
  - `connector.html` entry point pattern (hidden iframe + SDK + client.js)
  - `jhInstance` SDK API surface (compatible with Trello Power-Up shape, wraps `postMessage` protocol)
  - Plugin-scoped data storage (`t.get` / `t.set` with `shared` / `private` visibility)
  - Plugin Admin Dashboard — board-admin-gated UI for enabling/disabling plugins per board
  - Plugin Discovery Marketplace — data layer ready (`is_public`, `categories`, `capabilities`), no UI yet
  - Board-admin permission gate for all enable/disable actions
  - Full REST API surface for plugin management, plugin data storage, registry, and SDK delivery
  - Runtime message flow (postMessage between host board UI and plugin iframes)
  - Server folder structure for `server/extensions/plugins/`
  - Security model (deny-first, api_key scoping, origin isolation, private data gating, reserved auth token table)

## Technical Debt
- `plugin_auth_tokens` table and `t.getRestApi()` are reserved but not implemented — plugins currently authenticate via `api_key` only.
- The plugin marketplace UI (`is_public` plugins) is not built — data schema is ready.
- The `jhInstance` SDK (`server/extensions/plugins/sdk/jh-instance.ts`) needs to be implemented and built to a static bundle.
- No UI for the plugin admin dashboard has been built yet.
