# Sprint 59 — Iteration 1: DB Migration + MIME Allowlist

## Update
- `server/extensions/attachment/api/requestUploadUrl.ts`: Body parsing and validation now run immediately after auth (before any DB lookup). Disallowed MIME types return `{ name: 'mime-type-not-allowed' }` (400); files exceeding 100 MB return `{ name: 'file-too-large' }` (413).

## New
- `db/migrations/0033_attachments_enhanced.ts`: Adds `thumbnail_key`, `content_type`, `width`, `height`, `scan_completed_at`, `scan_vendor` columns to the `attachments` table.
- `server/extensions/attachment/config/allowedTypes.ts`: Centralised `ALLOWED_MIME_TYPES` array and `MAX_FILE_SIZE_BYTES` (100 MB) constant.
- `tests/integration/attachments/mimeValidation.test.ts`: 11 integration tests covering allowlist exports, disallowed MIME → 400, oversized files → 413, and boundary/valid-type pass-through.
