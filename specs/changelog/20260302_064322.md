# Changelog — 20260302_064322

## Update

- `src/extensions/Board/containers/BoardPage/BoardPage.tsx` — wired real-time sync, presence avatars, connected indicator, and drag-abort-on-disconnect logic.

## New

### WebSocket client layer (`src/extensions/Realtime/client/`)
- `socket.ts` — singleton `RealtimeSocket` class with reconnect exponential backoff (1 s → 30 s cap), subscribe/unsubscribe event handlers, `isConnected` property, and room-subscribe message on open.
- `messageQueue.ts` — in-memory `MessageQueue` with FIFO enqueue/dequeue, overflow detection (>100 ops → clear + notify), and FIFO replay order.

### React hooks (`src/extensions/Realtime/hooks/`)
- `useWebSocket.ts` — manages connection lifecycle (connect on mount, disconnect on unmount); on reconnect: fetches missed events via `GET /api/v1/boards/:id/events?since=`, then replays queued mutations in order with 409/422 discard logic.
- `useBoardSync.ts` — subscribes to incoming WS events, deduplicates by `type:sequence` key, and dispatches Redux actions to `listSlice` and `cardSlice` for reconciliation.

### Redux middleware (`src/extensions/Realtime/middleware/`)
- `wsMiddleware.ts` — intercepts actions with `meta.optimistic = true`, fires HTTP request, dispatches `CONFIRM` on 2xx or `ROLLBACK` on 4xx/5xx/network error; skips confirm if WS already confirmed own mutation (deduplication).

### Entity slices
- `src/extensions/List/listSlice.ts` — Redux slice for List entities: `hydrate`, `applyOptimistic`, `confirmOptimistic`, `rollbackOptimistic`, `applyOptimisticReorder`, `rollbackOptimisticReorder`, and WS event reducers (`remoteCreate`, `remoteUpdate`, `remoteArchive`, `remoteReorder`).
- `src/extensions/Card/cardSlice.ts` — Redux slice for Card entities: same pattern + `applyOptimisticMove`, `rollbackOptimisticMove`, `markConflicted`, `clearConflicted`, and WS event reducers (`remoteCreate`, `remoteUpdate`, `remoteMove`, `remoteArchive`).

### Presence UI (`src/extensions/Realtime/components/`)
- `PresenceAvatars.tsx` — avatar row with deterministic colour assignment; seeded from `GET /api/v1/boards/:id/presence`, updated via `presence_update` WS events; overflow badge for >5 users.
- `UserCursor.tsx` — optional component wrapping any card with a coloured outline + name badge when another user is editing it.

### Tests (`src/extensions/Realtime/__tests__/`)
- `wsMiddleware.test.ts` — unit tests: pass-through for non-optimistic actions, synchronous optimistic apply, CONFIRM on 2xx, ROLLBACK on 4xx, ROLLBACK on network error (17 tests total, all pass).
- `messageQueue.test.ts` — unit tests: FIFO order, peek, clear, overflow handler, size accounting.
- `sync.integration.test.ts` — integration tests: two simulated clients converge after `remoteCreate`, `remoteReorder`, `remoteMove`; optimistic move rollback; deduplication correctness.
