# Sprint 11 — Comments & Activity Log

## Update
- Registered `commentRouter` and `activityRouter` in `server/index.ts`.

## New
- **Migration** `db/migrations/0010_comments_activity.ts`: Creates `comments` and `activities` tables with proper indexes and FK constraints.
- **`server/extensions/activity/mods/write.ts`**: Append-only `writeActivity()` — no UPDATE or DELETE ops on activities table.
- **`server/extensions/comment/api/create.ts`**: `POST /api/v1/cards/:id/comments` — adds comment, writes Event + Activity rows, broadcasts WS `comment_added` event.
- **`server/extensions/comment/api/update.ts`**: `PATCH /api/v1/comments/:id` — owner-only edit, increments `version`, records `comment_edited` activity.
- **`server/extensions/comment/api/delete.ts`**: `DELETE /api/v1/comments/:id` — soft-delete (owner or ADMIN), sets `deleted: true` and `content: "[deleted]"`, records `comment_deleted` activity.
- **`server/extensions/comment/api/index.ts`**: Comment router.
- **`server/extensions/activity/api/cardActivity.ts`**: `GET /api/v1/cards/:id/activity` — returns all activity for a card, VIEWER+.
- **`server/extensions/activity/api/boardActivity.ts`**: `GET /api/v1/boards/:id/activity` — cursor-paginated board activity feed, VIEWER+.
- **`server/extensions/activity/api/index.ts`**: Activity router.
- **`src/extensions/Comment/components/CommentEditor.tsx`**: Markdown textarea for composing/editing comments.
- **`src/extensions/Comment/components/CommentDeletedItem.tsx`**: "[deleted]" placeholder for soft-deleted comments.
- **`src/extensions/Comment/components/CommentItem.tsx`**: Single comment with inline edit/delete, shows version badge.
- **`src/extensions/Comment/components/CommentThread.tsx`**: Full comment thread with CommentEditor for new comments.
- **`src/extensions/Activity/components/ActivityItem.tsx`**: Human-readable description generator for activity events (card_moved, comment_added, etc.).
- **`src/extensions/Activity/components/ActivityFeed.tsx`**: Paginated timeline of activity events with Load More support.
- **`specs/tests/comments-activity.test.ts`**: 16 unit tests covering: activity immutability, comment version increment, soft-delete shape, RBAC (owner/ADMIN), cursor pagination edge cases, and ActivityItem description generation.

## Technical Debt
- Activity backfill for sprint 03–08 mutations (card CRUD, list CRUD, board CRUD, etc.) should be added — current handlers already call `writeEvent` for real-time but not `writeActivity`. Add `writeActivity` calls alongside the existing `writeEvent` calls in all prior mutation handlers.
- The `activities` table has no DB-level trigger preventing UPDATE/DELETE — enforcement is application-only. A trigger should be added for defense-in-depth.
