# Changelog — 20260302_120000

## Update
- `server/index.ts`: Mounted the workspace router under `/api/v1/workspaces` and `/api/v1/invites`.

## New
- `server/extensions/workspace/common/config/invite.ts`: Centralised invite TTL (default 48 h, overridable via `INVITE_TTL_SECONDS` env var) and cache key prefix.
- `server/extensions/workspace/mods/invite/create.ts`: Invite creation mod — generates UUID token, persists to `invites` table, caches in memCache with TTL.
- `server/extensions/workspace/mods/invite/validate.ts`: Invite validation mod — checks DB for existence, expiry, and `accepted_at`.
- `server/extensions/workspace/mods/invite/consume.ts`: Invite consumption mod — marks `accepted_at`, upserts membership row, evicts cache; all in one DB transaction.
- `server/extensions/workspace/api/create.ts`: `POST /api/v1/workspaces` — creates workspace and OWNER membership in a single transaction.
- `server/extensions/workspace/api/get.ts`: `GET /api/v1/workspaces` (list) and `GET /api/v1/workspaces/:id` (detail), with VIEWER-level RBAC.
- `server/extensions/workspace/api/update.ts`: `PATCH /api/v1/workspaces/:id` — rename workspace; requires ADMIN role.
- `server/extensions/workspace/api/delete.ts`: `DELETE /api/v1/workspaces/:id` — hard-delete workspace; requires OWNER role.
- `server/extensions/workspace/api/invite/create.ts`: `POST /api/v1/workspaces/:id/invite` — creates invite; requires ADMIN role.
- `server/extensions/workspace/api/invite/inspect.ts`: `GET /api/v1/invites/:token` — public endpoint returning invite metadata, expiry, and used status.
- `server/extensions/workspace/api/invite/accept.ts`: `POST /api/v1/invites/:token/accept` — authenticated accept; returns 410 for expired, 409 for already-used.
- `server/extensions/workspace/api/members/list.ts`: `GET /api/v1/workspaces/:id/members` — lists members with role; requires VIEWER.
- `server/extensions/workspace/api/members/updateRole.ts`: `PATCH /api/v1/workspaces/:id/members/:userId` — changes role; requires ADMIN; enforces last-owner invariant (409).
- `server/extensions/workspace/api/members/remove.ts`: `DELETE /api/v1/workspaces/:id/members/:userId` — removes member; requires ADMIN; enforces last-owner invariant (409).
- `server/extensions/workspace/api/index.ts`: Workspace router — pattern-matches all 11 workspace/invite routes and delegates to handlers.
