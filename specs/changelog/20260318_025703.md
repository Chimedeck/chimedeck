# Changelog — 2026-03-18T02:57:03Z

## Update
- `src/layout/AppShell.tsx`: Mobile sidebar overlay now always renders (for CSS transitions) with `pointer-events` toggle. Added Escape-key close + focus-trap `useEffect` for the drawer. Added `useLocation` effect to auto-close drawer on route change. Hamburger button gains `aria-expanded`, `aria-controls`, and `data-testid="mobile-sidebar-toggle"`. Backdrop gains `data-testid="mobile-sidebar-backdrop"`. Drawer panel gains `role="dialog"`, `aria-modal="true"`, `aria-label="Navigation"`, `id="mobile-sidebar"`, `ref={drawerRef}`, and slide-in transition classes.
- `src/layout/Sidebar.tsx`: Added `SidebarProps` interface with `onClose?: () => void`. In mobile drawer mode (`isMobile=true`) the sidebar always renders expanded and hides the collapse toggle. Added `onNavigate` prop to `NavItem` and `NavButton` so clicking a nav item in the drawer triggers `onClose`.

## New
- Mobile off-canvas drawer with 300ms slide-in/out transition via Tailwind `translate-x`.
- Focus trap inside mobile drawer: Tab/Shift+Tab cycles within drawer focusable elements only.
- Escape key closes mobile drawer (desktop sidebar unaffected).
- Focus auto-moves to first focusable element when drawer opens.
- Drawer closes automatically on route change (location.pathname effect).

## Technical Debt
- No external focus-trap library used; the inline implementation covers common cases but does not handle dynamically inserted focusable elements after drawer open. Consider `focus-trap-react` if complex nested modals are added inside the drawer.
