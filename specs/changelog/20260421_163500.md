## Update
- Added schema-level short ID support for lists, comments, and attachments (not resolver-only).
- Extended entity ID resolution to support `lists`, `comments`, and `attachments` in addition to boards/cards.
- Updated list/comment/attachment route boundaries to resolve short IDs before handler execution.
- Updated board plugins and custom field router board-ID handling so `/api/v1/boards/:id/plugins` and `GET/POST /api/v1/boards/:id/custom-field-values` accept board short IDs.

## New
- New migration: `0110_list_comment_attachment_short_ids.ts`
  - Adds `short_id` columns to `lists`, `comments`, `attachments`
  - Adds unique partial indexes for each table
  - Backfills existing rows with generated 8-char short IDs
- Creation paths now assign short IDs for these entities:
  - List creation (`list/create`) and list duplication (`board/mods/duplicate`)
  - Comment creation (`comment/create`, `card/comments`, automation add-comment/mention-members actions)
  - Attachment creation (`attachment/requestUploadUrl`, `attachment/multipart/start`, `attachment/addUrl`, automation copy-to-board link attachment)

## Technical Debt
- Existing complexity lint warning in `comment/api/create.ts` remains pre-existing and was not part of this short-ID endpoint rollout.
- There is still no dedicated short-id support for checklist/checklist-item/label entity IDs themselves (only board/list/card/comment/attachment route handling covered here).
