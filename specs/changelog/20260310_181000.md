# Changelog — 2026-03-10T181000

## Update
- Applied `sanitizeText` (strip all HTML) to: card titles, board titles, list names, custom field `value_text` before DB writes.
- Applied `sanitizeRichText` (allow safe Markdown HTML subset) to: card descriptions, board descriptions, comment content before DB writes.

## New
- `server/common/sanitize.ts` — new module exporting `sanitizeText` and `sanitizeRichText` using `sanitize-html`.
- `sanitize-html@2.17.1` and `@types/sanitize-html@2.16.1` added as dependencies.
- `tests/e2e/input-sanitization.md` — 11 Playwright MCP markdown tests covering XSS stripping across all sanitized fields.

## Technical Debt
- `sanitizeRichText` currently allows raw HTML tags from clients. If the client renders Markdown to HTML client-side before sending, this is correct. If the server is expected to accept raw Markdown text (not HTML), a Markdown parser should be added upstream to convert before sanitizing.

## What Should Be Done Next
- Audit remaining user-facing string fields (e.g., label names, checklist item titles) and apply `sanitizeText` as needed.
- Add server-side rate limiting or length caps on rich-text fields to prevent DoS via oversized payloads.
