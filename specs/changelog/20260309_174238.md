# Changelog — 2026-03-09T17:42:38Z

## Sprint 44 — Admin Create External User API

### Update

- `server/config/env.ts` — Added `ADMIN_EMAIL_DOMAINS` (comma-separated, defaults to `"journeyh.io"`) and `ADMIN_INVITE_EMAIL_ENABLED` (boolean flag, defaults to `false`).
- `server/extensions/auth/api/index.ts` — Registered `POST /api/v1/admin/users` → `handleAdminCreateUser`.

### New

- `server/extensions/auth/common/isAdminEmailDomain.ts` — Helper that returns `true` when an email's domain is in `ADMIN_EMAIL_DOMAINS`. Fully independent of `isEmailDomainAllowed()` / Sprint 43.
- `server/extensions/auth/common/generatePassword.ts` — Cryptographically random 16-character alphanumeric password generator using `randomBytes`.
- `server/extensions/auth/api/adminCreateUser.ts` — `POST /api/v1/admin/users` handler: enforces admin-domain guard, validates input, auto-generates password when omitted, sets `email_verified = true`, optionally sends invite email via SES when all three conditions hold (`sendEmail && SES_ENABLED && ADMIN_INVITE_EMAIL_ENABLED`), returns `credentials.plainPassword` in response.
- `server/extensions/email/templates/adminInvite.ts` — Plain-text + HTML email template for admin-created account invitations.
- `tests/integration/auth/adminCreateUser.test.ts` — 23 integration tests covering helpers, authentication guards, admin-domain access control, input validation (invalid email, blank display name, weak password), and email-send flag logic.

### Technical Debt

- The `adminCreateUser` handler performs a DB query to fetch the inviter's display name for the email. If the caller's user row is missing (edge case with OAuth-only accounts), it falls back to the email address. A more robust solution would require ensuring all auth paths persist a name field.
