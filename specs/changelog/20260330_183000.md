# Changelog — 2026-03-30T18:30:00Z

## Update
- `server/index.ts`: Import and call `initSentry()` at the earliest point in server startup (before OTel and all other async init), guarded by the `SENTRY_SERVER_ENABLED` env flag.

## New
- `server/common/monitoring/sentry.ts`: Bun-compatible Sentry server init module. Exports `initSentry` (reads config from `server/config/env`, warns + skips if DSN missing), `captureError`, and `captureMessage` helpers. Mirrors the client-side encapsulation pattern. Privacy-safe by default (`sendDefaultPii: false`).

## Technical Debt
- `tracesSampleRate` is hardcoded to `1.0`; should be made configurable via env var once production traffic patterns are known.
- Error middleware integration (global capture of unhandled exceptions) is deferred to Iteration 5.

## What Should Be Done Next
