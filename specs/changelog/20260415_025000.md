# Changelog — 2026-04-15T02:50:00Z

## Update
- `server/extensions/webhooks/mods/dispatch.ts`: double-wrapped catch block so DB update failures inside the fire-and-forget IIFE are also swallowed silently.

## New
- `server/extensions/webhooks/mods/sign.ts`: HMAC-SHA256 signing utilities (`signPayload`, `buildSignatureHeader`) producing `t=<unix_seconds>,v0=<hex64>` header format per sprint-135 spec.
- `server/extensions/webhooks/mods/dispatch.ts`: fire-and-forget HTTP delivery — inserts a `webhook_deliveries` row synchronously, then sends the POST and updates status asynchronously without blocking the caller.
- `tests/unit/server/extensions/webhooks/mods/sign.test.ts`: 10 unit tests covering correct HMAC output, format, timestamp embedding, and edge cases.
- `tests/unit/server/extensions/webhooks/mods/dispatch.test.ts`: 7 unit tests covering non-blocking resolution, delivered/failed status transitions, request headers/body, and DB error resilience.
