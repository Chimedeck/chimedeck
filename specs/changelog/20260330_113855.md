# Changelog — 2026-03-30T11:38:55Z

## Iteration 6: Release Tagging and Source Map Pipeline

### Update

- `vite.config.ts`: Added `@sentry/vite-plugin` for source map upload, gated behind `SENTRY_AUTH_TOKEN`, `SENTRY_ORG`, and `SENTRY_PROJECT`. Plugin is omitted entirely in local/dev builds. Added `build.sourcemap: true` so maps are always emitted; plugin deletes them post-upload in deploy builds.
- `README.md`: Extended Sentry section with source map upload documentation, build-time env var table, deploy build command example, and explicit local dev skip behaviour.

### New

- `package.json`: Added `build:deploy` script — runs the standard build with `SENTRY_RELEASE` pre-seeded from the current git SHA, so the runtime SDK and source-map upload always use the same identifier.
- `@sentry/vite-plugin@5.1.1` added as a dev dependency.

### Technical Debt

- `SENTRY_RELEASE` and `VITE_SENTRY_RELEASE` must still be set separately (one for the server, one for the client at Vite build time). A future improvement could derive both from a single `RELEASE` var at the top of the CI pipeline.
- Source maps are currently generated for every local build too (not just deploy). This wastes a little disk space in dev. TODO: gate `sourcemap: true` on `SENTRY_AUTH_TOKEN` presence if bundle size matters.
