# Changelog — 20260324_085440

## Update

- `server/index.ts`: Imports and mounts `healthCheckExtensionRouter` after the `apiTokenRouter` call.

## New

- `server/extensions/healthCheck/common/validateUrl.ts`: URL validation utility. Enforces `http`/`https`-only scheme allow-list, rejects credentials in URLs, blocks `localhost`/`.local` hostnames, private IPv4 ranges (`10.x`, `172.16–31.x`, `192.168.x`, `127.x`, `169.254.x`) and private/loopback IPv6 addresses (`::1`, `fc00::/7`, `fe80::/10`).

- `server/extensions/healthCheck/api/presets.ts`: `GET /api/v1/health-check/presets` — returns the pre-configured service list from `health-check-services.json`. Auth: authenticated user only.

- `server/extensions/healthCheck/api/list.ts`: `GET /api/v1/boards/:boardId/health-checks` — returns all active health checks for a board with the latest probe result embedded per entry. Uses `DISTINCT ON` for efficient single-query latest-result retrieval.

- `server/extensions/healthCheck/api/create.ts`: `POST /api/v1/boards/:boardId/health-checks` — adds a new health check. Validates `name` (max 120 chars), validates and normalises `url` via `validateUrl`, enforces no duplicate URLs per board (case-insensitive, 409), rejects SSRF and invalid-scheme URLs (422).

- `server/extensions/healthCheck/api/remove.ts`: `DELETE /api/v1/boards/:boardId/health-checks/:id` — removes a health check (result history cascades via FK). Returns `200 { data: {} }` or `404`.

- `server/extensions/healthCheck/api/index.ts`: Routes dispatcher for all health-check paths. Handles `GET /api/v1/health-check/presets`, collection routes (`GET`/`POST`), and item routes (`DELETE`).

- `server/extensions/healthCheck/index.ts`: Extension entry point. When `HEALTH_CHECK_ENABLED=false`, all matching paths return `404` immediately (deny-first).

## Technical Debt

- `applyBoardVisibility` is called inside each handler (list, create, remove) rather than once at the router level. This mirrors the existing board API pattern but could be centralised in a future refactor.
- Probe endpoints (`/:id/probe`, `/probe-all`) and rate limiting are deferred to Iteration 4.
