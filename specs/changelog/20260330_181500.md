## Update
- `src/main.tsx`: Added `initSentry()` call at app bootstrap, before any React rendering, so all early errors are captured.
- `src/config/index.ts`: Added `sentryReplayEnabled` config field (`VITE_SENTRY_REPLAY_ENABLED`).

## New
- `src/common/monitoring/sentryClient.ts`: Encapsulates Sentry React SDK initialisation (`initSentry`), with `browserTracingIntegration` wired by default and opt-in Session Replay behind `sentryReplayEnabled`. Also exports `captureError` and `captureMessage` helpers for manual capture. All PII defaults are off (`sendDefaultPii: false`, Replay masks all text/media).
- `src/common/monitoring/README.md`: Documents client monitoring integration, feature flags table, usage patterns, manual smoke test checklist, and privacy policy for the module.
- `README.md`: Added `VITE_SENTRY_REPLAY_ENABLED` variable to the Sentry env table and a client bootstrap smoke test checklist section.

## Technical Debt
- Session Replay opt-in (`VITE_SENTRY_REPLAY_ENABLED`) is not yet exposed in `.env.example`; update it when the variable is promoted to standard config.
- `tracesSampleRate` is hardcoded to `1.0` — should be made configurable via a `VITE_SENTRY_TRACES_SAMPLE_RATE` env var before production use to avoid excessive quota consumption.
