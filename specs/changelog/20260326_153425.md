# Changelog — 2026-03-26 15:34

## Update

Migrated all `.tsx` files under `src/extensions/Board/` from hardcoded Tailwind colour classes to semantic design token classes. Every `bg-slate-*`, `bg-gray-*`, `text-slate-*`, `text-gray-*`, `border-slate-*`, `border-gray-*` and all `dark:` variants have been replaced with the project's semantic token system.

## New

No new features introduced.

## Technical Debt

- **Button component adoption**: Four components now use `<Button>` from `src/common/components/Button.tsx` instead of bare styled buttons:
  - `BoardDeleteDialog` — Cancel (secondary), Delete board (danger)
  - `CreateBoardModal` — Cancel (ghost), Create (primary)
  - `BoardListPage` — Create Board (primary)
- **Theme exceptions documented**: Label-style status chips (`BoardStateChip` Active state, `VisibilityBadge` Workspace/Public badges) and the search highlight `<mark>` carry `// [theme-exception]` comments explaining the intentional colour override.
- `ring-offset-slate-900` on toggle switches migrated to `ring-offset-bg-base`.

## What Should Be Done Next

- Audit remaining extensions (`Card/`, `List/`, `Automation/`, etc.) for the same hardcoded palette and apply the same migration pattern.
- Consider extracting the toggle-switch pattern (used in BoardNotificationToggle and BoardNotificationTypePreferences) into a shared `ToggleSwitch` component in `src/common/components/`.
