# Sprint 54 — Timeline Bar Rendering + Drag/Resize Interactions

## Update

- `TimelineRow.tsx` — now renders `TimelineBar` for every scheduled card (cards with both `start_date` and `due_date`) inside the bar area. Calls `useTimelineDrag` to get drag overrides and interaction handlers.
- `TimelineView.tsx` — passes `addToast` down to `TimelineRow` via conditional spread (respects `exactOptionalPropertyTypes`).
- `types.ts` — added `TimelineDragOverride`, `TimelineBarProps`, `UseTimelineDragOptions`, and `UseTimelineDragResult`; added `addToast` to `TimelineRowProps`.

## New

- `TimelineBar.tsx` — coloured bar spanning `start_date` to `due_date`. Has a left resize handle (changes `start_date`), a right resize handle (changes `due_date`), and a body drag area (shifts both). Uses label colour when available; falls back to blue. Includes `// TODO: dependency arrows` comment as required by spec.
- `useTimelineDrag.ts` — hook managing all drag/resize interactions. Uses `document` mouse event listeners so drag works even when the cursor leaves the handle. Applies optimistic Redux updates on `mouseup`, calls `PATCH /api/v1/cards/:id`, and reverts both Redux fields + shows an error toast on failure. Stores `dayWidth` in a ref so zoom changes mid-drag don't break calculations.
- `tests/e2e/timeline-bar.spec.ts` — E2E tests for bar rendering: bar appears for scheduled cards, correct handles are present, title shown, bars appear in the correct swimlane, unscheduled cards are NOT rendered as bars.
- `tests/e2e/timeline-drag.spec.ts` — E2E tests for drag/resize: right handle extends `due_date`, left handle moves `start_date`, body drag shifts both dates by equal delta, PATCH failure reverts and shows error toast.

## Technical Debt

- Bars in the same swimlane can visually overlap when multiple scheduled cards have overlapping date ranges. Row-level stacking is deferred.
- Drag interactions rely on `document` mouse events; touch/pointer events are not yet handled.

## What Should Be Done Next

- Implement dependency arrows between bars (as noted by the `// TODO` in `TimelineBar.tsx`).
- Add row-level stacking to avoid overlapping bars in the same swimlane.
- Add keyboard-accessible drag/resize for accessibility compliance.
