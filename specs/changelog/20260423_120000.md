## Update
- Reduced board drag-time CPU overhead by removing per-card auth selector subscriptions in card tiles.
- Reduced repeated drag hit-testing work by caching pointer-resolved list id across collision/drag handlers.
- Optimized list hit resolution to prefer cached board list rectangles and only fall back to DOM hit-testing when needed.

## New
- Added `currentUserId` prop threading path: BoardPage -> BoardCanvas -> SortableListColumn -> CardItem.
- Added `livePointerListIdRef` in BoardCanvas to reuse per-frame pointer list resolution during drag.

## Technical Debt
- BoardCanvas drag handlers remain high in cognitive complexity and should still be split into smaller pure helpers.
- Profiling in dev mode still includes React JSX validation overhead; a production-build profile is needed to isolate runtime-only gains.

## What Should Be Done Next
- Re-run Chrome Performance and React Profiler on the same 300+ card board scenario and compare p95 long-task duration before/after this change.
- If layout cost remains high, add a drag-time list-rect refresh strategy tied to horizontal scroll events.
- If render cost remains high, evaluate wiring list/card virtualization in the active ListColumn render path.
