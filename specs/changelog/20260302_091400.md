# Changelog — Sprint 20: Real-Time UI

**Date:** 2026-03-02

---

## Update

- `src/extensions/Board/components/BoardHeader.tsx` — replaced simple connection dot with `ConnectionBadge` (3-state: Live / Reconnecting… / Offline); added `connectionState` prop.
- `src/extensions/Board/containers/BoardPage/BoardPage.tsx` — wired up `useWebSocket` + `useBoardSync` for live board sync; added toast state management + `ToastRegion`.
- `src/extensions/Board/slices/boardSlice.ts` — added `remoteCardMove` and `updateList` actions for WS event handling.
- `src/extensions/Card/slices/cardDetailSlice.ts` — added `remoteUpdate` action so the open CardModal reflects live edits from other users.
- `src/extensions/Card/translations/en.json` — added realtime-related translation keys.
- `src/extensions/Realtime/hooks/useWebSocket.ts` — now returns `connectionState: 'connected' | 'reconnecting' | 'offline'` in addition to the legacy `connected` boolean.
- `src/extensions/Realtime/hooks/useBoardSync.ts` — extended to dispatch WS events to both entity slices (`listSlice`, `cardSlice`) AND `boardSlice` (for board rendering) and `cardDetailSlice` (for open modal).
- `src/reducers.ts` — added `listReducer` and `cardsReducer` exports.
- `src/store/index.ts` — registered `lists` and `cards` reducers; added `wsMiddleware` to the Redux middleware chain.

## New

- `src/common/components/ConnectionBadge.tsx` — pill badge with 3 connection states; used in `BoardHeader`.
- `src/common/components/Toast.tsx` — auto-dismissing toast notification item (4 s info/conflict, 6 s error).
- `src/common/components/ToastRegion.tsx` — fixed bottom-right toast stack container.
- `specs/tests/realtime-card-update.md` — Playwright spec: card moved in tab 1 appears in tab 2 within 1 s.
- `specs/tests/realtime-reconnect.md` — Playwright spec: ConnectionBadge transitions on disconnect/reconnect.
- `specs/tests/optimistic-rollback.md` — Playwright spec: card snaps back on API 500 with error toast.

## Technical Debt

- `boardSlice` and the entity slices (`listSlice`, `cardSlice`) hold overlapping card/list data. A future refactor should unify them so there is a single source of truth.
- `useWebSocket.fetchMissedEvents` is optional and not wired in `BoardPage` — missed-event re-sync after reconnect relies solely on the queue, not on a server-side catch-up endpoint.

## What Should Be Done Next

- Implement the `GET /api/v1/boards/:id/events?since=:seq` endpoint so the frontend can re-sync missed events after a reconnect gap.
- Add `src/extensions/Presence/` with `usePresence.ts` and `PresenceDot.tsx` for per-member online indicators.
- Write Playwright E2E tests for the three spec files added this sprint.
- Add conflict-detection logic to `boardSlice.applyOptimisticCardMove` to show a toast when a WS move beats a local drag in-flight.
