# Changelog — 2026-03-30T11:45:57Z

## Update

- `src/common/monitoring/sentryClient.ts`: Extended `beforeSend` to redact sensitive URL query-params (`token`, `access_token`, `api_key`, `secret`, `password`, `code`, `state`), strip sensitive request headers (`Authorization`, `Cookie`, `Set-Cookie`, `X-Auth-Token`, `X-API-Key`), and scrub breadcrumb `data.url` fields. Noise filtering logic is unchanged.
- `server/common/monitoring/sentry.ts`: Added `beforeSend` hook that redacts sensitive request headers, URL query-params, and top-level request body fields (`password`, `token`, `secret`, `credit_card`, `cvv`, `ssn`, etc.) before events reach Sentry ingest.
- `src/common/monitoring/README.md`: Replaced the brief privacy note with a full privacy policy table, per-module redaction constants guide, and an end-to-end manual validation checklist.
- `README.md`: Added a **Sentry privacy defaults** table summarising all active protections and a caveat about manual `captureMessage` / `captureError` calls.

## New

- Client-side `redactUrl`, `redactRequest`, `redactBreadcrumbs` helpers in `sentryClient.ts`.
- Server-side `redactUrl`, `redactRequest` helpers and `SENSITIVE_HEADERS`, `SENSITIVE_QUERY_PARAMS`, `SENSITIVE_BODY_FIELDS` constants in `server/common/monitoring/sentry.ts`.

## Technical Debt

- `TRACES_SAMPLE_RATE` (client) and `tracesSampleRate` (server) are both hardcoded to `1.0`. These should be made configurable via env vars once production traffic volume is known.
- Server `beforeSend` only scrubs shallow body keys. Nested credential objects (e.g. `{ card: { number: "…" } }`) require a recursive scrubber if any API accepts deeply nested payloads with sensitive data.

## What Should Be Done Next

- Add automated unit tests for `redactUrl`, `redactRequest`, and `redactBreadcrumbs` to prevent regressions as new sensitive field names are added.
- Consider a shared `SENSITIVE_QUERY_PARAMS` constant in `server/common/` or a shared package to keep client and server lists in sync.
- Complete the end-to-end privacy validation checklist (documented in `src/common/monitoring/README.md`) against a staging environment before first production deployment.
