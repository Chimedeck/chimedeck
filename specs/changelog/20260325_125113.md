# Changelog — 20260325_125113

## Update

- `src/common/utils/attachmentMarkdown.ts`: Updated `resolveAttachmentMarkdownUrl` to prefer `view_url` over raw S3 URLs for FILE-type attachments. Removed stale `attachment.url` fallbacks since `url` is no longer part of the `Attachment` type after the secure-proxy migration.
- `src/extensions/Card/components/CardDescriptionTiptap.tsx`: Added import of `rewriteS3UrlsToProxy`; applied rewrite in `getInitialEditorContent` and `buildPreviewMarkdown` so legacy S3 presigned URLs stored in card descriptions are silently rewritten to proxy paths before rendering.
- `src/extensions/Comment/components/CommentEditor.tsx`: Added import of `rewriteS3UrlsToProxy`; applied rewrite in both `getInitialEditorContent` and `resolvePendingHydratedContent` so legacy S3 URLs in comment bodies render via the authenticated proxy.

## New

- `src/common/utils/rewriteS3UrlsToProxy.ts`: New utility function `rewriteS3UrlsToProxy(content, attachments)` that scans Markdown/HTML content for raw S3 URLs (`*.amazonaws.com`) and rewrites them to the attachment's `view_url` proxy path. Matching is done first by direct URL comparison (with and without presigned query params) and then by S3 object key extraction — making it robust to expired or rotated presigned URLs.
- `tests/e2e/sprint-117-url-rewrite.md`: Playwright MCP test spec covering six scenarios: legacy S3 img src in description/comment renders via proxy; inserting attachment image into description/comment stores proxy path; non-S3 images are unaffected; attachment Comment action inserts proxy URL markdown.

## Technical Debt

- `src/common/utils/attachmentMarkdown.ts` still contains pre-existing TypeScript errors (`url` field references, `replaceAll` lib target) that pre-date this iteration and are not caused by these changes.

## What Should Be Done Next

- Verify end-to-end in a staging environment that all legacy card descriptions and comments with embedded S3 URLs now render images correctly via the proxy.
- Consider adding a server-side migration script to canonicalise stored content by replacing all S3 URLs with `attachment:name` placeholders, removing the need for client-side rewriting in the long term.
