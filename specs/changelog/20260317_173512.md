## Update

- Changed comment attachment persistence to store `attachment:<name>` placeholders instead of expiring presigned URLs.
- Hydrated comment attachments from the card attachment list before rendering comments and before reopening comments in the editor.
- Updated image insertion in the comment editor to use a real Tiptap image node so typing after an inserted image no longer corrupts the markdown payload.

## New

- Added shared comment attachment markdown helpers for placeholder encoding, hydration, dehydration, and safe pre-hydration fallback rendering.
- Wired the card activity feed to load card attachments for both the add-comment editor and rendered comment items.

## Technical Debt

- Placeholder resolution is keyed by attachment name as requested, so duplicate file names on the same card remain a collision risk.
- Workspace-wide `bun run typecheck` still reports unrelated pre-existing errors outside the touched comment files.

## What Should Be Done Next

- If duplicate attachment names become common, switch the placeholder format from name-based to a stable attachment identifier while keeping the same client hydration flow.
- Add a focused automated test around comment submit/reload for inline image attachments so the placeholder round-trip is covered without manual browser instrumentation.
