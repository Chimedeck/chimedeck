# Sprint 29 — Client Activity Feed UI (Iteration 2)

## Update
- `src/extensions/Card/slices/cardDetailSlice.ts`: Added `activities: ActivityData[]` field to state; modified `fetchCardDetailThunk` to request `/cards/:id?include=activities`; added `setActivities` action; added `selectCardDetailActivities` selector.
- `src/extensions/Card/components/CardModal.tsx`: Replaced `CommentThread` with `ActivityFeed`; added `activities` prop.
- `src/extensions/Card/containers/CardModal/index.tsx`: Reads `selectCardDetailActivities` from state; passes `activities` to `CardModal`.

## New
- `src/extensions/Card/config/activityEventsConfig.ts`: Client-side allowlist of visible system event types (`VISIBLE_ACTIVITY_EVENT_TYPES`), mirroring the server config.
- `src/extensions/Card/config/activityEventLabels.ts`: Maps each event type to a human-readable label and a Tailwind dot colour (`getActivityEventMeta`).
- `src/extensions/Card/containers/CardModal/ActivityFeed.tsx`: Unified activity feed that merges comments and system event rows, sorts them ascending by `created_at`, renders comment bubbles via `CommentItem` and system events as compact single-line rows with a coloured dot.
