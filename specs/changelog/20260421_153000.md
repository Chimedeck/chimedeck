## Update
- Extended short-id support in board/card-adjacent APIs that were still using raw route IDs.
- Added resolver-based normalization in realtime board events and board-view preference routing.
- Updated comment creation event/mention payloads to consistently use resolved card IDs.

## New
- Attachment API short-id compatibility now covers both standard and multipart flows:
  - `GET /api/v1/cards/:id/attachments`
  - `POST /api/v1/cards/:id/attachments/upload-url`
  - `POST /api/v1/cards/:id/attachments/url`
  - `POST /api/v1/cards/:id/attachments`
  - `POST /api/v1/cards/:id/attachments/multipart/start`
  - `POST /api/v1/cards/:id/attachments/multipart/part-url`
  - `POST /api/v1/cards/:id/attachments/multipart/complete`
  - `DELETE /api/v1/cards/:id/attachments/multipart/:uploadId`
- Internal attachment URL parsing now supports short card URL format `/c/:cardId[/slug]`.
