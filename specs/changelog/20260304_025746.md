## Update

### Sprint 29 — Configurable Events in Activity Feed (Server Side)

- `server/extensions/activity/config/visibleEventTypes.ts` (new): Created allowlist config exporting `VISIBLE_EVENT_TYPES` — the set of `action` values visible in the activity feed. Includes comment events, member add/remove, due date set/cleared, and `card.money.updated` (for Sprint 30).
- `server/extensions/activity/api/cardActivity.ts`: Applied `VISIBLE_EVENT_TYPES` filter (`.whereIn('action', VISIBLE_EVENT_TYPES)`) so `GET /api/v1/cards/:id/activity` only returns allowed event types.
- `server/extensions/activity/api/boardActivity.ts`: Applied same filter to `GET /api/v1/boards/:id/activity`.
- `server/extensions/card/api/get.ts`: Added `?include=activities` support — when requested, sideloads filtered activities for the card in `includes.activities`.
- `server/extensions/board/api/get.ts`: Added `?include=activities` support — when requested, sideloads filtered activities for all cards on the board in `includes.activities`.

## New

- `VISIBLE_EVENT_TYPES` config pattern: adding an event type to this list is the only change needed to surface it in all activity feeds.

## Technical Debt

- Comments use `event_type = null` or `action = 'comment_added'`; the filter uses `.whereIn('action', ...)` which excludes null rows. If legacy null-action comments exist they will be missed. A follow-up should add `.orWhereNull('action')` or migrate old rows.
