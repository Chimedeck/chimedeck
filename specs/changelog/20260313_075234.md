## Update
- Extended `server/extensions/notifications/mods/boardActivityDispatch.ts` to dispatch in-app notifications (DB insert + WS publish via `publishToUser`) alongside email for `card_created`, `card_moved`, and `card_commented` events. Actor is excluded from recipients; `preferenceGuard` is respected when `NOTIFICATION_PREFERENCES_ENABLED` is on.
- Added `?type=` query parameter filter to `GET /api/v1/notifications` — accepts any of the four valid types; invalid/absent param returns all notifications (backward compatible).

## New
- `db/migrations/0038_notification_types_extend.ts` — adds DB-level CHECK constraint on the `type` column of both `notifications` and `notification_preferences` tables, locking in the four valid values: `mention`, `card_created`, `card_moved`, `card_commented`.
- `tests/integration/notifications/boardActivityNotifications.test.ts` — 11 integration tests covering in-app dispatch contract, type mapping, fire-and-forget guarantee, and actor exclusion.
- `specs/tests/board-activity-notifications-server.md` — 9-scenario Playwright MCP spec for server-side in-app board activity notifications including preference guard and type filter scenarios.

## Technical Debt
