## Update
- Updated `server/extensions/board/api/members.ts` to act as a shim re-exporting all board member handlers from the new `members/` sub-directory.
- Updated `server/extensions/board/api/index.ts` to add POST `/members`, PATCH `/members/:userId`, and DELETE `/members/:userId` routes to the board router.
- Updated `server/extensions/board/api/list.ts` to filter workspace boards per caller role: GUESTs only see boards they have `board_guest_access` rows for; OWNER/ADMIN see all boards; MEMBER/VIEWER see WORKSPACE/PUBLIC boards plus PRIVATE boards they are in `board_members`.
- Updated `server/extensions/workspace/api/members/list.ts` to block GUEST callers with a 403 using `requireRole(scopedReq, 'VIEWER')`.

## New
- `server/extensions/board/api/members/get.ts` — `GET /api/v1/boards/:id/members`: lists explicit board members from the `board_members` table (replaces the old workspace-member listing).
- `server/extensions/board/api/members/create.ts` — `POST /api/v1/boards/:id/members`: adds a workspace member to the board with a given role; idempotent (updates role if already present).
- `server/extensions/board/api/members/update.ts` — `PATCH /api/v1/boards/:id/members/:userId`: changes a board member's role; enforces last-ADMIN invariant (409 if demoting the sole ADMIN).
- `server/extensions/board/api/members/remove.ts` — `DELETE /api/v1/boards/:id/members/:userId`: removes a board member; enforces last-ADMIN invariant.
- `server/extensions/board/api/members/index.ts` — barrel re-export of all four handlers.
- `specs/tests/sprint-78-iteration-3-board-members-api.md` — Playwright MCP test walkthrough for all CRUD operations and visibility filtering.

## Technical Debt
- The `board_members` table currently stores `board_id`/`user_id` UUIDs as plain strings (no FK enforcement at the query layer); relying on DB-level FK constraints added in migration `0040_board_members.ts`.
- `GET /api/v1/boards/:id/members` now returns `board_members` entries (explicit members only), not all workspace members. Any client code that relied on the old endpoint returning workspace members will need updating.
