## New

- `db/migrations/0025_email_change.ts` — adds `pending_email`, `email_change_token`, `email_change_token_expires_at` columns to `users`.
- `server/extensions/auth/api/changeEmail.ts` — `POST /api/v1/auth/change-email`: authenticated, password re-validated, rate-limited (3/hour), sends confirmation email when `EMAIL_VERIFICATION_ENABLED=true`, applies immediately when false.
- `server/extensions/auth/api/confirmEmailChange.ts` — `GET /api/v1/auth/confirm-email-change?token=`: verifies token, re-checks for email conflicts, commits change, invalidates all refresh tokens.
- `server/extensions/email/templates/emailChangeConfirmation.ts` — HTML + text email template for the change-confirmation link.
- `src/extensions/Auth/containers/ConfirmEmailChangePage/ConfirmEmailChangePage.duck.ts` — Redux slice for the confirmation page.
- `src/extensions/Auth/containers/ConfirmEmailChangePage/ConfirmEmailChangePage.tsx` — page that calls the confirm endpoint on mount and redirects to `/login` on success.
- `src/extensions/Auth/components/ChangeEmailForm.tsx` — form for requesting an email change (new email + current password).
- `src/extensions/Auth/components/EmailChangePending.tsx` — dismissible banner shown after the request is submitted.
- `specs/tests/change-email.md`, `change-email-taken.md`, `change-email-no-flag.md` — test specs for the new feature.
