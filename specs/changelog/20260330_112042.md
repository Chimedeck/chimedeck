# Changelog — 2026-03-30T11:20:42Z

## Update
- `src/main.tsx`: wrapped `<App />` with the new `<ErrorBoundary>` so no unhandled render crash produces a blank screen.
- `src/common/monitoring/sentryClient.ts`: added `beforeSend` noise filter that drops known un-actionable browser errors (ResizeObserver, network failures, cross-origin script errors, chunk-load errors, QuotaExceeded) before they reach Sentry's ingest.
- `src/common/monitoring/README.md`: documented the ErrorBoundary usage pattern and the noise-filtering table.

## New
- `src/common/monitoring/ErrorBoundary.tsx`: top-level React error boundary using `@sentry/react`'s `ErrorBoundary`. Catches render errors, reports to Sentry, and shows a recovery UI with "Try again" / "Reload page" actions. Can also be reused for feature-level boundaries.

## Technical Debt
- `IGNORED_ERROR_PATTERNS` in `sentryClient.ts` should be reviewed as real production noise is observed and new patterns added.
- Fallback UI in `ErrorBoundary.tsx` is generic; a product designer may want to brand it.
