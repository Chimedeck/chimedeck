# Changelog — 2026-03-05 16:26:31

## Update

- **PluginModal.tsx**: Extended `PluginModalState` with optional `boardPlugin` and `boardId` fields. When provided (board admin settings context), the `PluginAllowedDomainsPanel` is rendered below the plugin iframe. Removed `rounded-b-lg` from the iframe so it no longer rounds when the panel is present.

- **PluginDashboardPage.tsx**: `handleSettings` now passes `boardPlugin` and `boardId` into the `PluginModalState` so the settings modal can display the Allowed Domains panel.

- **usePluginBridge.ts**: Added `getEffectiveDomains`, `isDomainAllowed`, and `sendDomainError` helpers. `UI_MODAL` and `UI_POPUP` handlers now validate the target URL's origin against `boardPlugin.config.allowedDomains` before opening. Blocked origins receive a `domain-not-allowed` error response to the plugin's `SdkResponse` handler.

## New

- **PluginAllowedDomainsPanel.tsx** (`src/extensions/Plugins/components/`): New React component rendered inside the plugin settings modal for board admins. Displays a checkbox list of the plugin's `whitelistedDomains`. Initial check-state is inferred from `boardPlugin.config.allowedDomains` (null → all checked; array → only listed domains checked). The "Save domain settings" button is disabled while pristine (no changes). On submit, PATCHes `/boards/:boardId/plugins/:pluginId/allowed-domains` and shows inline success/error feedback.

## Technical Debt

- The `PluginAllowedDomainsPanel` does not yet propagate the updated `boardPlugin` config back to the Redux store after a successful save; a page reload is needed to reflect the new allowed domains in `usePluginBridge`. This can be addressed by dispatching a `loadPlugins()` refresh or by adding an `onSave` callback in a future iteration.

## What Should Be Done Next

- Refresh the `boardPlugins` list in Redux after a successful PATCH in `PluginAllowedDomainsPanel` so that `usePluginBridge` immediately enforces the new domain restriction without a page reload.
- Add E2E tests for the full settings → allowed domains → save flow.
- Consider UI/UX polish: show the connector URL origin as always-permitted (it is already allowed via `findPluginByOrigin`).
