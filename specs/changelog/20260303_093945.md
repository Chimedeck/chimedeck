# Changelog — 2026-03-03

## Update

### Bug Fixes (13 issues resolved)

#### 1. Label creation 500 error
**File:** `server/extensions/board/api/labels.ts`
- Removed `created_at` field from the INSERT object. The `labels` table migration schema only defines `id, workspace_id, name, color` — inserting `created_at` caused a PostgreSQL column-not-found error.

#### 2. WebSocket path mismatch
**File:** `server/extensions/realtime/api/index.ts`
- Server was rejecting upgrades because it checked `pathname !== '/ws'` but the client connects to `/api/v1/ws`. Fixed path check to `/api/v1/ws`.

#### 3. WebSocket token delivery
**File:** `server/extensions/realtime/api/index.ts`
- Browser WebSocket connections cannot send custom headers. Server only accepted `Authorization` header tokens. Added fallback to read the token from the `?token=` query parameter.

#### 4. Vite WebSocket proxy not forwarding upgrades
**File:** `vite.config.ts`
- Added `ws: true` to the `/api` proxy entry so Vite correctly proxies WebSocket upgrade requests from the dev frontend (port 5173) to the API server (port 3000).

#### 5. Wrong table name in realtime subscription handler
**File:** `server/extensions/realtime/api/index.ts`
- Changed `workspace_members` table reference to `memberships`, which is the actual table name defined in the DB migrations.

#### 6. Missing Radix UI `DialogTitle` accessibility requirement
**File:** `src/extensions/Card/components/CardModal.tsx`
- Added `<Dialog.Title className="sr-only">Card: {card.title}</Dialog.Title>` inside `<Dialog.Content>`. Radix UI requires every `DialogContent` to have an associated title for screen-reader accessibility.

#### 7. Label picker backdrop blocking Close button
**File:** `src/extensions/Card/components/CardLabels.tsx`
- The label picker renders a `<div className="fixed inset-0 z-10">` backdrop overlay. After a label was created successfully, `setPickerOpen(false)` was never called, leaving the backdrop in the DOM and intercepting all pointer events — including the card modal's Close button. Fixed by calling `setPickerOpen(false)` immediately after a successful `handleCreate`.

#### 8–14. WebSocket event type/payload mismatches (7 events)
The client `useBoardSync.ts` hook and Redux slice actions expected specific event type strings and payload shapes. The server was emitting wrong type names and/or stale/partial payload objects, causing Redux to crash with `Cannot read properties of undefined (reading 'id')`.

| File | Was | Now |
|------|-----|-----|
| `server/extensions/card/api/create.ts` | `{ listId, title }` | `{ card }` (full DB object) |
| `server/extensions/list/api/create.ts` | `{ title }` | `{ list }` (full DB object) |
