# Changelog — 20260310_071708

## Update

- **`src/extensions/Card/api.ts`** — Added `start_date: string | null` to the client-side `Card` interface to match what the server already returns in `GET /api/v1/boards/:id` (Sprint 46 field).
- **`src/extensions/BoardViewSwitcher/BoardViewTab.tsx`** — Added `data-testid="board-view-tab-{viewType}"` attribute to each tab button to support Playwright E2E tests.
- **`src/extensions/Board/containers/BoardPage/BoardPage.tsx`** — Replaced the TABLE view placeholder with a live `<TableView>` component, passing Redux card and list data.

## New

- **`src/extensions/TableView/types.ts`** — TypeScript types: `ColumnKey`, `SortDirection`, `SortState`, `TableColumn`, `TableRowData`, `TableViewProps`.
- **`src/extensions/TableView/useTableSort.ts`** — Custom hook that builds flat `TableRowData[]` from cards + lists, manages sort state (asc → desc → none cycle), and returns sorted rows.
- **`src/extensions/TableView/TableHeader.tsx`** — Sortable `<thead>` component; renders column headers with sort direction icons; accessible `aria-sort` attributes.
- **`src/extensions/TableView/TableRow.tsx`** — Renders a single card as a `<tr>` with cells for Title (clickable → opens card modal), List, Assignees (avatar initials), Labels (colour chips), Due Date (red if overdue), Start Date, and Value.
- **`src/extensions/TableView/TableView.tsx`** — Root table component; composes `TableHeader` + `TableRow` list; shows an empty-state message when no cards exist.
- **`src/extensions/TableView/README.md`** — Documents structure, usage, columns, sorting, data source, and extension points.
- **`tests/e2e/table-view.spec.ts`** — Playwright E2E tests covering: column headers render, card rows render, column sort cycling, and card-title click opens the card detail modal.

## Technical Debt

- The Table view is client-side sorted only; large boards (1000+ cards) may feel slow. Consider server-side sort/pagination for boards at that scale.
- Card detail modal auth flow in E2E tests uses `localStorage` token injection — depends on the app consuming `access_token` from `localStorage` on boot.
