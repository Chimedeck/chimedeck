# Sprint 09 — Real-Time Infrastructure

## Update

- `server/mods/cache/index.ts` — replaced the standalone in-process `memCache` implementation with a feature-flag-driven singleton (`cache: CacheProvider`) that resolves to `NodeCacheAdapter` (local dev) or `RedisCacheAdapter` (`USE_REDIS` flag + `REDIS_URL`). `memCache` is re-exported for backward compatibility.
- All board/list/card mutation handlers — replaced `console.log('[event] …')` stubs with real `writeEvent(…)` calls that persist to the `events` table and publish to the board's pub/sub channel.
- `server/extensions/board/api/index.ts` — added routes `GET /api/v1/boards/:id/events` and `GET /api/v1/boards/:id/presence`.
- `server/index.ts` — wired WebSocket upgrade (`handleWsUpgrade`) and handlers (`wsHandlers`) into `Bun.serve`.

## New

- `db/migrations/0008_events.ts` — creates `events` and `board_snapshots` tables.
- `server/mods/events/write.ts` — `writeEvent()`: persists an event row, triggers snapshot policy, and publishes to pub/sub.
- `server/mods/events/read.ts` — `readEventsSince()`: paged event query by board and sequence.
- `server/mods/events/snapshot/write.ts` — upsert `board_snapshots` by `board_id`.
- `server/mods/events/snapshot/read.ts` — `readSnapshotWithDelta()`: snapshot + delta events for client catch-up.
- `server/mods/events/snapshot/policy.ts` — `checkAndWriteSnapshot()`: every-50-events snapshot trigger.
- `server/mods/cache/types.ts` — `CacheProvider` interface.
- `server/mods/cache/adapters/nodeCache.ts` — `NodeCacheAdapter` (async) + `memCache` (legacy sync).
- `server/mods/cache/adapters/redis.ts` — `RedisCacheAdapter` using ioredis.
- `server/mods/pubsub/types.ts` — `PubSubProvider` interface.
- `server/mods/pubsub/adapters/inMemory.ts` — `InMemoryPubSubAdapter` (EventEmitter, single-node).
- `server/mods/pubsub/adapters/redis.ts` — `RedisPubSubAdapter` (ioredis, dual-client).
- `server/mods/pubsub/index.ts` — feature-flag-driven singleton `pubsub: PubSubProvider`.
- `server/mods/pubsub/publisher.ts` — `publisher.publish(boardId, message)` wrapper.
- `server/mods/pubsub/subscriber.ts` — `subscriber.subscribeBoard/unsubscribeBoard` wiring pub/sub → room broadcast.
- `server/extensions/realtime/mods/rooms/index.ts` — `rooms` registry (`Map<boardId, Set<ServerWebSocket>>`).
- `server/extensions/realtime/mods/rooms/subscribe.ts` — subscribe a WS client to a board room + presence key.
- `server/extensions/realtime/mods/rooms/unsubscribe.ts` — unsubscribe a WS client and clean up presence.
- `server/extensions/realtime/mods/rooms/broadcast.ts` — broadcast a message to all sockets in a room.
- `server/extensions/realtime/mods/heartbeat.ts` — 30s ping loop; evicts dead connections after 60s.
- `server/extensions/realtime/mods/auth.ts` — `verifyWsToken()` for WebSocket Bearer auth.
- `server/extensions/realtime/api/index.ts` — `handleWsUpgrade` + `wsHandlers` (open/message/close).
- `server/extensions/realtime/api/events.ts` — `GET /api/v1/boards/:id/events?since=`.
- `server/extensions/realtime/api/presence.ts` — `GET /api/v1/boards/:id/presence`.

## Technical Debt

- Snapshot state is copied verbatim from the last snapshot row rather than being recomputed by replaying events. A proper state-reduction step should be added in a later sprint.
- The heartbeat loop is started at module-load time in `realtime/api/index.ts`; this should be moved to an explicit server-start hook.
- `RedisCacheAdapter` and `RedisPubSubAdapter` are instantiated but not explicitly connected (`lazyConnect: true`). Connection health should be monitored.
