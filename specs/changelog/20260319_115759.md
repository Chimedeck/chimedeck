### Technical Debt

- The `Card` extension and several other pre-Sprint-90 extensions still contain hardcoded `aria-label` strings (e.g., `"Text formatting"`, `"Bold"`, `"Assign members"`). These were not in scope for Sprints 93–94 and should be addressed in a dedicated i18n cleanup sprint.
- `CommandPalette` footer uses a two-part pattern (`footerEscPrefix` + `footerEscClose`) around a `<kbd>` element. A template-string approach with a slot placeholder would be cleaner but requires a translation helper utility.

---

### What Should Be Done Next

- **i18n cleanup sprint**: Audit and migrate remaining hardcoded strings in the `Card`, `Board`, `Auth`, `User`, `Workspace`, `Search`, and `List` extensions. These predate the i18n effort (Sprints 90–94).
- **Translation helper**: Introduce a lightweight `t(key, vars)` helper to support interpolation (e.g., `{workspaceName}`) without adopting a full i18n library.
- **RTL / locale support**: Once all strings are in translation files, the next step is supporting alternate locales by swapping the `en.json` files at runtime.
