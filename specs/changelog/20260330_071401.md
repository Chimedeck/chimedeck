# Sprint 122 Iteration 2 — Multi-Instance Notification Delivery via Pub/Sub

## Update

- `server/extensions/realtime/api/index.ts`: Calls `subscribeUserChannel` on WebSocket open and `unsubscribeUserChannel` on close, wiring the pub/sub subscription lifecycle to the WebSocket connection lifecycle.

## New

No new modules added.

## Technical Debt

All `publishToUser` call sites in `.then()` chains now `return publishToUser(...)` so the async promise is properly chained and errors propagate to the `.catch()` handler instead of being silently dropped.

## What Should Be Done Next
