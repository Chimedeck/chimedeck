# Changelog

## Update

- Refactored the shared realtime socket lifecycle to use connection reference counting, so one hook cleanup no longer tears down the socket while other consumers are still mounted.
- Added board-room reference counting in the socket client and explicit unsubscribe messaging when the last board consumer leaves.
- Updated board realtime hook cleanup to release only its board room ownership via `disconnect({ boardId })`.
- Kept workspace sync behavior intact while preventing route navigation close/reopen churn from shared singleton ownership.

## New

- No new APIs, database migrations, or UI modules.

## Technical Debt

- The shared socket now supports multi-owner lifecycle, but token rotation across active owners still assumes a compatible auth state.

## What Should Be Done Next

- Add a focused test that mounts multiple realtime consumers and verifies the socket remains connected until the last owner unmounts.
- Add a navigation test for workspace list <-> board detail transitions to ensure no unnecessary reconnect churn in dev/proxy logs.
