## Update
- `server/extensions/board/api/delete.ts`: DELETE `/api/v1/boards/:id` now returns **409** `delete-requires-confirmation` with `{ listCount, cardCount }` when the board has nested lists or cards and `confirm:true` is not present in the request body. With `confirm:true` deletion proceeds normally.
- `server/extensions/list/api/delete.ts`: DELETE `/api/v1/lists/:id` now returns **409** `delete-requires-confirmation` with `{ cardCount }` when the list has cards and `confirm:true` is not present. With `confirm:true` deletion proceeds normally.
- `src/extensions/Board/api.ts`: `deleteBoard` now accepts an optional `confirm` parameter and passes it as `{ confirm: true }` in the request body when truthy.
- `src/extensions/List/api.ts`: `deleteList` now accepts an optional `confirm` parameter and passes it as `{ confirm: true }` in the request body when truthy.
- `src/extensions/Board/components/BoardDeleteDialog.tsx`: New modal dialog that shows board title, list count, card count, and a permanent-deletion warning before confirming.
- `src/extensions/List/components/ListDeleteDialog.tsx`: New modal dialog that shows list title, card count, and a permanent-deletion warning before confirming.
- `src/extensions/Board/containers/BoardPage/BoardPage.tsx`: Replaced `window.confirm()` for board/list deletion with the new dialog components. On 409 response, the appropriate dialog is shown with counts; on confirm it resends with `confirm:true`.
