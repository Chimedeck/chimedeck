# Changelog — 20260415_025738

## Update

- Wired `webhooksRouter` into `server/index.ts` after `apiTokenRouter`
- Added `WEBHOOKS_ENABLED` env var to `server/config/env.ts` (default `false` in local dev)

## New

- `server/extensions/webhooks/mods/registry.ts` — `getActiveWebhooksForEvent` query: returns all active webhooks for a workspace subscribed to a given event type using `event_types @> ?::jsonb`
- `server/extensions/webhooks/api/ssrfGuard.ts` — `isEndpointAllowed` helper: rejects non-HTTPS URLs and endpoints resolving to private/loopback/link-local IP ranges (RFC-1918, 169.254.x.x, ::1)
- `server/extensions/webhooks/api/list.ts` — `GET /api/v1/webhooks?workspaceId=` — lists webhooks without `signing_secret`
- `server/extensions/webhooks/api/create.ts` — `POST /api/v1/webhooks` — creates a webhook, generates 32-byte hex `signing_secret`, returns it once only
- `server/extensions/webhooks/api/update.ts` — `PATCH /api/v1/webhooks/:id` — updates label/endpointUrl/eventTypes/isActive; requires ownership or ADMIN+
- `server/extensions/webhooks/api/delete.ts` — `DELETE /api/v1/webhooks/:id` — deletes webhook; requires ownership or ADMIN+; deliveries cascade via DB FK
- `server/extensions/webhooks/api/eventTypes.ts` — `GET /api/v1/webhooks/event-types` — returns canonical event type list
- `server/extensions/webhooks/api/index.ts` — `webhooksRouter` mounting all above routes; gated by `WEBHOOKS_ENABLED` flag (returns 501 when off)
- `tests/integration/webhooks/api-crud.test.ts` — 33 tests covering SSRF guard, event-type constant shape, create/update body validation, and secret-not-returned invariant

## Technical Debt

- SSRF guard performs a DNS lookup on every create/update — consider caching or rate-limiting DNS lookups per request in high-traffic scenarios
- `event_types` is stored as `jsonb` and serialized with `JSON.stringify` on write; Knex may double-encode depending on the driver version — worth adding a smoke test against a real DB to confirm
