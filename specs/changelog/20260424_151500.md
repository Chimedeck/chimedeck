# Update
- Restored card member assignment and unassignment notifications when actions are performed from Card Modal flows.
- Notification recipient resolution for member events now guarantees inclusion of the assignment target user.

# New
- `mapActivityToNotification` now explicitly adds target users for `card_member_assigned` and `card_member_unassigned` to the recipient set (excluding actor self-notifications).
- `only_related_to_me` filtering now treats the direct member target as related for member assignment/unassignment notification types.

# Technical Debt
- Recipient resolution logic still mixes board-participant fanout and per-type recipient overrides in one function; extracting a per-notification-type recipient policy map would reduce branching complexity.
