## Update
- `server/extensions/card/api/create.ts`: After a card is successfully inserted and re-fetched from the DB, both `dispatchEvent` and the new `emitCardCreated` are fired concurrently via `Promise.all`. The `actorId` is resolved once and reused by both calls.
- `server/extensions/activity/config/visibleEventTypes.ts`: Added `card_created` to the allow-list so it surfaces in both the card and board activity feed APIs.

## New
- `server/extensions/activity/mods/createActivityEvent.ts`: Introduces `emitCardCreated` — a typed wrapper around `writeActivity` that enforces the `card_created` payload contract (`cardId`, `cardTitle`, `listId`, `listName`, `boardId`, `workspaceId`) and threads through `ipAddress` / `userAgent` for auditability.
- `specs/tests/card-activity-create-move-assign.md`: Five Playwright MCP markdown scenarios covering: activity row written on success (CACA-CREATE-01), UI rendering in card modal (CACA-CREATE-02), no record on failed creation (CACA-CREATE-03), payload immutability after title edit (CACA-CREATE-04), and auth guard on activity feed endpoint (CACA-CREATE-05).
