# Changelog — 20260313_160000

## Update
- `src/extensions/Board/api.ts`: Added `background?: string | null` field to `Board` interface; added `uploadBoardBackground()` and `deleteBoardBackground()` client API functions (using `fetch` with multipart/form-data and DELETE respectively).
- `src/extensions/Board/slices/boardSlice.ts`: Added `updateBoardBackground` reducer action to update `board.background` in Redux state.
- `src/extensions/Realtime/hooks/useBoardSync.ts`: Added handler for `board.background_changed` WS event — dispatches `updateBoardBackground` so all connected clients update in real time.
- `src/extensions/Board/containers/BoardPage/BoardPage.tsx`: Applied board background as CSS `background-image` on the outer container with a `bg-black/50` scrim overlay so list columns remain legible.
- `src/extensions/Board/containers/BoardSettings/BoardSettings.tsx`: Imported and rendered the new `BackgroundPicker` component as a new section in the settings panel.
- `src/extensions/Board/components/BoardCard.tsx`: Added background thumbnail (`h-20` image or gradient fallback) at the top of each board card in the workspace grid.
- `src/extensions/Search/api.ts`: Added `background?: string | null` field to `SearchResult` interface.
- `src/extensions/Search/components/SearchResultItem.tsx`: Board search results now show a 32×20 px background thumbnail (`w-8 h-5`) when `result.background` is set; falls back to the indigo icon.

## New
- `src/extensions/Board/containers/BoardSettings/BackgroundPicker.tsx`: New component for uploading/removing a board background image. Shows current background preview, "Upload image" label (hidden file input), and "Remove" button. Validates JPEG/PNG client-side by accept attribute; dispatches Redux action on success.
- `specs/tests/board-background-ui.md`: Playwright test spec covering: background section in settings (Test 1), valid JPEG upload (Test 2), background removal (Test 3), invalid MIME rejection (Test 4), workspace grid thumbnail (Test 5), search result thumbnail (Test 6), and real-time WS update (Test 7).

## Technical Debt
- `BackgroundPicker` does not enforce the 10 MB file-size limit client-side before upload — the server rejects oversized files with 413 but no UI warning is shown pre-flight. A future iteration could add a client-side size check.
- `BoardPage.tsx` passes the background image directly into `style={{ backgroundImage }}` without caching the URL; very large images may cause a flash on first render.
