## Update
- `db/migrations/0023_plugin_whitelisted_domains.ts`: Added `whitelisted_domains` (JSONB nullable) column to the `plugins` table via a new migration.
- `server/extensions/plugins/api/registry/create.ts`: Updated `POST /api/v1/plugins` to accept, validate, and persist `whitelistedDomains`; enforces max 20 entries and HTTPS-only origins.
- `server/extensions/plugins/api/registry/update.ts`: Updated `PATCH /api/v1/plugins/:id` to accept, validate, and replace `whitelistedDomains`; same validation rules as registration.
- `server/extensions/plugins/api/registry/list.ts`: Updated `GET /api/v1/plugins` to always include `whitelisted_domains` in every plugin in the response.
- `server/extensions/plugins/api/index.ts`: Extended the board-plugins regex to also match `/allowed-domains` sub-resource and mounted the new `PATCH .../allowed-domains` handler.

## New
- `server/extensions/plugins/common/isValidHttpsOrigin.ts`: Utility that validates a string is a well-formed HTTPS origin (scheme + host + optional port, no path/query/hash).
- `server/extensions/plugins/api/board-plugins/allowed-domains.ts`: `PATCH /api/v1/boards/:boardId/plugins/:pluginId/allowed-domains` — board-admin-only endpoint that sets `allowedDomains` in `board_plugins.config`; validates every requested domain is declared in the plugin's `whitelistedDomains`.
- `server/extensions/plugins/api/__tests__/whitelisted-domains.test.ts`: Unit tests for `isValidHttpsOrigin` covering valid HTTPS origins and all rejection cases.
