# Changelog — 20260312_035741

## Sprint 68: Run log API endpoints + quota API

### Update

- `server/extensions/automation/config/index.ts`: Added `monthlyQuota` field (reads `AUTOMATION_MONTHLY_QUOTA` env var, defaults to `1000` on missing/invalid value).
- `server/extensions/automation/api/index.ts`: Wired three new GET routes into the automation router.

### New

- `server/extensions/automation/api/runs.ts`: `GET /api/v1/boards/:boardId/automations/:automationId/runs` — paginated run log for a single automation. Supports `page`, `perPage` (max 50, default 20), and `status` (SUCCESS|PARTIAL|FAILED) query params. Returns `data[]` + `metadata.{totalPage, perPage}`.
- `server/extensions/automation/api/boardRuns.ts`: `GET /api/v1/boards/:boardId/automation-runs` — board-wide log (last 200 runs, all automations), paginated at 50 per page.
- `server/extensions/automation/api/quota.ts`: `GET /api/v1/boards/:boardId/automation-quota` — returns `{usedRuns, maxRuns, resetAt, percentUsed}` for the current calendar month. Publishes a `quota_warning` WS event (via `broadcast`) when `percentUsed >= 80`.
- `tests/integration/automation/runs.test.ts`: Integration tests covering auth guards, query-param handling, response shape, and perPage cap enforcement.
- `tests/integration/automation/quota.test.ts`: Integration tests covering auth guards, response shape, UTC month boundary, config defaults, and quota percentage calculation.

### Technical Debt

- `runCardButton.ts` and `runBoardButton.ts` use a `board_members` table that does not exist in the schema (correct table is `memberships`). This pre-existing bug was not introduced by this iteration but should be fixed in a follow-up.
