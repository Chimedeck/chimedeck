# Changelog — 2026-03-31 15:00:00

## Update
- `terraform/environments/staging/backend.tf` — terraform block with S3 backend config (placeholder values for bucket/key/region/table); operators fill in before `terraform init`
- `terraform/environments/stable/backend.tf` — same structure with `stable` state key
- `terraform/environments/production/backend.tf` — same structure with `production` state key

## New
- `terraform/environments/staging/` — staging environment directory:
  - `backend.tf` — `terraform {}` block with S3 backend and required providers
  - `main.tf` — provider + all module calls; compute modules gated by `deployment_mode`
  - `variables.tf` — all variables (core, VPC, EC2, ECR, S3, Secrets, ACM, RDS, Redis, ASG blue/green)
  - `terraform.tfvars.example` — fully commented example with staging-appropriate defaults (`deployment_mode = "single"`, small instances, no deletion protection)
  - `outputs.tf` — ECR URL, RDS endpoint, Redis endpoint, public IP (single), ALB DNS name (fleet/ASG), blue/green TG ARNs and ASG names

- `terraform/environments/stable/` — stable (pre-production) environment directory with the same file layout; uses `deployment_mode = "fleet"`, `create_vpc = true`, deletion protection enabled

- `terraform/environments/production/` — production environment directory; uses `deployment_mode = "asg-bluegreen"`, `create_vpc = true`, multi-AZ RDS, cluster Redis, highest instance types, full deletion protection and 14-day backup retention

## Technical Debt
- `backend.tf` in each environment contains placeholder bucket/table names. Operators must fill these in from the bootstrap output before running `terraform init`. Consider scripting this step in `scripts/deploy.sh` (Iteration 11).
- `terraform.tfvars` files are not committed (by design, to avoid secret leakage) — this means a fresh checkout requires manual setup. A future iteration could introduce a `scripts/setup-env.sh` that generates `terraform.tfvars` by reading secrets from a secure store.
