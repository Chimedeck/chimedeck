# Changelog 20260304_110000

## Update
- `src/extensions/Card/components/CardItem.tsx` — wired `monetizationType` and `listName` props into actual rendering logic; now conditionally renders `CardPaymentButtons` when `monetizationType === 'pay-to-paid'`, card has an `amount`, and `shouldShowPaymentButtons(listName)` returns true.

## New
- `src/config/index.ts` — central client config module exposing `STRIPE_PUBLIC_KEY` from `VITE_STRIPE_PUBLIC_KEY` env var.
- `src/extensions/Board/config/stripePaymentButtonsConfig.ts` — `PAYMENT_BUTTONS` array with "Pay 10%" and "Pay in Full" entries; each button defines a `getAmount` function deriving the payment amount from the card's amount in cents.
- `src/extensions/Payment/config/stripeClient.ts` — singleton `loadStripe` promise; only initialised when `VITE_STRIPE_PUBLIC_KEY` is set.
- `src/extensions/Payment/components/StripePaymentModal.tsx` — modal component that calls `POST /api/v1/payments/stripe/create-payment-intent`, then renders Stripe `EmbeddedCheckout` inside an overlay.
- `src/extensions/Payment/components/CardPaymentButtons.tsx` — renders payment action buttons from `PAYMENT_BUTTONS` config; manages modal open/close state; stops click propagation so card detail drawer is not triggered.
- Installed npm packages: `@stripe/stripe-js@8.9.0`, `@stripe/react-stripe-js@5.6.1`.

## Technical Debt
- `StripePaymentModal` creates a new PaymentIntent on every open; in production, consider reusing or expiring intents.
- Board view components still need to thread `monetizationType` and `listName` into `CardItem` for the payment buttons to appear (confirmed they already receive these props from iteration 2).
