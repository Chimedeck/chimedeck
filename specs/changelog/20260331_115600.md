# Changelog — 20260331_115600

## Update
- `terraform/modules/ecr/` — new ECR repository module
- `terraform/modules/s3/` — new S3 bucket module

## New
- **`terraform/modules/ecr/`** — parameterized ECR repository with:
  - `scan_on_push = true` (vulnerability detection on every push)
  - Lifecycle policy: expire untagged images after 1 day; keep last N tagged images (default 10)
  - Configurable `image_tag_mutability` (default `MUTABLE`)
  - Outputs: `repository_url`, `repository_arn`, `repository_name`

- **`terraform/modules/s3/`** — secure general-purpose S3 bucket with:
  - Block all public access (all four flags enabled by default)
  - SSE-S3 (AES256) server-side encryption on all objects
  - Versioning controlled via `versioning_enabled` variable (default `true`)
  - Outputs: `bucket_name`, `bucket_arn`, `bucket_region`

## Technical Debt
- ECR lifecycle policy uses `tagPrefixList = [""]` to match all tagged images; if repos adopt strict prefix conventions this could be tightened.
- S3 uses SSE-S3 (AWS-managed keys). Buckets holding sensitive data should be migrated to SSE-KMS.
