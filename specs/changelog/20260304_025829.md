## Update

### Sprint 29 — Activity Config & Server Filter

- `server/extensions/activity/config/visibleEventTypes.ts` (new): Created `VISIBLE_EVENT_TYPES` allowlist exporting all action strings that may appear in activity feeds. Currently includes comment actions (`comment_added`, `comment_edited`, `comment_deleted`) and system events for sprint 29+ (`card.member.added`, `card.member.removed`, `card.due_date.set`, `card.due_date.cleared`, `card.money.updated`). Adding/removing entries from this array controls visibility across all feed endpoints without touching business logic.
- `server/extensions/activity/api/cardActivity.ts`: Added `.whereIn('action', VISIBLE_EVENT_TYPES)` filter to `GET /api/v1/cards/:id/activity` so only allowed event types are returned.
- `server/extensions/activity/api/boardActivity.ts`: Added `.whereIn('action', VISIBLE_EVENT_TYPES)` filter to `GET /api/v1/boards/:id/activity` (paginated).
- `server/extensions/card/api/get.ts`: Sideloads filtered activities when `?include=activities` is passed. Activities are ordered ascending (oldest first) for the card modal feed. Without the param the `activities` array defaults to `[]` (no extra query).
- `server/extensions/board/api/get.ts`: Sideloads filtered activities for all cards on the board when `?include=activities` is passed. Returns activities keyed under `includes.activities`. Without the param it defaults to `[]`.
