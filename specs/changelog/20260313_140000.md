# Changelog — 20260313_140000

## Auth Client Public Endpoint Guard

### Update

- `src/common/api/client.ts`: Access tokens are no longer auto-attached to requests targeting public API endpoints such as `POST /api/v1/auth/token`, `POST /api/v1/auth/register`, password-reset flows, email-verification flows, and invite inspection.
- `src/common/api/interceptors.ts`: The global 401 interceptor now skips silent refresh/logout redirects for public endpoints, so public auth flows no longer get treated like expired protected sessions.

### New

- `src/common/api/requestPolicy.ts`: Added a shared client-side route policy that classifies public API endpoints and centralises decisions for token injection and auth recovery.
- `src/common/api/requestPolicy.test.ts`: Added regression coverage for login, verify-email, confirm-email-change, forgot/reset-password, refresh, and invite inspection route classification.

### Technical Debt

- The server route `POST /api/v1/auth/token` itself does not require `Authorization`; the observed behaviour came from client-side request/recovery policy. If production still shows auth gating after this change, the next place to inspect is any external reverse proxy/CDN layer in front of Bun.
