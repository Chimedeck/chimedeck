# Changelog — 2026-03-31 14:00:00

## Update
- `terraform/modules/ec2-fleet-fixed/` — new module implementing a fixed-count EC2 fleet behind an ALB.

## New
- `terraform/modules/ec2-fleet-fixed/main.tf` — ALB (public subnets), IP-based target group, HTTP→HTTPS redirect listener, HTTPS listener with ACM cert, primary-region EC2 instances in private subnets, primary and additional IP target group attachments.
- `terraform/modules/ec2-fleet-fixed/variables.tf` — all input variables per sprint-126 spec; includes `additional_target_ips` for cross-region IP registration.
- `terraform/modules/ec2-fleet-fixed/outputs.tf` — `alb_dns_name`, `alb_arn`, `alb_zone_id`, `target_group_arn`, `instance_ids`.
- `terraform/modules/ec2-fleet-fixed/README.md` — usage examples (same-region and cross-region), variable table, output table, caveats.

## Technical Debt
- Terraform does not support dynamic provider aliases, so cross-region instances cannot be created inside this module. The spec's `additional_regions` map variable is replaced by `additional_target_ips` (a list of private IPs). The environment directory is responsible for creating cross-region EC2 instances with explicit provider aliases and for calling `modules/vpc-peering` + `modules/nat-gateway`.
