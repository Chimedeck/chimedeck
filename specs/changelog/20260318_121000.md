# Changelog — 2026-03-18 Sprint 88 Iteration 7: Activity feed query integration

## Update

- `server/extensions/activity/api/cardActivity.ts`: Added stable secondary `ORDER BY id DESC` to the card activity feed query. Previously, events sharing the same `created_at` timestamp could return in non-deterministic order across repeated calls. Now matches the same two-column sort strategy already used by `boardActivity.ts`.

## New

- `specs/tests/card-activity-create-move-assign.md`: Added **Section 7 — Feed ordering stability** with five Playwright MCP scenarios:
  - 7.1 Stable order across repeated calls for co-timestamp events
  - 7.2 Merged client feed: comments and activity events interleaved correctly
  - 7.3 Legacy activity event types remain visible alongside new Sprint 88 types
  - 7.4 Empty/single-event feed returns a valid response shape
  - 7.5 Determinism guarantee when two rows share an exact `created_at`

## Technical Debt

- No pagination has been added to the card activity endpoint. For cards with high activity volume this will become a performance concern. Consider adding cursor-based pagination (matching the board activity endpoint) in a future iteration.
