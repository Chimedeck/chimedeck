# Update
- Hardened realtime dispatch so websocket updates continue even when one client event handler throws at runtime.
- Fixed potential board sync runtime crash on `comment_reply_added` and `card_activity_created` when payloads are partial/malformed.

# New
- Shared websocket client now isolates subscriber callback failures per handler during message fanout.
- Board sync now validates `reply`, `activity`, and `activity.entity_id` before accessing member-sync logic.

# Technical Debt
- `useBoardSync` still exceeds the project's cognitive-complexity threshold; this change intentionally avoided broad refactor while addressing runtime stability.

# What Should Be Done Next
- Add a small test around realtime fanout asserting that one throwing subscriber does not block other subscribers.
- Refactor `useBoardSync.handleEvent` into per-event handler helpers to satisfy complexity lint rules and improve maintainability.
