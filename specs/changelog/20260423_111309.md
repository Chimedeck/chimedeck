## Technical Debt
- Drag resolution remains centralized in `BoardCanvas` and still has high cognitive complexity; future changes in this area carry elevated regression risk.
- Long-distance drag coverage currently validates one path (left-to-right across many columns); reverse-direction and dense-card-list scenarios should still be expanded.
- Existing static-analysis complexity warnings in `BoardCanvas` predate this fix and still need a dedicated refactor pass.

## What Should Be Done Next

- Add targeted unit tests for the pure list-hit resolution helper logic to reduce dependence on full UI drag E2E tests for edge-case confidence.
