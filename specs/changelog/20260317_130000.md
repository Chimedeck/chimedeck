# Changelog — 20260317_130000

## Update

- `src/extensions/Comment/components/CommentEditor.tsx`: Integrated offline draft behaviour. On every keystroke, content is debounce-persisted to IndexedDB (800 ms) and background-synced to the server when online (3 s). On card open, the draft is restored from local storage first and reconciled with the server snapshot. Pressing **Comment** while offline queues a POST mutation with a stable `idempotency_key` and sets intent to `submit_pending`. Footer status UI shows all draft states (`Draft saved locally`, `Synced draft`, `Will post when back online`, `Sync failed` with Retry/Discard). Draft recovery banner appears when a restored draft is available.

## New

- `src/extensions/OfflineDrafts/hooks/useOfflineCommentDraft.ts`: New hook mirroring `useOfflineDescriptionDraft` but for comment drafts. Key differences: `draftType: 'comment'`, `handleSubmitIntent` queues a POST (not PATCH) with a UUID `idempotency_key` for server-side deduplication on replay, `boardId` required for the message queue.
- `specs/tests/offline-comment-draft-recovery.md`: Playwright MCP scenario file with 19 scenarios covering local persistence while offline, hard-refresh restore, server sync, offline submit queuing, idempotent replay, draft privacy, cross-device continuity, reconcile winner logic, draft cleanup, and sync failure/retry.

## Technical Debt

- The queue replay listener (iteration 12) must call `clearDraft()` after a successful comment POST replay to remove the `submit_pending` draft from IndexedDB and the server.
- `boardId` is typed as `string | undefined` in `CommentEditor` — callers that omit it will skip offline queueing silently. Should be made required once all call-sites are confirmed.
