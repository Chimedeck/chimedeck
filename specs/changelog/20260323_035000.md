# Changelog — Sprint 98: card_commented Notification Dispatch

## Update
- `server/extensions/notifications/mods/boardActivityDispatch.ts`: Removed `comment_added` from the events-pipeline `SUPPORTED_EVENTS`/`SupportedEventType` to prevent duplicate notifications. Extended `DirectCardNotificationPayload` union with `card_commented` (fields: `cardTitle`, `commentPreview`, `commentId`). Updated `dispatchDirectCardNotification` to store `commentId` in `source_id` for `card_commented`, and to dispatch email via `dispatchNotificationEmail` with full template data (actorName, cardTitle, boardName, commentPreview, cardUrl).
- `server/extensions/comment/api/create.ts`: Added import of `dispatchDirectCardNotification`. After the comment is persisted and the WS broadcast fires, builds `commentPreview` (strips HTML tags from `trimmedContent`, truncates to 120 chars with ellipsis) and calls `dispatchDirectCardNotification` fire-and-forget for `card_commented`.
- `server/mods/events/dispatch.ts`: Updated inline comment to reflect that `comment_added` is no longer handled by the events pipeline for notifications.

## New
- `card_commented` notifications are now dispatched directly from the comment creation endpoint with the exact `commentId` stored in `source_id`, enabling precise navigation to the originating comment.
