# Changelog — 2026-03-11T11:09:49Z

## Update
- `server/extensions/attachment/mods/virusScan/update.ts` — now fires `generateThumbnail` as a fire-and-forget side-effect when an attachment transitions to `READY`
- `server/extensions/attachment/api/index.ts` — added `GET /api/v1/cards/:id/attachments` route wired to the new `handleListAttachments` handler

## New
- `server/extensions/attachment/common/presign.ts` — shared pre-signed GET URL helper with configurable TTL (default 1 h)
- `server/extensions/attachment/workers/thumbnail.ts` — sharp-based image thumbnail worker; generates 400×300 max WebP thumbnails stored at `thumbnails/<card_id>/<attachment_id>.webp` in S3; updates `thumbnail_key`, `width`, and `height` columns
- `server/extensions/attachment/workers/orphanCleanup.ts` — periodic worker (30 min interval) that deletes `PENDING` FILE attachments older than 1 hour from both S3 and the DB
- `server/extensions/attachment/api/list.ts` — `GET /api/v1/cards/:id/attachments` handler; returns attachments with pre-signed `url` (TTL 1 h) and `thumbnailUrl` (only when `thumbnail_key` is set)
- `tests/integration/attachments/thumbnails.test.ts` — 7 tests covering `generateThumbnail` guard conditions and list-endpoint auth/response shape
- `tests/integration/attachments/orphanCleanup.test.ts` — 8 tests covering TTL boundaries, module exports, and worker startup

## Technical Debt
- `mods/orphanCleanup.ts` (original, 15-min interval) still exists alongside the new `workers/orphanCleanup.ts` (30-min interval); the old file should be removed or consolidated once callers are confirmed.
- Thumbnail worker uses `mime_type` (client-supplied) to detect images; future improvement: confirm via S3 HEAD `ContentType` and persist to the `content_type` column from migration 0033.
