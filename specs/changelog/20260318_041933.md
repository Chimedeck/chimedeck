## Update
- Replaced the stub `writeEvent` call in `server/extensions/board/api/delete.ts` with a call to the new `publishBoardDeleted` mod, ensuring proper event persistence and workspace-wide real-time notification on board deletion.

## New
- Added `server/extensions/events/mods/publishBoardDeleted.ts`: writes a `board_deleted` event (with `boardId` and `workspaceId` in the payload, `board_id = null` since the board no longer exists), publishes to the `workspace:{workspaceId}` pubsub channel, and fans out directly to each workspace member's personal WS socket via `publishToUser` for immediate real-time propagation.
- Created `specs/tests/board-delete-auto-refresh.md`: Playwright MCP scenarios covering API success/failure paths (BDAR-API-01–07), event payload correctness and idempotency (BDAR-EVENT-01–03), and workspace WS fan-out (BDAR-WS-01). UI and realtime cross-user scenarios are stubbed as deferred placeholders for Iterations 2–4.

## Technical Debt
- Guest role handling in delete.ts is checked before ADMIN role; this ordering is correct but could be consolidated into a single middleware pipeline in a future refactor.
- The `workspace:{workspaceId}` pubsub channel has no corresponding WebSocket subscriber in the current server — clients only subscribe to `board:{boardId}`. Real-time delivery relies on the `publishToUser` fan-out until a workspace-scoped WS subscription is wired up in Iteration 4.
