# Sprint 89 — Guest Role Split: Iteration 4
**Client Guest Management Panel + UI Permission Guards**

## Update

- `server/extensions/board/api/get.ts` — Board GET response now includes `callerGuestType` (the caller's guest sub-type from `boardVisibility` middleware, or `null` for non-guests). Enables the client to derive write access without a second round-trip.
- `src/extensions/Board/api.ts` — `Board` interface extended with `callerGuestType?: 'VIEWER' | 'MEMBER' | null`.
- `src/extensions/Board/containers/BoardPage/BoardPage.tsx` — Derives `isViewerGuest` from `isGuest && !canBoardGuestWrite(board.callerGuestType)`. Passes it to `BoardCanvas`.
- `src/extensions/Board/components/BoardCanvas.tsx` — Accepts `isViewerGuest` prop and forwards it to each `SortableListColumn`.
- `src/extensions/List/containers/BoardPage/ListColumn.tsx` — Accepts `isViewerGuest` prop; hides the "+ Add a card" button and add-card form when `true`.
- `src/extensions/Card/containers/CardModal/index.tsx` — Derives `isViewerGuest` from board state + workspace role; passes it to `CardModal`.
- `src/extensions/Card/components/CardModal.tsx` — Accepts `isViewerGuest` prop; passes `canWrite={!isViewerGuest}` to `AttachmentPanel` and `canAddComment={!isViewerGuest}` to `ActivityFeed`.
- `src/extensions/Card/containers/CardModal/ActivityFeed.tsx` — Accepts `canAddComment` prop (default `true`); hides `CommentEditor` when `false`.
- `src/extensions/Attachments/components/AttachmentPanel.tsx` — Accepts `canWrite` prop (default `true`); hides "Attach file" button, "Attach a link" form, file input, and paste listener when `false`. Existing attachments remain visible.
- `src/extensions/Board/components/BoardMembersPanel/GuestsTab.tsx` — Guest list rows now show a VIEWER/MEMBER badge. ADMIN users see interactive toggle buttons per row (uses `useUpdateBoardGuestMutation`); non-admins see a read-only badge. Per-row error message on type-change failure.

## New

- `specs/tests/sprint-89-guest-role-split.md` — Appended 13 new scenarios (34–46) covering: badge display, inline type change by ADMIN, error on failure, non-admin read-only badge, VIEWER write-guard on Add Card / Comment / Attachment, MEMBER full access, and `callerGuestType` in board GET response.

## Technical Debt

- The `AttachmentDropZone` receives a no-op `onFiles` callback when `canWrite=false`; ideally it should disable drag events entirely to avoid confusion, but this is visually sufficient for now.
- `CardModalBottomBar` sidebar actions (assign member, labels, etc.) are not yet permission-guarded for VIEWER guests — deferred to a future polish iteration.

## What Should Be Done Next

- Hide sidebar quick-action buttons in CardModal (assign member, labels, due date) for VIEWER guests.
- Consider bulk guest type management (select all → change type).
- Add optimistic update for inline guest type change (currently waits for server confirmation).
