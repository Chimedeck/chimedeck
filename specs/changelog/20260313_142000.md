# Changelog — 2026-03-13T14:20:00Z

## Update

- `server/extensions/notifications/mods/createNotifications.ts` — Refactored `createNotificationsForMentions` to iterate per-recipient and consult `preferenceGuard` before inserting a notification row or publishing a WebSocket event. Notification delivery is skipped when `in_app_enabled` is `false` for the recipient's `mention` preference. Fails open (delivers notification) when `preferenceGuard` throws. When `NOTIFICATION_PREFERENCES_ENABLED=false` the guard is bypassed entirely, preserving Sprint 26 backward-compatible behaviour.

## New

- `specs/tests/notification-preferences-mention-guard.md` — Playwright MCP test spec covering five scenarios: guard off (disabled pref blocks notification), guard on (enabled pref delivers notification), feature flag disabled (always deliver), self-mention (no notification), and guard DB error (fail-open).

## Technical Debt

- The bulk `trx('notifications').insert(rows)` was replaced with individual per-recipient inserts to allow per-user preference checks. A future optimisation could batch the DB write and only skip the WS publish per preference, but individual inserts are clearer and correctness is prioritised.
