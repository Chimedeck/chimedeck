# Changelog — 2026-03-18T02:43:25Z

## Sprint 84 — Iteration 3: Result Interactions and Routing State

### Update

- **BoardSearchBar**: Added `initialQuery` prop (restores search from URL on mount), `onQueryChange` callback (fires on every input change for URL sync), and board-switch reset (clears input/results when `boardId` changes). Result titles now render with matched-text highlight via a `HighlightedTitle` sub-component (substring, case-insensitive, uses `<mark>`).
- **BoardHeader**: Threaded new `initialSearchQuery`, `onSearchQueryChange`, and `onSearchResultSelect` props through to `BoardSearchBar`.
- **BoardPage**: Reads `boardSearch` URL param on mount (`searchParams.get`), passes it as `initialSearchQuery`. Added `handleSearchResultSelect` (card → sets `?card=<id>` URL param; list → `scrollIntoView` on `board-list-<id>` element). Added `handleSearchQueryChange` (sets/deletes `?boardSearch=` with `{ replace: true }` to avoid polluting history). Passes both handlers and `initialSearchQuery` to `BoardHeader`.
- **ListColumn**: Added `id="board-list-${list.id}"` to the outer column div so list results from board search can be scrolled into view.
