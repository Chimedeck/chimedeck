# Changelog — 20260324_165909

## Update

## New

- **`src/extensions/HealthCheck/modals/AddServiceModal.tsx`** — Modal for adding a health check service. Supports preset mode (dropdown from Redux presets) and custom mode (name + URL fields). Resets all local state on close; clears inputs/error on mode switch. Inline error display via `role="alert"`. Dispatches `addHealthCheckThunk` and calls `onClose` on success. Lazy-loads presets via `fetchPresetsThunk` when first opened.

- **`src/extensions/HealthCheck/hooks/useHealthCheckAutoRefresh.ts`** — 60-second auto-refresh hook. Uses `setInterval` with 1-second ticks to decrement a countdown. Pauses when the browser tab is hidden (Page Visibility API). Exposes `secondsUntilRefresh` for the countdown display and `triggerRefresh` to reset + fire immediately. `onRefresh` callback is kept in a ref so the interval never needs to re-register.

- **`src/extensions/HealthCheck/hooks/useHealthCheckProbe.ts`** — Per-row probe hook. Subscribes once to the full `selectProbingIds` array from Redux. Returns `isProbing(id)` (pure predicate over the array) and `probe(id)` (dispatches `probeSingleThunk`). Avoids the Rules-of-Hooks violation by using a selector at the hook level rather than inside a callback.

## Technical Debt

- `useHealthCheckProbe` returns a plain function rather than a derived boolean, meaning callers incur one subscription for the entire probingIds list. This is fine for the expected scale (< 20 entries per board) but could be optimised with per-id subscriptions if the list grows large.
