# Changelog — 20260324_064626

## Update

- **PluginSearchBar** extended with an optional `onStatusChange` prop and a Status dropdown (All / Active / Inactive). The dropdown only renders when `onStatusChange` is provided, so existing uses in `PluginDashboardPage` are unaffected.
- **PluginRegistryPage** overhauled: now imports and renders `PluginRegistryTable` and `PluginSearchBar`, wires up all three filter dispatches (search/q, category, status), manages local state for the `EditPluginModal`, and tracks per-row deactivating/reactivating IDs to show inline loading.
- Translations (`en.json`) extended with table column headers, row action labels (Edit, Deactivate, Reactivate, inline confirm), and search bar status labels.

## New

- **`PluginRegistryTable.tsx`** — table wrapper with five columns (icon, name/author, description, categories, actions); renders one `PluginRegistryRow` per plugin.
- **`PluginRegistryRow.tsx`** — single row component with:
  - Plugin icon with React-state fallback (`PluginIcon` sub-component uses `useState` to switch to a `PuzzlePieceIcon` if the `img` load fails).
  - Name, author email, "Inactive" badge (when `isActive === false`).
  - Category chips (blue).
  - **Edit** button that calls the parent's `onEdit` callback.
  - **Deactivate** button with two-step inline confirm ("Really deactivate?" → Yes / No).
  - **Reactivate** button (shown instead of Deactivate when `plugin.isActive === false`).
  - Per-row `isDeactivating` / `isReactivating` props for loading feedback.
- **`specs/tests/plugin-registry-admin.md`** — Playwright MCP scenario covering: table renders, default active filter, search debounce, category filter, status filter, inline deactivate confirm (cancel + confirm), deactivated plugin appears in Inactive view, reactivate flow, Edit modal opens, non-admin redirect.

## Technical Debt

- Deactivate in `PluginRegistryPage` uses an optimistic remove (local) followed by an async `deletePluginThunk`. On thunk failure the row will have disappeared silently — a rollback/toast on failure is deferred.
