# Changelog — 2026-04-16T02:40:40Z

## Update
- `src/extensions/Card/components/CardItem.tsx`: Replaced fixed `coverHeightClass` (h-20/h-28) with adaptive `coverClass`. Image covers now use `aspect-video` (16:9) or `aspect-square` (1:1) based on `cover_aspect_ratio`; defaults to `aspect-square` when null. Color-only covers retain the fixed strip height. Added `overflow-hidden` to the cover container. Changed `<img>` from `object-cover` to `object-contain`.

## New
- `src/extensions/Card/api.ts`: Added `cover_aspect_ratio?: '16:9' | '1:1' | null` and `cover_is_gif?: boolean` fields to the `Card` interface, reflecting the new server-side fields from sprint-137 iteration 3.

## Technical Debt
- `cover_size` (SMALL/FULL) is now irrelevant for image covers since aspect ratio is auto-derived. The toggle still applies to color covers. A future cleanup task should hide the size toggle when an image cover is active.
- Existing cards with image covers will have `cover_aspect_ratio: null` until thumbnail worker re-processes those attachments; client falls back to `aspect-square` gracefully.
