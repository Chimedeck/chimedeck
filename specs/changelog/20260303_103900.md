# Changelog — 20260303_103900

## Update

### Board Settings Click-Outside Fixed
- **Problem**: The `BoardHeader` settings dropdown used a `fixed inset-0 z-10` backdrop div to catch outside clicks. Due to CSS stacking context rules (the `sticky z-10` header creates its own stacking context), the backdrop could interfere with other dropdown menus on the page and both menus could end up open simultaneously.
- **Fix**: Replaced the backdrop approach with a `useRef` + `useEffect` document `mousedown` listener on the menu container. Click events are only caught when `menuOpen` is true, and the listener is properly cleaned up on unmount/close. No z-index conflicts, no interaction with other dropdowns.
- **Files**: `src/extensions/Board/components/BoardHeader.tsx`

## New

### Comments Integration in Card Modal
- **Problem**: The `CommentThread`, `CommentItem`, `CommentEditor`, and `CommentDeletedItem` components under `src/extensions/Comment/` existed but were not wired into the card detail modal at all. Clicking a card showed no comments section.
- **Server**: Added `GET /api/v1/cards/:id/comments` endpoint (`server/extensions/comment/api/list.ts`). Returns all comments for a card ordered by `created_at` asc. Requires MEMBER-level workspace membership. Updated `commentRouter` in `server/extensions/comment/api/index.ts` to route the GET.
- **Client — API**: Added `CommentData` type plus four functions to `src/extensions/Card/api/cardDetail.ts`: `getCardComments`, `postCardComment`, `patchComment`, `deleteComment`.
- **Client — State**: Added `comments: CommentData[]` field to `CardDetailState` in `cardDetailSlice.ts`. Added `setComments`, `addComment`, `updateComment`, `removeComment` reducers. `closeModal` now clears comments. Exported `selectCardDetailComments` selector.
- **Client — Container**: `CardModalContainer` now imports `selectCardDetailComments`, `selectCurrentUser`, and the four comment API functions. Comments are fetched immediately after the card detail fetch on modal open. Three handlers (`handleAddComment`, `handleEditComment`, `handleDeleteComment`) manage comment mutations via Redux dispatch.
- **Client — UI**: `CardModal` now imports `CommentThread` and accepts `comments`, `currentUserId`, `onAddComment`, `onEditComment`, `onDeleteComment` props. `CommentThread` is rendered below the `CardChecklist` in the main column.
- **Files**: `server/extensions/comment/api/list.ts` (new), `server/extensions/comment/api/index.ts`, `src/extensions/Card/api/cardDetail.ts`, `src/extensions/Card/slices/cardDetailSlice.ts`, `src/extensions/Card/containers/CardModal/index.tsx`, `src/extensions/Card/components/CardModal.tsx`

## Technical Debt

- `CommentThread` uses a light-gray/white colour palette designed for a light UI, while the card modal uses a dark (`slate-900`) theme. Comments will appear with a white card background inside the dark modal — a visual inconsistency that needs a themed dark-mode variant.
- Comment edit/delete success currently dispatches optimistic state updates without a server re-fetch, so PATCH/DELETE errors silently leave the client out of sync. Should add error rollback.
- The `selectCardDetailMeta` selector returns a new object reference on every render (two Redux warnings visible in the console). Should use `createSelector` from `reselect`.

## What Should Be Done Next

1. **Dark-theme comments**: Update `CommentItem` and `CommentThread` styles to match the `slate-800/900` dark palette used throughout the modal.
2. **Comment selector memoisation**: Wrap `selectCardDetailMeta` with `createSelector` to eliminate the repeated Redux equality warnings.
3. **Comment error handling**: Add rollback / re-fetch on failed PATCH/DELETE comment calls.
4. **Real-time comment push**: When a WS `comment_added` event arrives for the currently open card, dispatch `cardDetailSliceActions.addComment` so other users see new comments live.
