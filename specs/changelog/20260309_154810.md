## New
- `POST /api/v1/auth/forgot-password`: Always returns 200, generates reset token, sends password reset email, rate-limited.
- `POST /api/v1/auth/reset-password`: Validates token, enforces password strength, resets password, invalidates sessions.
- Email template for password reset with reset link and expiry.
- ForgotPasswordPage and ResetPasswordPage (UI + ducks) for requesting and completing password reset.
- New routes: `/forgot-password`, `/reset-password`.
- Four test specs covering E2E, expired/invalid tokens, unknown emails, and weak passwords.
