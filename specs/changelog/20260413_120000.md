## Update
- Updated attachment deletion flow to delete both `s3_key` (original file) and `thumbnail_key` (generated thumbnail) from S3 for FILE attachments.
- Updated orphan cleanup worker to delete both object keys before removing orphan DB rows.
- Updated legacy orphan cleanup module to match worker behavior and avoid thumbnail leftovers.
- Replaced attachment `view` and `thumbnail` 302 redirect behavior with true server-side streaming proxy responses so clients no longer follow direct S3 links.

## New
- No new endpoints or schema changes.

## Technical Debt
- S3 deletion remains best-effort (errors are swallowed), so hard-delete guarantees still depend on S3 availability and observability.
