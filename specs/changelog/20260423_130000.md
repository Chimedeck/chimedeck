## Update
- Reduced drag start/end render fan-out by scoping `activeDragCardId` propagation to only the list currently showing the drag placeholder.
- Reduced non-source column invalidations by making `SortableListColumn` memo comparison ignore `activeDragCardId` changes unless placeholder mode is active for that column.
- Disabled card layout animation in `CardItem` sortable hooks because placeholder mode already communicates insertion location and avoids timer churn.
- Moved list hydration subscriptions from BoardCanvas-wide state reads to per-list subscriptions in `ListColumn`.
- Paused progressive list hydration dispatch while card drag is active to avoid competing main-thread work during interaction.

## New
- Added placeholder-mode-aware memo logic in `ListColumn` to keep non-participating columns stable during card drags.
- Added `isDragActive` gating to `ProgressiveHydrationDispatcher` in BoardCanvas.

## Technical Debt
- `BoardCanvas` drag handlers remain complex and should be split into smaller pure utilities to make further micro-optimizations safer.
- Final target verification still depends on production-build profiling; dev profiling includes JSX runtime validation overhead.

## What Should Be Done Next
- Re-profile drag start on the same 300+ card board and compare INP p75/p95 against the prior baseline.
- If INP is still above target, evaluate rendering an ultra-light card shell during active drag for non-source/target lists without introducing board-wide prop flips.
- Add a focused regression test that asserts non-placeholder columns do not rerender when active drag card changes.
