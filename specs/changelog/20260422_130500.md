## Update
- Optimized board load SQL in [server/extensions/board/api/get.ts](server/extensions/board/api/get.ts):
  - switched card count fields (`comment_count`, `attachment_count`, `linked_card_count`, `checklist_total`, `checklist_done`) from per-card scalar subqueries to pre-aggregated joins.
  - replaced per-list initial card id fan-out queries with one ranked top-N-per-list query using `ROW_NUMBER()`.
  - normalized board lookups to use resolved board id (`board.id`) for list/star/background reads.
  - stopped returning `includes.activities` unless activities were explicitly requested.
- Improved client board render path in [src/extensions/Board/containers/BoardPage/BoardPage.tsx](src/extensions/Board/containers/BoardPage/BoardPage.tsx):
  - memoized filtered board card maps.
  - introduced adaptive initial hydration batch size (25 on narrow screens, 50 otherwise).
  - refactored tab rendering flow to reduce nested conditional rendering churn.
- Improved list-column rerender boundaries in [src/extensions/List/containers/BoardPage/ListColumn.tsx](src/extensions/List/containers/BoardPage/ListColumn.tsx):
  - memo equality now compares only cards belonging to the list instead of full board card map reference.
  - includes hydration flags in memo equality checks.

## New
- Added [db/migrations/0113_board_get_perf_indexes.ts](db/migrations/0113_board_get_perf_indexes.ts) with targeted board-load indexes:
  - `cards (list_id, archived, position)`
  - `card_labels (card_id)`
  - `card_members (card_id)`
  - `checklists (card_id)`
  - `checklist_items (checklist_id)`
  - partial indexes for board-load counters:
    - `comments (card_id) WHERE deleted = false`
    - `attachments (card_id) WHERE status = 'READY'`
- Extended [src/extensions/Board/api.ts](src/extensions/Board/api.ts) `getBoard` query builder to support optional `includeActivities` and to emit query params via `URLSearchParams`.
- Extended [src/extensions/Board/slices/boardSlice.ts](src/extensions/Board/slices/boardSlice.ts) `fetchBoardDataThunk` to accept an optional initial hydration override.

## Technical Debt
- `ROW_NUMBER()` ranking assumes runtime DB support for window functions; supported in Postgres (primary target) and modern SQLite, but should be verified in all deployment variants.
- Board rendering still passes full `cards` map through [src/extensions/Board/components/BoardCanvas.tsx](src/extensions/Board/components/BoardCanvas.tsx); further selector-level isolation or list-level card projection can reduce rerenders further.
