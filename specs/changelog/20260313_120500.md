# Changelog — 2026-03-13 12:05:00

## Update

- Kept the email verification login block in `POST /api/v1/auth/token` when `EMAIL_VERIFICATION_ENABLED=true` and `email_verified=false`.
- Added auto-resend behavior in login: when blocked for `email-not-verified`, the server regenerates verification token/expiry and sends a fresh verification email (rate-limited).
- Updated Trello import seed user upsert to also update `email_verified`, so re-running the seed fixes previously imported users that remained unverified.

## New

- No new endpoints or modules.

## Technical Debt

- Login now performs email sending side effects in the auth path. If this becomes a bottleneck, move resend orchestration to a dedicated queue worker.
