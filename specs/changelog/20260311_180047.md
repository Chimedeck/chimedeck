## Update
- Refactored `requestUploadUrl.ts` to perform MIME type and file size validation before DB lookup, improving efficiency and error handling.
- Error responses for invalid MIME type and file size now use standardized REST error envelopes (`name: 'mime-type-not-allowed'`, `name: 'file-too-large'`).

## New
- Migration `0033_attachments_enhanced.ts` adds six new columns to the `attachments` table: `thumbnail_key`, `content_type`, `width`, `height`, `scan_completed_at`, `scan_vendor`.
- Centralized config `allowedTypes.ts` defines `ALLOWED_MIME_TYPES` (24 types) and `MAX_FILE_SIZE_BYTES` (100 MB).
- Integration test `mimeValidation.test.ts` verifies allowed/disallowed MIME types and file size validation (11 tests, all passing).
