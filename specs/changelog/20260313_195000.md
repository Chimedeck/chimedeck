## Update

- `src/extensions/Workspace/api.ts`: Added `GUEST` to the `Role` type and added an optional `callerRole` field to the `Workspace` interface. The `GET /api/v1/workspaces` endpoint already returns `callerRole` in its payload; surfacing it in the type allows the client to derive the current user's workspace role without a separate members request.
- `src/extensions/Workspace/components/Sidebar.tsx`: Imports the new `selectIsGuestInActiveWorkspace` selector. GUEST users no longer see the **Members** nav link. The active workspace name in the switcher and the Boards nav link both display a small amber `guest` badge when the caller is a GUEST.
- `src/extensions/Workspace/containers/WorkspacePage/WorkspacePage.tsx`: Imports `selectIsGuestInActiveWorkspace`. When the current user is a GUEST, the page renders an **Access Restricted** panel instead of the member list and invite controls.
- `src/extensions/Workspace/routes.ts`: Added route `/workspaces/:workspaceId/boards` pointing to the new `WorkspaceDashboard` container. This matches the path already used by the Sidebar's Boards nav link, replacing the previously mismatched `/workspace/:workspaceId/boards` path in the Board extension.

## New

- `src/extensions/Workspace/slices/workspaceSlice.ts` — two memoised selectors:
  - `selectCurrentUserWorkspaceRole` — reads `callerRole` from the active `Workspace` object in the Redux store.
  - `selectIsGuestInActiveWorkspace` — convenience boolean predicate (`role === 'GUEST'`).
- `src/extensions/Workspace/containers/WorkspaceDashboard/index.tsx` — guest-aware workspace board listing page. Reuses `BoardListPage.duck` selectors and thunks. Differences from plain `BoardListPage`:
  - **Create Board** button hidden for GUEST users.
  - **"guest access"** badge rendered in the page heading for GUEST users.
  - Each board card is wrapped in a `<div>` that positions a **"guest"** badge in the top-right corner when the user is a GUEST.
  - Empty-state copy is contextualised for guests.
- `specs/tests/sprint-80-iteration-8-guest-workspace-view.md` — Playwright MCP test walkthrough covering: guest login, sidebar badge verification, board grid scoping, Members link hidden, Members page access-restricted notice, opening a granted board, and regular-member control test.

## Technical Debt

- `selectCurrentUserWorkspaceRole` depends on `callerRole` being populated in the `workspaceDuck` workspaces array. If the store is pre-populated from a source that omits `callerRole` (e.g. a cached response), the selector will return `null` and the UI will fall back to non-guest rendering. Consider adding a migration/version guard for the persisted Redux store if local persistence is introduced.
- The Board extension's `/workspace/:workspaceId/boards` route in `Board/routes.ts` is now shadowed by the Workspace extension's `/workspaces/:workspaceId/boards` route. The Board extension route should be removed or updated to avoid confusion.
