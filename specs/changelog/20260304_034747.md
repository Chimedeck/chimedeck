# Changelog – 20260304_034747

## Update
- `src/extensions/Board/api.ts`: Added `MonetizationType` type (`'pre-paid' | 'pay-to-paid'`) and `monetization_type: MonetizationType | null` field to the `Board` interface. Added `patchBoardMonetizationType` function for PATCH requests that update only monetization_type.
- `src/extensions/Board/slices/boardSlice.ts`: Added `optimisticUpdateBoardMonetization` reducer action.
- `src/extensions/Board/components/BoardHeader.tsx`: Added `onOpenSettings` optional prop; settings dropdown now shows "Board settings" button when provided.
- `src/extensions/Board/components/BoardCanvas.tsx`: Added `monetizationType` prop, threads it into each `SortableListColumn`.
- `src/extensions/Card/components/CardItem.tsx`: Added `monetizationType?` and `listName?` props (pass-through, ready for Sprint 33 payment buttons).
- `src/extensions/List/containers/BoardPage/ListColumn.tsx`: Added `monetizationType` prop, passes it along with `listName` (the list's title) to each `CardItem`.
- `src/extensions/Board/containers/BoardPage/BoardPage.tsx`: Wires up settings panel state, `handleSaveMonetizationType` with optimistic update + rollback, passes `monetizationType` to `BoardCanvas`, and renders `BoardSettings` panel when open.

## New
- `src/extensions/Board/config/payToPaidConfig.ts`: Exports `shouldShowPaymentButtons(listName: string): boolean` — a configurable predicate that returns true for qualifying column names (Done, Completed, Delivered, Ready to Pay, Invoiced). Used by Sprint 33 for payment button visibility.
- `src/extensions/Board/containers/BoardSettings/BoardSettings.tsx`: Slide-in settings panel with a **Monetization** radio group (None / Pre-paid / Pay to Paid). Includes optimistic save, error display, and RBAC-aware disabled state for non-admins.
