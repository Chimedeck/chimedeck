# Changelog — 20260317_122357

## Update
- Registered `offlineDraftsRouter` in `server/index.ts` alongside all other extension routers.

## New
- `db/migrations/0083_user_card_drafts.ts` — Migration creates `user_card_drafts` table with unique constraint on `(user_id, card_id, draft_type)` and indexes on `(user_id, updated_at desc)` and `(card_id)`.
- `server/extensions/offlineDrafts/mods/drafts.ts` — Service layer: `getDraftsByCard`, `upsertDraft` (insert-or-update), `deleteDraft`. All operations are scoped to `userId` — callers never pass arbitrary user IDs.
- `server/extensions/offlineDrafts/api/index.ts` — API router for `GET /api/v1/cards/:cardId/drafts`, `PUT /api/v1/cards/:cardId/drafts/:type`, `DELETE /api/v1/cards/:cardId/drafts/:type`. Auth required; reads/writes forcibly scoped to current user; 404 returned for missing drafts (no existence leak).
- `specs/tests/offline-description-draft-recovery.md` — 17 Playwright MCP scenarios covering CRUD, privacy isolation, error cases, intent states, and cross-device continuity for the draft API.

## Technical Debt
- `resolveCard` in the draft API uses a raw SQL join to check board membership. This logic is duplicated across several other card-access paths. Should be extracted to a shared `requireCardAccess` middleware in a future refactor.
