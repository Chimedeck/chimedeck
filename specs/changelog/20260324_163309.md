# Changelog — 2026-03-24 16:33

## Update
- `src/reducers.ts`: Exported `healthCheckTabReducer` from the HealthCheck feature.
- `src/store/index.ts`: Registered `healthCheckTab` slice in the root Redux store.

## New
- `src/extensions/HealthCheck/api.ts`: API client functions for all Health Check endpoints (fetch list, add, remove, probe single, probe all, fetch presets). TypeScript types match backend response shapes.
- `src/extensions/HealthCheck/containers/HealthCheckTab/HealthCheckTab.duck.ts`: RTK slice with state (`entries`, `probingIds`, `lastCheckedAt`, `status`, `error`, `presets`, `presetsStatus`), six async thunks (`fetchHealthChecksThunk`, `fetchPresetsThunk`, `addHealthCheckThunk`, `removeHealthCheckThunk`, `probeSingleThunk`, `probeAllThunk`), and typed selectors.
- `src/extensions/HealthCheck/reducers.ts`: Re-exports the HealthCheckTab reducer for integration into root reducers.
- `src/extensions/HealthCheck/config/healthCheckConfig.ts`: Frontend config constants — poll interval, amber threshold, feature flag.

## Technical Debt
- `createAppAsyncThunk` in `src/utils/redux.ts` does not include the `extraArgument` type, causing `extra` to be `unknown` in all async thunks project-wide. The HealthCheck duck works around this with `(extra as any).api`. This should be fixed at the utility level in a future refactor.
