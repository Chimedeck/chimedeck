# Changelog — 2026-03-18T04:53:43Z

## Update

- `src/extensions/Realtime/client/socket.ts`: Made `boardId` optional in `connect()`. The socket now connects and registers in the user's personal WS channel even without subscribing to a board room. Board room subscribe message is only sent when `boardId` is present.

## New

- `src/extensions/Board/realtime.ts`: New `useWorkspaceSync` hook. Connects the singleton WebSocket (personal user channel only) and dispatches `boardRemovedByRealtime` when a `board_deleted` event arrives for the current workspace. Used on the workspace boards page to propagate remote deletions to all connected users without a page reload.
- `src/pages/WorkspaceBoardsPage.tsx`: Thin re-export of `BoardListPage` for consumers that import from the canonical pages directory.

## Technical Debt

- The `extra` is of type `unknown` TypeScript errors in `BoardListPage.duck.ts` thunks are pre-existing and unrelated to this iteration.
- The `useWorkspaceSync` hook shares the same singleton socket as `useWebSocket` (used on board pages). In concurrent mount scenarios (unlikely in SPA) a second `connect()` call would close the first connection. A multi-socket approach would be more robust but is deferred.
