# Sprint 25 — @Mentions in Cards & Comments

## Update

- `server/extensions/card/api/update.ts` — wrapped card update in a DB transaction; added `syncMentions` call when `description` is saved.
- `server/extensions/comment/api/create.ts` — wrapped comment insert in a DB transaction; added `syncMentions` call on creation.
- `server/extensions/comment/api/update.ts` — wrapped comment update in a DB transaction; added `syncMentions` call on edit.
- `server/extensions/board/api/index.ts` — registered new `GET /api/v1/boards/:id/members/suggestions` route.
- `src/extensions/Card/components/CardDescription.tsx` — replaced bare `<textarea>` with `<MentionInput>`; added required `boardId` prop; removed now-unnecessary auto-resize logic (handled by the component).
- `src/extensions/Card/components/CardModal.tsx` — added `boardId` prop; passed it to `CardDescription` and `CommentThread`.
- `src/extensions/Card/containers/CardModal/index.tsx` — passed `boardId` to `CardModal`.
- `src/extensions/Comment/components/CommentThread.tsx` — added optional `boardId` prop; forwarded to `CommentEditor` and `CommentItem`.
- `src/extensions/Comment/components/CommentItem.tsx` — added optional `boardId` prop; forwarded to inline `CommentEditor`.
- `src/extensions/Comment/components/CommentEditor.tsx` — added optional `boardId` prop; renders `<MentionInput>` when provided, falls back to bare `<textarea>`.

## New

- `db/migrations/0016_mentions.ts` — creates `mentions` table with source-type/source-id/user indexes; no soft-delete (re-derived on save).
- `server/common/mentions/parse.ts` — `extractMentions({ text })` extracts unique @-nickname strings using `/\B@([\w-]{1,50})/g`.
- `server/common/mentions/resolve.ts` — `resolveNicknames({ nicknames, boardId })` resolves nicknames to user records scoped to the board's workspace membership.
- `server/common/mentions/sync.ts` — `syncMentions(...)` diffs old vs new mentions inside a transaction; deletes stale rows and inserts new ones; returns `addedUserIds` for future notification dispatch.
- `server/extensions/board/api/members/suggestions.ts` — `GET /api/v1/boards/:boardId/members/suggestions?q=` returns ≤10 members whose nickname or name starts with `q`.
- `src/common/components/MentionInput/useMentionInput.ts` — hook detecting `@<partial>` trigger, debounced fetch (150 ms), keyboard navigation, and insertion logic.
- `src/common/components/MentionInput/MentionChip.tsx` — suggestion row component (avatar/initials + nickname + name).
- `src/common/components/MentionInput/MentionSuggestions.tsx` — floating dropdown (`absolute z-50 bg-slate-800`) rendering `MentionChip` rows.
- `src/common/components/MentionInput/MentionInput.tsx` — drop-in `<textarea>` replacement with `boardId` prop wiring mention autocomplete.
- `src/common/components/MentionInput/renderMentions.tsx` — converts plain text @nicknames into indigo chip `<span>` elements with hover tooltip.

## Technical Debt

- `syncMentions` returns `addedUserIds` but no code yet consumes it — notification dispatch is wired in Sprint 26.
- `renderMentions` is implemented but not yet called in read/preview mode of `CardDescription` (description preview currently renders via `marked`). Mention chips in preview mode should be layered on top of the markdown render in a future pass.
