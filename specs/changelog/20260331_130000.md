## New

### `terraform/modules/ec2-single/`

Provisions a single EC2 instance with a static Elastic IP. User data is rendered from a `startup.sh` templatefile receiving `ecr_image_url` and `secrets_arn` so the instance can pull its Docker image and load secrets at boot without storing credentials in the AMI. The instance uses `create_before_destroy` to enable clean replacement on image updates.

Files: `main.tf`, `variables.tf`, `outputs.tf`, `README.md`

### `terraform/modules/vpc-peering/`

Creates a cross-region VPC peering connection between two VPCs and updates route tables on both sides. Requires two provider aliases (`aws.requester`, `aws.accepter`) declared via `configuration_aliases` in `required_providers`. Default scope is same-account peering; cross-account requires additional configuration not in this module.

Files: `main.tf`, `variables.tf`, `outputs.tf`, `README.md`

## Technical Debt

- `ec2-single`: `create_before_destroy` on `aws_instance` may cause downtime if the EIP is disassociated before the new instance is associated. A more robust solution would use a secondary EIP swap, deferred to when lifecycle management becomes a concern.
- `vpc-peering`: Cross-account peering support (requires `aws_vpc_peering_connection_options` on both sides) is not yet implemented. Flagged for a future iteration if multi-account deployments are required.
