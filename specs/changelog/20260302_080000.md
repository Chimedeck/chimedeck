# Sprint 15 — UI Foundation

## Update

- Added `paths: { "~/*": ["./src/*"] }` to `tsconfig.json` so Bun's test runner
  resolves `~/…` module aliases in `src/extensions/` correctly.
- Added `@reduxjs/toolkit` extra-argument pattern to the Redux store so async
  thunks receive `extra.api` (the axios instance) instead of importing it
  directly — this is consistent with the pattern already used by Board / List
  ducks.

## New

- **Build pipeline**: `vite.config.ts`, `tailwind.config.ts`, `postcss.config.js`,
  `index.html`, `tsconfig.client.json` — Vite 5 + React 18 + Tailwind CSS 3
  client build targeting port 5173 with `/api` proxy to Bun server.

- **React entry points**: `src/main.tsx`, `src/App.tsx`, `src/index.css` —
  React root with Redux Provider; Tailwind base styles.

- **Redux store**: `src/store/index.ts` with `auth` (from Auth extension duck)
  and `ui` slices; axios `apiClient` injected via `thunk.extraArgument`.

- **Auth slice**: `src/slices/authSlice.ts` (legacy shell),
  `src/slices/uiSlice.ts` (theme, sidebar, modal state).

- **Auth extension duck**: `src/extensions/Auth/duck/authDuck.ts` — RTK slice
  with `loginThunk` / `logoutThunk`, `setCredentials`, `clearAuth`; selectors
  for `isAuthenticated`, `user`, `accessToken`, `status`.

- **Auth extension**: `src/extensions/Auth/routes.ts`,
  `src/extensions/Auth/PrivateRoute.tsx`,
  `src/extensions/Auth/reducers.ts`,
  stub `LoginPage` / `SignupPage` containers (full UI in Sprint 16).

- **API client**: `src/common/api/client.ts` — axios instance with lazy token
  getter wired from `main.tsx`; `src/common/api/interceptors.ts` — 401
  handler with refresh-or-logout logic via lazy `clearAuth` callback.

- **Common extension API**: `src/extensions/common/api/index.ts`,

- **Routing shell**: `src/routing/index.tsx` — `BrowserRouter` with lazy-loaded

- **Shared components**:

## Technical Debt

- `src/slices/authSlice.ts` is kept for historical reference but its reducer is
  NOT registered — `authDuckReducer` in `src/extensions/Auth/duck/authDuck.ts`
  is the active implementation. Should be removed or converted to re-exports in
  a future cleanup.
- `AxiosInstance` type cast in `authDuck.ts` thunks (`extra as { api: AxiosInstance }`)
  — add a typed `AppExtra` interface to `src/store/index.ts` and
  `src/utils/redux.ts` in Sprint 16.
- `TopbarContainer` and `FooterContainer` are stubs; full implementation in
  Sprint 17.
