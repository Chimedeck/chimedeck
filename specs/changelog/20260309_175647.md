# Changelog — 20260309_175647

## Sprint 45 — Admin Invite External Users UI

### Update
- **`src/extensions/Workspace/components/Sidebar.tsx`** — Added "Invite External User" nav item with `UserPlusIcon`. Visible only to users whose email domain matches `ADMIN_EMAIL_DOMAINS` (evaluated client-side from the `featureFlags` Redux slice). Clicking dispatches `openInviteModal`.
- **`src/layout/AppShell.tsx`** — Now fetches `featureFlags` on mount alongside workspaces and profile. Renders `InviteExternalUserModal` globally.
- **`src/store/index.ts`** — Added `adminInvite` and `featureFlags` reducers.
- **`src/reducers.ts`** — Exported `adminInviteReducer`.
- **`server/index.ts`** — `/api/v1/flags` endpoint now returns `sesEnabled`, `adminInviteEmailEnabled`, and `adminEmailDomains` (previously returned empty `{}`).

### New
- **`src/extensions/AdminInvite/types.ts`** — TypeScript types: `AdminCreateUserRequest`, `AdminCreateUserResponse`, `PasswordMode`, `AdminInviteState`.
- **`src/extensions/AdminInvite/api.ts`** — `adminInviteApi.createUser()` wrapping `POST /api/v1/admin/users`.
- **`src/extensions/AdminInvite/adminInvite.slice.ts`** — Redux slice managing `isOpen`, `credentials`, and `emailSent`. Actions: `openInviteModal`, `closeInviteModal`, `setInviteCredentials`.
- **`src/extensions/AdminInvite/InviteExternalUserModal.tsx`** — Modal with email, display name, password mode radio (auto/manual), password strength bar (4 levels), send-email toggle (gated by `selectShowEmailToggle`), inline field errors, and server error display. On success shows CredentialSheet.
- **`src/extensions/AdminInvite/CredentialSheet.tsx`** — Formatted copyable block showing email, password, and login URL. "Copy to clipboard" and "Done" buttons. Email-sent notice when `emailSent=true`.
- **`src/extensions/AdminInvite/README.md`** — Feature documentation.
- **`src/slices/featureFlagsSlice.ts`** — Global feature flags slice. Fetches from `/api/v1/flags` via `fetchFeatureFlagsThunk`. Exposes `selectAdminEmailDomains`, `selectShowEmailToggle`, etc.
- **`tests/integration/adminInvite/adminInvite.ui.test.ts`** — 17 integration tests covering: admin domain detection, Redux slice open/close/credential behaviour, feature flag toggle logic, and credential clipboard format.

### Technical Debt
- The `/api/v1/flags` endpoint returns flags without authentication. Future hardening: add rate-limiting or restrict to authenticated callers.
- `WorkspaceModal.test.tsx` has pre-existing TypeScript errors (partial store missing many `RootState` keys) unrelated to this sprint.

### What Should Be Done Next
- Playwright E2E tests against the running server to verify: sidebar item visibility per user domain, modal open/close via Escape, full form submit flow, and credential sheet copy.
- Consider moving `featureFlagsSlice.ts` fetching to be authenticated only (after login) rather than on any app boot.
