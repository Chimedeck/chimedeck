## New

- **`db/migrations/0032_custom_fields.ts`**: Creates `custom_fields` table (id, board_id, name, field_type, options, show_on_card, position, created_at) and `card_custom_field_values` table (id, card_id, custom_field_id, value_text, value_number, value_date, value_checkbox, value_option_id). FK cascades ensure values are cleaned up when a field or board is deleted.
- **`server/extensions/customFields/api/fieldDefinitions.ts`**: Implements GET (list), POST (create), PATCH (update), DELETE handlers for `/api/v1/boards/:id/custom-fields[/:fieldId]`. Only ADMIN+ roles can mutate; GET respects board visibility rules. Field types validated against `TEXT | NUMBER | DATE | CHECKBOX | DROPDOWN`. DROPDOWN options validated as an array.
- **`server/extensions/customFields/api/index.ts`**: Router that pattern-matches `/api/v1/boards/:boardId/custom-fields[/:fieldId]` and dispatches to the appropriate handler.
- **`server/extensions/customFields/index.ts`**: Extension entry point re-exporting the router.
- **`tests/e2e/custom-fields-api.md`**: Playwright MCP markdown test covering field creation (all 5 types), listing, rename, option update, show_on_card toggle, validation errors (missing name, invalid type, bad options), 404 on unknown field, 204 on delete, and 403 for non-ADMIN role.
