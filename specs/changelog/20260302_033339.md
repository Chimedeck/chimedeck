# Changelog — 2026-03-02T03:33:39Z

## Sprint 02: Build Tooling & Docker

### Update
- `Dockerfile` — replaced single-stage with multi-stage production build (deps → build → production). Non-root user `app`, layer caching on `package.json`+`bun.lock`, `wget`-based healthcheck (alpine has no curl by default).
- `docker-compose.yml` — updated to use `Dockerfile.dev`, added named compose project (`kanban-dev`), switched from inline `environment` to `env_file: .env`, added volume mounts for hot-reload, improved postgres healthcheck.
- `.github/workflows/ci.yml` — split single `ci` job into `quality` + `docker-build` jobs. `docker-build` uses `docker/build-push-action@v5` with GitHub Actions layer cache.
- `package.json` — expanded scripts: dev/build/start/lint/format/typecheck/test/docker flows. Added `concurrently` as devDependency.
- `.env.example` — added `APP_PORT`, `NODE_ENV`, `DOCKER_IMAGE`, `IMAGE_TAG` entries.

### New
- `Dockerfile.dev` — development Dockerfile (hot-reload, source mounted as volume, no source copy).
- `docker-compose.prod.yml` — production compose file using pre-built image + `env_file: .env.production`, no hardcoded secrets, Redis always on, deploy replicas config.
- `Makefile` — convenience wrappers for common `bun run` commands (`dev`, `build`, `test`, `lint`, `typecheck`, `docker`, `clean`).

### Technical Debt
- `bun run build:client` and `bun run dev:client` reference Vite which is not yet installed — these scripts are present for future use (Sprint 03+ will add the client build pipeline).
- Production Dockerfile build stage runs `typecheck` only; once client assets exist, `build:client` should be added back.
