# Update
- Fixed direct archive/update/delete notification fanout to avoid being suppressed by board-activity dedupe when the same card triggers the same notification type multiple times.

# New
- Direct non-comment card notifications now use an event-unique `source_id` (`cardId:timestamp`) so each action instance persists as its own notification row.
- Comment notifications still keep `commentId` as `source_id` for deep-link compatibility.

- `boardActivityDispatch.ts` still has broad pre-existing unsafe-type lint debt unrelated to this focused fix.
