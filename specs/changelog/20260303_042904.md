# Changelog — 20260303_042904

## Update

### Real-time Comment Sync Fixed

Three separate gaps prevented comments from syncing to other browsers over WebSocket:

1. **`server/extensions/comment/api/create.ts`** — `publisher.publish()` was broadcasting only `{ commentId }`. Changed to broadcast `{ comment }` (full object) so the receiving client can render it without a follow-up API request.
2. **`server/extensions/comment/api/update.ts`** — Never called `publisher.publish()` at all. Added broadcast of `{ type: 'comment_updated', payload: { comment } }`.
3. **`server/extensions/comment/api/delete.ts`** — Never called `publisher.publish()` at all. Added broadcast of `{ type: 'comment_deleted', payload: { commentId, cardId } }`.
4. **`src/extensions/Realtime/hooks/useBoardSync.ts`** — The `handleEvent` switch had no `case` entries for any `comment_*` event types, so all incoming comment WS messages were silently dropped. Added handlers for `comment_added` → `cardDetailSliceActions.addComment`, `comment_updated` → `updateComment`, `comment_deleted` → `removeComment`.

### Drag-and-Drop Card Move Fixed

Two bugs combined to prevent card moves from persisting:

1. **HTTP method mismatch (404 root cause)** — The client (`src/extensions/Board/api/card.ts`) was issuing `PATCH /api/v1/cards/:id/move`, but the server route guard matched only `req.method === 'POST'`. Discovered via Playwright network inspection. Fixed `server/extensions/card/api/index.ts` and the handler comment in `server/extensions/card/api/move.ts` to use `PATCH`.
2. **Missing server broadcast** — `handleMoveCard` wrote the event to the DB log but never called `publisher.publish()`, so other connected clients never received the `card_moved` WS event. Added the broadcast with `{ card, fromListId }` payload matching what `useBoardSync` already expected.

### DndContext Re-render Loop Fixed

A secondary React crash (`Maximum update depth exceeded` in `DndContext`) appeared after the HTTP method fix. Root cause: `cardsByList` was added to `handleDragStart`'s `useCallback` dependency array to capture the drag source list. Every optimistic card position update during a drag changed `cardsByList`, which gave `DndContext` a new `onDragStart` reference mid-drag and triggered an infinite re-render.

Fixed in `src/extensions/Board/components/BoardCanvas.tsx` by introducing a `cardsByListRef` that is kept current on every render (`cardsByListRef.current = cardsByList`) but is read inside the callback without being listed as a dependency. `handleDragStart` now stays referentially stable for the lifetime of a drag.

## Technical Debt

- `server/extensions/card/api/move.ts` has a cognitive complexity warning (23 vs limit 15) — should be refactored into smaller helpers (`computePosition`, `validateTarget`).
- `BoardCanvas` `handleDragEnd` is at complexity 16 (limit 15) — minor, but worth extracting the card-move and list-reorder branches into named functions.
- `selectCardDetailMeta` still returns a new object reference on every render (two Redux equality warnings). Should use `createSelector`.
- WebSocket connection to `ws://localhost:5173` fails in Playwright tests because Vite dev server does not proxy WS — only affects test env, not production.
