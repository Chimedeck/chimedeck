## New
- **Table `user_card_drafts`**: stores per-user draft content for card descriptions and comment compose boxes.
  - Columns: `id`, `user_id`, `workspace_id`, `board_id`, `card_id`, `draft_type` (description|comment), `content_markdown`, `intent` (editing|save_pending|submit_pending), `client_updated_at`, `synced_at`, `created_at`, `updated_at`.
  - Unique constraint on `(user_id, card_id, draft_type)` — one draft row per user/card/type, enabling upsert semantics.
  - Index on `(user_id, updated_at)` for efficient per-user draft listing.
  - Index on `(card_id)` for efficient per-card draft lookup.
