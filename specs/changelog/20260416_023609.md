# Changelog — 2026-04-16T02:36:09Z

## Update

- `server/common/cards/cover.ts`: Extended `CardWithResolvedCover` interface with `cover_aspect_ratio: '16:9' | '1:1' | null` and `cover_is_gif: boolean`.

## New

- `resolveCoverImageUrls` now SELECTs `mime_type`, `width`, and `height` from the `attachments` table for cover attachments.
- Aspect ratio is derived: `width / height >= 1.5` → `'16:9'`; otherwise `'1:1'`; `null` when dimensions are missing.
- GIFs (`mime_type === 'image/gif'`) always receive a `/view` proxy URL instead of `/thumbnail` to preserve animation.
- `cover_is_gif` flag propagated on all cover resolution code paths (including empty/no-cover fallbacks).

## Technical Debt

- Existing image covers will have `cover_aspect_ratio: null` until the thumbnail worker reprocesses their attachments. Clients must default to `'1:1'` when null.

## What Should Be Done Next

- Iteration 4: Update `CardItem.tsx` to use adaptive aspect-ratio containers based on `cover_aspect_ratio`.
- Iteration 5: Update `CardModal.tsx` banner + GIF file input support.
