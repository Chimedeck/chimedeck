# Changelog — 20260311_112813

## Update
- `src/extensions/Attachments/hooks/useAttachmentUpload.ts` — new hook orchestrating single-file (≤5 MB, `XMLHttpRequest` `onprogress`) and multipart (>5 MB, up to 3 concurrent parts) upload flows with per-entry progress tracking and abort support.
- `src/extensions/Attachments/hooks/useClipboardPaste.ts` — new hook that listens for `paste` events on `document` while `enabled`, extracts `image/*` blobs from `DataTransfer`, and names them `pasted-image-<timestamp>.png`.
- `src/extensions/Attachments/components/AttachmentDropZone.tsx` — new full-panel overlay component that activates on drag-enter, shows dashed blue border + `ArrowUpTrayIcon`, and fires `onFiles` on drop.
- `src/extensions/Attachments/components/PasteListener.tsx` — new invisible component that wraps `useClipboardPaste` and forwards extracted image files to the upload handler.

## New
- Upload orchestration: single-file path uses XHR `onprogress` for real-time progress; multipart path splits into 5 MB chunks with `runConcurrent` (max 3 parts in flight simultaneously).
- Multipart abort on error: if any part fails, `abortMultipart` is called automatically to clean up the incomplete upload on S3.
- `AbortController` per upload entry allows individual uploads to be cancelled via `removeEntry`.

## Technical Debt
- `useClipboardPaste` calls `ev.preventDefault()` only when image items are found; if the user pastes inside a focused text input and the clipboard also contains an image, the text paste will be suppressed. A future improvement could check if the active element is a text input before preventing default.
