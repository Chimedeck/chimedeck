# Changelog — 20260313_154200

## Update
- `tailwind.config.ts`: Already had `darkMode: 'class'` — no change needed.
- `src/main.tsx`: Added theme initialisation block that reads `localStorage('theme')` and applies/removes the `dark` class on `<html>` before React hydration, preventing flash of wrong theme.

## New
- `src/index.css`: Added CSS custom properties for both palettes (`:root` for light, `.dark` for dark). Covers `--bg-base`, `--bg-surface`, `--text-base`, and `--border` tokens so non-Tailwind surfaces respond to theme changes.
- `src/common/hooks/useTheme.ts`: New hook that reads/writes theme from `localStorage` and toggles the `dark` class on `<html>`. Returns `{ theme, toggle }`.
- `src/common/components/ThemeToggle.tsx`: New component using `useTheme`. Shows `SunIcon` in dark mode (click to switch to light) and `MoonIcon` in light mode (click to switch to dark) from `@heroicons/react/24/outline`. Accessible `aria-label` per mode.
- `specs/tests/theme-toggle-persist.md`: Playwright MCP test covering toggle to light mode, persistence after refresh, toggle back to dark mode, and default-to-dark when no preference stored.

## Technical Debt
