# Iteration 10: Plugin Popup + Modal + show-settings + End-to-End Wiring

## Update
- `usePluginBridge.ts`: Added handlers for `UI_MODAL`, `UI_CLOSE_MODAL`, `UI_UPDATE_MODAL`, `UI_POPUP`, `UI_CLOSE_POPUP`, `UI_SIZE_TO` messages from the plugin SDK. The hook now accepts optional callbacks (`onOpenModal`, `onCloseModal`, `onUpdateModal`, `onOpenPopup`, `onClosePopup`, `onSizeTo`) that are invoked when the corresponding postMessage is received from a plugin iframe.
- `PluginIframeContainer.tsx`: Now owns modal and popup React state. Passes the new bridge callbacks and renders `<PluginModal>` and `<PluginPopup>` overlays (via `createPortal`) so they appear above all other UI.
- `PluginDashboardPage.tsx`: Added settings gear icon flow — clicking the gear on an active plugin opens a `PluginModal` loaded with the plugin's `connectorUrl?context=show-settings`.
- `PluginList.tsx`: Added optional `onSettings` prop and passes it to `PluginCard` for active plugin rows.
- `PluginCard.tsx`: Added gear icon button for active plugins (`mode='disable'`) that have `show-settings` in their capabilities list.

## New
- `src/extensions/Plugins/modals/PluginModal.tsx`: Full modal overlay rendered via `createPortal`. Contains an `<iframe>` loaded with the URL returned by `t.modal()`. Supports `fullscreen` toggle, `accentColor` header tint, `updateModal({ title, fullscreen })`, and closes on Escape or backdrop click.
- `src/extensions/Plugins/modals/PluginPopup.tsx`: Small floating popup rendered via `createPortal`. Positioned near the triggering mouse coordinates (clamped to viewport). Contains an `<iframe>` with `args` passed as query params. Closes on Escape or click-outside.

## Technical Debt
- Modal `sizeTo` callback is wired through the bridge but not yet applied to the modal iframe container height (requires ResizeObserver on the iframe content). Deferred for now.
- The `show-settings` flow on PluginDashboardPage opens the connector URL directly in a PluginModal rather than going through `bridge.resolve('show-settings', ctx)` (the bridge is not active on the settings route). A proper implementation would require mounting a lightweight bridge on the settings page.

## What Should Be Done Next
- Implement `sizeTo` height application in `PluginModal` / `PluginPopup` using the `UI_SIZE_TO` height value.
- Write E2E Playwright tests for the modal open/close/update flow.
- Add a proper `show-settings` capability resolution path from the PluginDashboardPage (lightweight bridge or navigate-and-invoke pattern).
