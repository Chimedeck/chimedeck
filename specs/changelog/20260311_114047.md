## Technical Debt

- `AttachmentPanel` polls the server list on each successful upload rather than using a WebSocket subscription. Status transitions (SCANNING → READY / REJECTED) need a WS listener to update in real time; currently the user must refresh.
- Toast notifications for MIME/size errors are described in tests but not yet wired (the server returns 400/413; the panel currently shows no UI feedback for these — needs error-handling in `useAttachmentUpload`).
