# Changelog — 2026-03-05T02:19:00Z

## Update
- `server/extensions/plugins/config/index.ts` was already referencing `sdkBundlePath` pointing to `public/sdk/jh-instance.js` and `server/index.ts` was already serving the file at `/sdk/jh-instance.js` with `Content-Type: application/javascript` (implemented in a prior iteration).

## New
- **`server/extensions/plugins/sdk/jh-instance.ts`** — Source TypeScript for the `jhInstance` browser SDK. Exposes `window.jhInstance` with:
  - `initialize(capabilities, config)` — registers capability handlers and posts `PLUGIN_READY` to the host.
  - `iframe()` — returns a `FrameContext` populated from the URL query string (for modal/section pages).
  - `FrameContext` class — provides `t.get/set`, `t.card/list/board/member`, `t.modal/popup/closeModal/updateModal/sizeTo/render`, and `t.getRestApi()`.
  - `window.TrelloPowerUp = window.jhInstance` compatibility shim so existing Power-Up `client.js` files work unchanged.
- **`public/sdk/jh-instance.js`** — Minified browser bundle built from `jh-instance.ts` via `bun build --target browser --minify` (2.18 KB).

## Technical Debt
- `RestApiClient.request()` returns a raw `Promise<Response>` — serialisation across the postMessage boundary is not yet implemented; will be needed in iteration 10.
- `UI_SIZE_TO` message is posted fire-and-forget; the host postMessage bridge (iteration 8) must handle it.
