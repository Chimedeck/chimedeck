# Changelog — 2026-03-10 03:30:09

## Update
- `src/extensions/Board/api.ts`: Added `isStarred?: boolean` to `Board` interface; added `starBoard()` and `unstarBoard()` API functions.
- `src/extensions/Board/components/BoardCard.tsx`: Added star/unstar toggle button (filled yellow star when starred, outline when not) to each board tile. Clicking the star stops event propagation and calls `onStar`/`onUnstar`.
- `src/extensions/Board/containers/BoardListPage/BoardListPage.duck.ts`: Added `showStarredOnly` filter state, `starBoardThunk`, `unstarBoardThunk` thunks, `toggleStarredFilter` action, `showStarredOnlySelector`, and `visibleBoardsSelector` (returns only starred boards when filter is active).
- `src/extensions/Board/containers/BoardListPage/BoardListPage.tsx`: Uses `visibleBoardsSelector` and new star/unstar dispatchers; renders a "⭐ Starred boards" filter chip that toggles the filter.
- `src/extensions/Board/containers/BoardPage/BoardPage.tsx`: Added a Board / Activity / Comments / Archived Cards tab bar beneath `BoardHeader`. Switching tabs renders the matching panel; restoring an archived card refreshes board data and switches back to the Board tab.

## New
- `src/extensions/BoardViews/types.ts`: TypeScript interfaces — `BoardActivityEntry`, `BoardComment`, `ArchivedCard`, `PaginatedResponse<T>`.
- `src/extensions/BoardViews/api.ts`: Client API helpers — `getBoardActivity`, `getBoardComments`, `getArchivedCards`, `starBoard`, `unstarBoard`, `getStarredBoards` — all using the shared `apiClient`.
- `src/extensions/BoardViews/BoardActivityPanel.tsx`: Paginated activity feed for a board, reuses `ActivityFeed` / `ActivityItem` components; cursor-based load-more.
- `src/extensions/BoardViews/BoardCommentsPanel.tsx`: Paginated cross-board comments list showing author, card title, content, and timestamp; cursor-based load-more.
- `src/extensions/BoardViews/BoardArchivedCardsPanel.tsx`: Lists all archived cards in a board; each card has a Restore button that calls `PATCH /cards/:id { archived: false }` and optimistically removes the card from the list.

## Technical Debt
- The `extra is unknown` TypeScript errors in `BoardListPage.duck.ts` are pre-existing across all thunks (not introduced here); they stem from the `createAppAsyncThunk` type configuration.
- `BoardArchivedCardsPanel` calls `apiClient` directly (not through Redux) for the restore action — acceptable for a simple one-off mutation but could be unified with the card slice later.
