# Sprint 12 — Attachments

## Update

- `server/config/env.ts` — added `VIRUS_SCAN_ENABLED` and `VIRUS_SCAN_API_KEY` env vars.
- `server/index.ts` — registered `attachmentRouter` and orphan-cleanup scheduler.

## New

### Migration
- `db/migrations/0011_attachments.ts` — creates `attachments` table with `type` (FILE | URL), `status` (PENDING | READY | REJECTED), S3 fields, and a cascade delete on card removal.

### Server extension: `server/extensions/attachment/`
- `common/config/s3.ts` — singleton `S3Client` and config sourced exclusively from `env`.
- `mods/s3/presignPut.ts` — pre-signed S3 PUT URL (TTL 5 min).
- `mods/s3/presignGet.ts` — pre-signed S3 GET URL (TTL 15 min) with `expiresAt`.
- `mods/s3/headObject.ts` — checks S3 object existence without downloading.
- `mods/s3/deleteObject.ts` — deletes an S3 object by key.
- `mods/virusScan/enqueue.ts` — pushes scan job to pubsub queue; no-op when `VIRUS_SCAN_ENABLED=false`.
- `mods/virusScan/worker.ts` — subscribes to scan queue, calls scan API, delegates to `update`.
- `mods/virusScan/update.ts` — updates `status`, fires `attachment_updated` WS event.
- `mods/orphanCleanup.ts` — interval job (every 15 min) removes PENDING FILE attachments > 1 hour old and their S3 objects.
- `api/requestUploadUrl.ts` — `POST /api/v1/cards/:id/attachments/upload-url` (MEMBER).
- `api/confirmUpload.ts` — `POST /api/v1/cards/:id/attachments` (MEMBER, verifies HeadObject, enqueues scan).
- `api/addUrl.ts` — `POST /api/v1/cards/:id/attachments/url` (MEMBER, SSRF guard).
- `api/delete.ts` — `DELETE /api/v1/attachments/:id` (owner or ADMIN, S3 cleanup).
- `api/getSignedUrl.ts` — `GET /api/v1/attachments/:id/url` (VIEWER, gates on status).
- `api/index.ts` — router wiring all attachment routes.

### Frontend extension: `src/extensions/Attachment/`
- `components/AttachmentStatusBadge.tsx` — PENDING / READY / REJECTED badge.
- `components/AttachmentItem.tsx` — file icon, name, status, download/open, delete button.
- `components/AttachmentUploader.tsx` — drag-and-drop zone with native XHR progress.
- `components/AttachmentUrlModal.tsx` — modal form for adding external URL attachments.
- `components/AttachmentSection.tsx` — full panel integrating all attachment sub-components.
- `hooks/useAttachmentUpload.ts` — three-step S3 upload orchestrator.

### Tests
- `server/extensions/attachment/__tests__/ssrfValidator.test.ts` — 13 cases covering all blocked IP ranges and IPv6.
- `server/extensions/attachment/__tests__/orphanCleanup.test.ts` — 4 TTL boundary tests.
- `server/extensions/attachment/__tests__/uploadFlow.test.ts` — VIRUS_SCAN_ENABLED bypass, URL validation, delete module shape.

## Technical Debt

- `GET /cards/:id/attachments` list endpoint is not implemented; `AttachmentSection` has a `refresh()` stub. This should be added in a future sprint.
- `virusScan/worker.ts` uses VirusTotal API stub; full streaming of S3 objects for ClamAV/VirusTotal is deferred.
- Virus-scan queue uses pubsub channel; should migrate to a dedicated Redis `LPUSH`/`BRPOP` queue for durability.

## What Should Be Done Next

- Add `GET /api/v1/cards/:id/attachments` to list attachments.
- Wire `AttachmentSection` into the card-detail modal in the Board UI.
- Implement robust ClamAV/VirusTotal streaming integration.
- Consider presigned URL caching to reduce S3 API calls for frequently accessed attachments.
