# Changelog — 2026-03-05T02:32

## Update

- `src/extensions/Board/containers/BoardPage/BoardPage.tsx`: imports and mounts `<PluginIframeContainer boardId={boardId} />` at the bottom of the board root so plugin iframes are injected whenever a board is viewed.

## New

- `src/extensions/Plugins/iframeHost/usePluginBridge.ts`: React hook that manages the two-way postMessage protocol between the host board and all active plugin iframes. Validates `event.origin` against each plugin's registered `connectorUrl` — messages from unknown origins are silently ignored. Handles `PLUGIN_READY` (registers capabilities), `DATA_GET` / `DATA_SET` (proxies to server `/api/v1/plugins/data`), and `RESOLVE_CAPABILITY_RESPONSE`. Exposes `PluginBridgeContext` so deep components can call `bridge.resolve(capability, ctx)` without prop-drilling.

- `src/extensions/Plugins/iframeHost/PluginIframeHost.tsx`: Renders a single hidden `<iframe>` for one `BoardPlugin`. Sets `src` to `connectorUrl` with `boardId`, `pluginId`, and `origin` query params. Uses `sandbox="allow-scripts allow-same-origin allow-popups allow-forms"` and `display:none`.

- `src/extensions/Plugins/iframeHost/PluginIframeContainer.tsx`: Mounted once per board view. Fetches active board plugins on mount, renders a `PluginIframeHost` per plugin, and wraps children in `PluginBridgeContext.Provider`. Adding/removing a plugin causes the iframe list to re-render within one cycle.
