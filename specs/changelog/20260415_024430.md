# Changelog — 20260415_024430

## Update
- Migration number adjusted from `0022_webhooks.ts` (spec reference) to `0106_webhooks.ts` to avoid conflict with existing `0022_plugin_data_board_id.ts`.
- All ID columns use `string` type (varchar) to match the project-wide convention; the spec's `uuid` column type is incompatible with the FK targets (`workspaces.id`, `users.id`).

## New
- `db/migrations/0106_webhooks.ts`: Adds `webhooks` and `webhook_deliveries` tables. `webhooks` stores registration data (workspace scope, endpoint URL, signing secret, jsonb event_types array, active flag). `webhook_deliveries` logs every dispatch attempt with status, HTTP response, and delivery timestamp. Both tables use `gen_random_uuid()` for primary keys and cascade-delete appropriately.
- `server/extensions/webhooks/common/eventTypes.ts`: Exports `WEBHOOK_EVENT_TYPES` const array (22 event type strings) and the derived `WebhookEventType` union type. Serves as the single source of truth for event type validation across DB constraints and API guards.
