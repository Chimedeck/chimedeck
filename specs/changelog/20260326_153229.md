# Changelog — 2026-03-26 15:32

## Update
- Migrated all hardcoded Tailwind colour classes to semantic token classes across 20 `.tsx` files in 6 extension directories: `Activity`, `AdminInvite`, `ApiToken`, `CustomFields`, `List`, and `Mention`.
- Replaced bare styled `<button>` elements with the `<Button>` component from `src/common/components/Button.tsx`, using appropriate variants (`primary`, `secondary`, `ghost`, `danger`).
- Removed all `dark:bg-*`, `dark:text-*`, and `dark:border-*` prefixed classes throughout.

## New
_(none)_

## Technical Debt
- `bg-gray-100`, `text-gray-700/800/900`, `border-gray-300`, `hover:bg-gray-100`, `hover:bg-gray-200` in the `List` components were intentionally left as-is — these light-mode surface colours are not covered by the semantic token mappings and would require a design decision to map.
- The "Add a list" trigger button and inline form in `AddListForm`/`AddListButton` retain light-mode `bg-white/40` / `bg-gray-100` backgrounds for the same reason.

## What Should Be Done Next
- Extend the semantic token set with light-surface aliases (e.g. `bg-bg-canvas`, `bg-bg-elevated`) to cover `bg-white`, `bg-gray-100`, and other light-mode surfaces currently left as raw Tailwind classes.
- Audit remaining extensions (`Board`, `Card`, `Label`, etc.) for the same hardcoded-colour pattern and migrate them in a follow-up iteration.
