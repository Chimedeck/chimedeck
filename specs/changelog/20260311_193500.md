# Changelog — 2026-03-11T19:35:00Z

## Update
- `server/extensions/automation/engine/actions/index.ts`: Registered 4 new list action handlers in the action registry; added exports for all list handlers.
- `server/extensions/automation/api/index.ts`: Wired `GET /api/v1/automation/action-types` endpoint into the automation API router.

## New
- `server/extensions/automation/engine/actions/list/sortByDueDate.ts`: `list.sort_by_due_date` action — sorts all non-archived cards in a list by due date ascending (cards without due dates sorted last).
- `server/extensions/automation/engine/actions/list/sortByName.ts`: `list.sort_by_name` action — sorts all non-archived cards in a list alphabetically by title.
- `server/extensions/automation/engine/actions/list/archiveAllCards.ts`: `list.archive_all_cards` action — archives all non-archived cards in a list atomically.
- `server/extensions/automation/engine/actions/list/moveAllCards.ts`: `list.move_all_cards` action — moves all non-archived cards from one list to another, appending to the bottom of the target list in original order.
- `server/extensions/automation/api/actionTypes.ts`: `GET /api/v1/automation/action-types` handler — returns all registered action types with `type`, `label`, `category`, and JSON Schema `configSchema` for dynamic UI rendering.
- `tests/integration/automation/actions.test.ts`: Integration tests covering action registry completeness (18 total), list action config schema validation, and the action-types API endpoint.
- `tests/integration/automation/variables.test.ts`: Unit tests for `substituteVariables()` covering all 6 variables, unknown variable preservation, and multi-variable substitution.
