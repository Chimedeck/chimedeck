# Changelog ‚Äî Sprint 30: Card Money DB & API

## Update
- `server/extensions/card/api/update.ts`: Added `amount` and `currency` field handling to `PATCH /api/v1/cards/:id`. Validates non-negative numeric amounts and 3-letter ISO 4217 currency codes. Defaults currency to `USD` when amount is set without a currency. Emits `card.money.updated` activity event and `card_updated` WebSocket event on change.
- `server/extensions/board/api/get.ts`: Added `c.amount` and `c.currency` to the explicit SELECT columns for cards in the board detail response.

## New
- `db/migrations/0019_card_money.ts`: Adds `amount` (decimal 15,4) and `currency` (varchar 3) nullable columns to the `cards` table.
- `server/extensions/card/api/patch.ts`: Created as a standalone money PATCH handler (not currently wired to router ‚Äî money fields are handled via the existing `update.ts` PATCH endpoint instead).

## Technical Debt
- `patch.ts` was created as a standalone file but is not registered in the router. If a separate money endpoint is needed in the future, it can be wired up at `PATCH /api/v1/cards/:id/money`.
