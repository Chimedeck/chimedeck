# Sprint 14 — Observability, Rate Limiting & Security Hardening

## Update

- `server/mods/helmet.ts` — Extended with `Permissions-Policy` header and more specific CSP (`style-src`, `img-src`, `connect-src`, `frame-ancestors`).
- `server/extensions/activity/mods/write.ts` — Added `ipAddress` and `userAgent` optional fields to `WriteActivityInput` and `WrittenActivity`; these are stored for audit purposes.

## New

- `server/config/env.ts` — Added `OTEL_ENABLED`, `OTEL_EXPORTER_URL`, and `RATE_LIMIT_ENABLED` environment variables.
- `server/mods/observability/tracer.ts` — OTEL TracerProvider initialised when `OTEL_ENABLED=true`; falls back to no-op tracer otherwise.
- `server/mods/observability/metrics.ts` — OTEL MeterProvider with `mutation_latency_ms`, `sync_delay_ms`, `http_error_total`, `conflict_total`, and `ws_disconnect_total`; no-op when `OTEL_ENABLED=false`.
- `server/mods/observability/exporters.ts` — OTLP HTTP exporter config helper driven by `OTEL_EXPORTER_URL`.
- `server/middlewares/rateLimiter.ts` — Redis sliding-window rate limiter with Lua script; per-route-class limits (auth 10/min, mutations 120/min, reads 600/min, uploads 20/min); returns 429 + `Retry-After` on exceeded; degrades gracefully when Redis is unavailable.
- `db/migrations/0013_activity_audit.ts` — Adds nullable `ip_address` and `user_agent` columns to the `activities` table.
- `src/extensions/Board/components/CardSkeleton.tsx` — Shimmer placeholder for a single card.
- `src/extensions/Board/components/BoardSkeleton.tsx` — Shimmer placeholder for full board (lists + cards) shown while `isFetching`.
- `src/extensions/Board/components/VirtualCardList.tsx` — Virtualised card column via `@tanstack/react-virtual`; direct render when ≤ 100 cards, virtualised when > 100.
- `tests/load/board-load.js` — k6 load test: 50 VUs, board GET < 2 s at p95.
- `tests/load/drag-latency.js` — k6 load test: 20 VUs, card move round-trip < 500 ms at p95.
- `tests/load/ws-broadcast.js` — k6 load test: 100 VUs, WS fan-out < 500 ms at p95.
- `tests/integration/securityHeaders.test.ts` — Asserts all required security headers are present.
- `tests/integration/rateLimiter.test.ts` — Tests key generation, graceful degradation, and limit logic.
- `tests/integration/otel.test.ts` — Tests no-op behaviour when `OTEL_ENABLED=false`.
- `.github/workflows/ci.yml` — Added `bun audit --audit-level=high` step (non-blocking with `|| true`).
- `package.json` — Added `test:load` script.

## Technical Debt

- `@opentelemetry/sdk-trace-node`, `@opentelemetry/api`, `@opentelemetry/exporter-trace-otlp-http`, `@opentelemetry/sdk-metrics`, `@opentelemetry/exporter-metrics-otlp-http` are loaded via dynamic `import()` so they must be installed separately (`bun add @opentelemetry/...`) before enabling `OTEL_ENABLED=true`.
- `@tanstack/react-virtual` must be installed (`bun add @tanstack/react-virtual`) before the `VirtualCardList` component renders properly for > 100 cards.
- The `bun audit` CI step uses `|| true` to avoid blocking CI until the project has a confirmed clean bill of health.
