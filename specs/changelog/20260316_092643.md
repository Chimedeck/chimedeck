# Changelog — 2026-03-16 09:26

## Update
- `src/extensions/Card/components/CardDescription.tsx` — removed the explicit "Edit" / "Preview" toggle button. View mode now renders markdown (or a placeholder) with a text cursor on hover; clicking anywhere enters inline edit mode.
- `src/common/components/MentionInput/MentionInput.tsx` — added optional `onKeyDown` and `autoFocus` props so callers can compose keyboard shortcuts without losing mention-autocomplete key handling.

## New
- Edit mode shows a `<MentionInput>` textarea with `autoFocus`, a **Save** button, and a **Cancel** button below it.
- `Ctrl+Enter` / `Cmd+Enter` saves and exits edit mode; `Escape` cancels and restores the previous value.
- View mode element carries `role="button"` + `tabIndex=0` so keyboard users can activate it with `Enter` or `Space`.
- Empty description shows the placeholder "Add a more detailed description…" as a clickable muted text region.
- `tests/e2e/card-description-inline-edit.md` — new Playwright test spec covering edit mode entry, Ctrl+Enter save, Escape cancel, Save/Cancel buttons, placeholder behaviour, and keyboard accessibility.

## Technical Debt
- The debounced auto-save from the previous implementation has been removed; saves are now explicit (button or keyboard shortcut). If auto-save is needed in the future, it can be reintroduced as a debounced side-effect inside `handleSave`.
