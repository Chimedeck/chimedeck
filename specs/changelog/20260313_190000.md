## New
- `src/extensions/Board/components/BoardSettings/VisibilitySelector.tsx` — presentational radio-group component for selecting board visibility (PRIVATE / WORKSPACE / PUBLIC). Identical in appearance to the container version but lives in `components/` for reuse by future panels.
- `src/extensions/Board/components/BoardSettings/index.tsx` — role-aware Board Settings panel component. Uses `useGetBoardMembersQuery` from `boardMembersSlice` to derive the current user's board role and conditionally renders the `VisibilitySelector` only for ADMIN/OWNER users. Non-admin users see the panel without the Visibility section. Default fallback visibility is `WORKSPACE`.
- `src/extensions/Board/slices/boardMembersSlice.ts` — RTK Query slice (`boardMembersApi`) with four endpoints:
  - `getBoardMembers` — `GET /api/v1/boards/:boardId/members`
  - `addBoardMember` — `POST /api/v1/boards/:boardId/members`
  - `updateBoardMember` — `PATCH /api/v1/boards/:boardId/members/:userId`
  - `removeBoardMember` — `DELETE /api/v1/boards/:boardId/members/:userId`
  All endpoints tag by `boardId` for cache invalidation.
- `specs/tests/sprint-79-iteration-4-visibility-selector.md` — Playwright MCP test walkthrough covering: selector visible to admin, admin can change visibility, selector hidden for non-admin member, API error rollback.
