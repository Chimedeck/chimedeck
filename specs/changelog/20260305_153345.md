# Sprint 38 §2–4: Plugin Data Board Isolation & Resource Validation

## Update
- Required `boardId` in plugin-data GET (query) and SET (body) endpoints; requests without it now return 400 errors.
- All plugin-data GET/SET requests now validate that the resourceId (card/list) belongs to the specified boardId, returning 403 on mismatch.
- Database queries for plugin-data now include `board_id` in WHERE clauses for both retrieval and upsert logic.

## New
- Added `server/extensions/plugins/common/validateResourceBelongsToBoard.ts`: a helper that throws on resource-board mismatches for cards/lists, enforcing strict board isolation for plugin data.
