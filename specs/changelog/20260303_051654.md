# Sprint 24 — Profile Settings: Avatar & Nickname

## Update

- `server/extensions/users/api/index.ts` — router updated to delegate `GET /api/v1/users/me` and `PATCH /api/v1/users/me` to new profile handlers, and adds `POST /api/v1/users/me/avatar` and `DELETE /api/v1/users/me/avatar` routes.
- `src/extensions/Workspace/components/Sidebar.tsx` — user menu now shows the profile avatar (or initials fallback) and nickname/name, plus a "Settings" link to `/settings/profile`.
- `src/routing/index.tsx` — added lazy-loaded route `/settings/profile` inside the private `AppShell` wrapper.
- `src/store/index.ts` — registered `profile` reducer.
- `src/reducers.ts` — exported `profileDuckReducer`.

## New

- `db/migrations/0015_user_profile.ts` — adds `nickname TEXT UNIQUE` column to users table. (`avatar_url` was already present from migration 0002.)
- `server/mods/imageProcessor/index.ts` — `resizeAvatar()` using `sharp` (256×256 cover crop); graceful fallback when sharp unavailable; `isValidAvatarFile()` and `avatarExtension()` helpers.
- `server/extensions/users/api/profile/get.ts` — `GET /api/v1/users/me` returning full profile including `nickname` and `email_verified`.
- `server/extensions/users/api/profile/update.ts` — `PATCH /api/v1/users/me` with nickname uniqueness check (409 `nickname-taken`) and name length validation.
- `server/extensions/users/api/avatar/upload.ts` — `POST /api/v1/users/me/avatar` multipart upload; validates mime/size; resizes with imageProcessor; stores in S3 at `avatars/<userId>.<ext>`; deletes old avatar.
- `server/extensions/users/api/avatar/remove.ts` — `DELETE /api/v1/users/me/avatar` removes file from S3 and nullifies `avatar_url` in DB; returns 204.
- `src/extensions/User/api/user.ts` — typed API client wrappers for all four profile endpoints.
- `src/extensions/User/containers/ProfilePage/ProfilePage.duck.ts` — Redux slice with `fetchProfileThunk`, `updateProfileThunk`, `uploadAvatarThunk`, `removeAvatarThunk`; selectors for profile, status, avatarUploading, error.
- `src/extensions/User/containers/ProfilePage/ProfilePage.tsx` — profile settings page: avatar section + profile form, fetches on mount.
- `src/extensions/User/components/AvatarUploader.tsx` — click/drag-drop avatar widget with progress overlay, remove button, client-side type/size validation.
- `src/extensions/User/components/NicknameField.tsx` — `@`-prefixed nickname input with format validation and inline error display.
- `src/extensions/User/components/ProfileForm.tsx` — display name + nickname form with save/saved feedback and server error propagation.
- `src/extensions/User/translations/en.json` — English strings for profile UI.

## Technical Debt

- Workspace unit tests (`WorkspaceModal.test.tsx`, `WorkspaceShell.test.tsx`) use partial Redux state mocks that were already missing several required keys before this sprint; adding `profile` increases the count by one. These tests need their mock state objects updated to include all reducers.
- The `me.ts` file is now superseded by `profile/get.ts` and `profile/update.ts` but remains in place. It should be removed in a follow-up cleanup.
