# Changelog — 20260311_120748

## Update

- Updated `server/extensions/automation/common/types.ts`: Added `label: string` and `configSchema: ZodTypeAny` fields to the `TriggerHandler` interface.
- Updated `server/extensions/automation/engine/registry.ts`: Added `getAllTriggerHandlers()` function that returns all registered handler instances (needed for trigger-types discovery API in iteration 11).
- Updated `server/index.ts`: Added side-effect import of `server/extensions/automation/engine/triggers/index` to register all trigger handlers at startup.

## New

- **`server/extensions/automation/engine/triggers/card/`** — 13 card trigger modules, each exporting a `TriggerHandler` with `type`, `label`, `configSchema` (Zod), and `matches()`:
  - `created.ts` — `card.created` (optional `listId` filter)
  - `movedToList.ts` — `card.moved_to_list` (required `listId` = `toListId`)
  - `movedFromList.ts` — `card.moved_from_list` (required `listId` = `fromListId`)
  - `labelAdded.ts` — `card.label_added` (optional `labelId` filter)
  - `labelRemoved.ts` — `card.label_removed` (optional `labelId` filter)
  - `memberAdded.ts` — `card.member_added` (optional `memberId` filter)
  - `memberRemoved.ts` — `card.member_removed` (optional `memberId` filter)
  - `dueDateSet.ts` — `card.due_date_set` (no config)
  - `dueDateRemoved.ts` — `card.due_date_removed` (no config)
  - `checklistCompleted.ts` — `card.checklist_completed` (optional `checklistId` filter)
  - `allChecklistsCompleted.ts` — `card.all_checklists_completed` (no config)
  - `archived.ts` — `card.archived` (no config)
  - `commentAdded.ts` — `card.comment_added` (no config)

- **`server/extensions/automation/engine/triggers/validate.ts`** — `validateTrigger(triggerType, config)` returning `{ valid, errorName?, errorData? }`. Returns `trigger-type-unknown` (422) for unknown types and `trigger-config-invalid` (422) for Zod validation failures.

- **`server/extensions/automation/engine/triggers/index.ts`** — Registers all 13 card triggers with the engine registry and re-exports them for use in tests and the trigger-types API.

- **`tests/integration/automation/triggers.test.ts`** — 44 tests covering registry population, `validateTrigger`, and positive/negative `matches()` cases for all 13 trigger types.

- Installed `zod@4.3.6` as a project dependency.
