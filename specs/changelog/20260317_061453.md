# Changelog — 2026-03-17T06:14:53Z
## Sprint 83 — Iteration 12: Cross-device continuity, replay conflict handling, and final QA

### Update
- **`src/extensions/OfflineDrafts/reconcile.ts`**: Added `loserPendingIntent` field to `ReconcileResult`. When the losing side of a last-write-wins reconcile had a `save_pending` or `submit_pending` intent, the field is set so callers can surface a conflict warning. Added private `isPendingIntent()` helper.
- **`src/extensions/OfflineDrafts/hooks/useOfflineDescriptionDraft.ts`**: Added `isSavePending` state and exposed it in the hook result. Restore logic now checks `loserPendingIntent` — if the server wins and local had `save_pending`, status is set to `sync_failed` with `isSavePending=true` (shows "Save failed / Retry Save" UI). `retrySync` re-enqueues the PATCH when `isSavePending=true` instead of retrying the background PUT. `clearDraft`, `discardDraft`, and the restore reset all clear `isSavePending`.
- **`src/extensions/OfflineDrafts/hooks/useOfflineCommentDraft.ts`**: Added `isSubmitPending` state and exposed it in the hook result. Parallel changes to description hook: restore handles `loserPendingIntent === 'submit_pending'`, `retrySync` re-enqueues the POST, `clearDraft`/`discardDraft` clear `isSubmitPending`.
- **`src/extensions/Card/components/CardDescriptionTiptap.tsx`**: Consumes `isSavePending` from hook. Footer now shows `Save failed` + `Retry Save` when `isSavePending && sync_failed`, and `Will save when back online` when `isSavePending && will_sync_when_online`. Generic `Sync failed` + `Retry` otherwise.
- **`src/extensions/Comment/components/CommentEditor.tsx`**: Consumes `isSubmitPending` from hook. Footer now shows `Post failed` + `Retry Post` when `isSubmitPending && sync_failed`, and `Will post when back online` when `isSubmitPending && will_sync_when_online`. Generic `Sync failed` + `Retry` otherwise.

### New
- **`specs/tests/offline-replay-save-and-comment.md`**: Added Part 4 (Scenarios 17–20, cross-device draft continuity) and Part 5 (Scenarios 21–27, replay conflict UI) covering: cross-device merge on card open, server/local win cases, tie-break, save conflict "Save failed / Retry Save / Discard" flow, post conflict "Post failed / Retry Post / Discard" flow, and generic non-pending sync failure.
- **`specs/tests/offline-comment-draft-recovery.md`**: Added Part 8 (Scenarios 20–25) covering cross-device comment draft sync, offline device wins over stale server, submit_pending conflict UI with Retry Post, Discard flow, login/logout survival of pending drafts.

### Technical Debt
- `messageQueue` does not yet notify individual hooks when their specific queued mutation fails (only the global `onMutationConflict` is called). The current implementation sets `sync_failed` from the reconcile result at card-open time. A future improvement would listen for per-mutation replay failures in real time.
