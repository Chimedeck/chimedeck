# Changelog — 2026-04-10T14:35:00Z

## Sprint 130 — Comment Emoji Reactions UI

### Update
- Extended `CommentData` interface in `src/extensions/Card/api/cardDetail.ts` to include `reactions?: ReactionSummary[]`.
- Extended `Comment` interface in `CommentItem.tsx` with `reactions?: ReactionSummary[]`.
- Modified `CommentItem.tsx` to accept `onAddReaction`/`onRemoveReaction` props and render `CommentReactions` below the comment bubble.
- Modified `CommentThread.tsx` to accept and pass through reaction props to `CommentItem`.
- Modified `ActivityFeed.tsx` (Card container) to accept and pass reaction callbacks to `CommentItem`.
- Modified `CardModal.tsx` to accept and forward `onAddReaction`/`onRemoveReaction` props.
- Modified `CardModal/index.tsx` container to create `handleAddReaction` and `handleRemoveReaction` with optimistic updates and rollback.
- Added `addReaction`/`removeReaction` Redux actions to `cardDetailSlice.ts` with full optimistic update logic.
- Added `comment_reaction_added` and `comment_reaction_removed` WS event handlers in `useBoardSync.ts`.

### New
- `src/extensions/Comment/api.ts` — `addReaction` and `removeReaction` API helpers wrapping `POST /api/v1/comments/:id/reactions` and `DELETE /api/v1/comments/:id/reactions/:emoji`.
- `src/extensions/Comment/components/CommentReactions.tsx` — pill row component displaying emoji reactions with active/inactive styling and an add-reaction trigger button.
- `src/extensions/Comment/components/EmojiPickerPopover.tsx` — popover wrapping `@emoji-mart/react` Picker, positioned below-left of the trigger, closes on outside click or Escape key.
- Translation keys: `comment.reactions.add` and `comment.reactions.aria.pill`.

### Technical Debt
- The `CommentData` interface in `cardDetail.ts` and the `Comment` interface in `CommentItem.tsx` are separate and partially duplicated; they should be unified in a future refactor.
- Emoji picker theme is hardcoded to `"dark"` — should respect `prefers-color-scheme` via a CSS variable in a future iteration.
