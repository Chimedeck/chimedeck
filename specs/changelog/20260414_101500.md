# Update
- Fixed CLI comment creation payload in `cli/commands/comment.ts` to send `content` instead of `text` to `/api/v1/cards/:cardId/comments`.
- Fixed CLI move-card payload in `cli/commands/moveCard.ts` to send `targetListId` and `afterCardId` expected by `/api/v1/cards/:cardId/move`.
- Added backward compatibility for deprecated CLI `--position` by supporting only `0` (maps to top insertion) and returning a clear error for other values.
- Fixed MCP `write_comment` payload in `server/extensions/mcp/tools/writeComment.ts` to send `content` and accept both `content` and legacy `text` inputs.
- Fixed MCP `move_card` payload in `server/extensions/mcp/tools/moveCard.ts` to send `targetListId` and `afterCardId` and handle deprecated `position=0`.
- Normalized search argument naming by preferring `query` in CLI and MCP search tools, while maintaining compatibility for legacy `q`.
- Updated server search handlers to accept both `query` and legacy `q` query parameters.
- Updated user-facing docs in `cli/README.md` and `server/extensions/mcp/README.md` for the new parameter contracts.

# New
- Added MCP input compatibility aliases:
  - `write_comment`: `content` (preferred), `text` (legacy)
  - `search_cards`, `search_board`: `query` (preferred), `q` (legacy)
  - `move_card`: `afterCardId` (preferred), `position` (legacy, limited)

# Technical Debt
- MCP tool registration still uses the deprecated `server.tool(...)` API across this extension; migration to `registerTool` is pending.
- CLI positional move semantics remain intentionally constrained because server move ordering is based on `afterCardId` rather than absolute index.
