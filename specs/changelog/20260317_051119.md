# Changelog — 2026-03-17T05:11:19Z

## Update
- `useAttachmentUpload` — `onSuccess` callback now receives the full `Attachment` object (payload) instead of just the attachment id string.
- `OneLineToolbar` — new optional `onAttach?: () => void` prop; when provided a paperclip button appears in the toolbar between the primary format controls and the `+` overflow button.
- `CardDescriptionTiptap` — new optional `cardId?: string` prop; when present the toolbar shows a paperclip button and inline upload preview rows (with image thumbnail or file-type icon, progress bar, cancel ×, and error state) are rendered below the editor content.
- `CommentEditor` — same as CardDescriptionTiptap: new `cardId?: string` prop, paperclip toolbar button, and inline upload preview rows inside the editor container.

## New
- `src/extensions/Attachments/components/InlineUploadPreview.tsx` — reusable compact upload-row component used by both editors. Renders object-URL thumbnail for image files and a file-type icon for non-images. Shows `UploadProgressBar` while in-flight, a green "Uploaded" caption on completion (auto-dismisses after 2 s), and a red error message (with `role="alert"`) on failure. The × button cancels or dismisses.

## Technical Debt
- `cardId` is not yet wired from `CardModal` or `ActivityFeed` to the editors; until iteration 5 the attachment button will not appear in production (prop is absent → toolbar button hidden by design).

## What Should Be Done Next
