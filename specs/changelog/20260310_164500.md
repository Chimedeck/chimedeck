# Changelog — 20260310_164500

## Update

- `src/extensions/Board/containers/BoardSettings/BoardSettings.tsx` — added a **Custom Fields** section above the Plugins button, rendering `BoardCustomFieldsPanel` inside the board settings slide-in panel.

## New

- `src/extensions/CustomFields/types.ts` — TypeScript types for custom field definitions (`CustomField`, `DropdownOption`, `FieldType`) and card values (`CustomFieldValue`, `CardCustomFieldValueInclude`) with payload types for API calls. Uses `exactOptionalPropertyTypes`-safe optional fields.

- `src/extensions/CustomFields/api.ts` — plain async API functions (`listCustomFields`, `createCustomField`, `updateCustomField`, `deleteCustomField`, `upsertCardFieldValue`, `deleteCardFieldValue`) following the established `apiClient`-based pattern. Includes a `useCustomFields(boardId)` React hook for components that need simple read access without a Redux duck.

- `src/extensions/CustomFields/DropdownFieldEditor.tsx` — colour-coded dropdown option editor. Renders a list of option label inputs with swatch-based colour pickers (8 preset colours) and "Add option" / remove buttons.

- `src/extensions/CustomFields/BoardCustomFieldsPanel.tsx` — board settings panel section for managing custom field definitions. Supports: create (all field types), inline rename (click-to-edit), toggle "show on card tile", expand/collapse dropdown option editor, and delete with confirmation dialog. Uses `useCustomFields` hook.

- `tests/e2e/custom-fields-board-panel.md` — Playwright MCP markdown E2E test covering: open board settings panel, verify custom fields section, create TEXT field, create DROPDOWN field with options, expand options editor, rename a field, toggle show_on_card, delete a field, close the panel.

## Technical Debt

- The project does not use RTK Query despite the sprint spec mentioning it. The existing architecture uses plain async thunks + `apiClient`. The `api.ts` hooks use `useState`/`useEffect` instead. If RTK Query is added to the store in a future iteration, this module should be migrated.
- `window.confirm` is used for delete confirmation. A proper modal dialog should replace this in the future.
