# Changelog — 20260310_101542

## Sprint 55 — Custom Fields Client UI Part 2

### Update
- `server/extensions/customFields/api/cardValues.ts`: Added `GET /api/v1/cards/:cardId/custom-field-values` endpoint (`handleListCardFieldValues`) that returns all custom field values for a card. Refactored `buildValuePayload` to accept the client payload format (`value_text`, `value_number`, `value_date`, `value_checkbox`, `value_option_id`) instead of a single `value` key, fixing the client-server contract mismatch. Backward-compatible — still accepts `value` as a legacy fallback.
- `server/extensions/customFields/api/index.ts`: Registered the new list-all-values route (`GET /cards/:cardId/custom-field-values`).
- `src/extensions/CustomFields/api.ts`: Added `useCardCustomFieldValues` hook that fetches all values for a given card via the new list endpoint. Added re-export of `CustomField` and `CustomFieldValue` types.
- `src/extensions/Card/components/CardTile.tsx`: Extended with optional `customFields` and `customFieldValues` props; renders `CustomFieldBadge` for each field with `show_on_card=true` and a non-null value.
- `src/extensions/Card/components/CardItem.tsx`: Added optional `boardId` prop; when present, renders `CardCustomFieldBadges` component below plugin badges.
- `src/extensions/List/containers/BoardPage/ListColumn.tsx`: Added `boardId` prop and passes it through to `CardItem`.
- `src/extensions/Board/components/BoardCanvas.tsx`: Passes `boardId` prop to `SortableListColumn`.

### New
- `src/extensions/CustomFields/CustomFieldValueEditor.tsx`: Type-aware input component for editing a custom field value. Handles TEXT (text input with blur-to-save), NUMBER (number input with blur-to-save), DATE (date picker), CHECKBOX (toggle), and DROPDOWN (select with colour swatch). Each type calls `upsertCardFieldValue` on change and `deleteCardFieldValue` on clear.
- `src/extensions/CustomFields/CustomFieldBadge.tsx`: Compact badge component for displaying a custom field value on card tiles. Formats values per type (date → locale string, checkbox → ✓/✗, dropdown → option label with colour swatch). Renders nothing for empty values.
- `src/extensions/CustomFields/CustomFieldsSection.tsx`: Self-contained section rendered inside the card modal. Fetches field definitions (`useCustomFields`) and card values (`useCardCustomFieldValues`) independently. Renders `CustomFieldValueEditor` for each field. Updates local state on save without Redux round-trip.
- `src/extensions/Card/components/CardCustomFieldBadges.tsx`: Lightweight component that reads field definitions and card values from hooks, renders `CustomFieldBadge` for fields with `show_on_card=true` and a non-null value.
- `src/extensions/CustomFields/index.ts`: Public entry point exporting all components (`BoardCustomFieldsPanel`, `DropdownFieldEditor`, `CustomFieldValueEditor`, `CustomFieldBadge`, `CustomFieldsSection`) and all types/hooks.
- `tests/e2e/custom-fields-ui.md`: Playwright MCP markdown tests covering editing all 5 field types in the card modal, verifying badge appearance on tiles, clearing values, and `show_on_card=false` behaviour.

### Technical Debt
- `CardCustomFieldBadges` makes N API calls (one per card visible on the board) when `boardId` is present. A future optimisation: include a `customFieldValueSummary` in the card list API response to avoid per-card fetches in the board view.
- `useCardCustomFieldValues` refetches on every card mount. Consider a board-scoped cache keyed by `cardId` to deduplicate concurrent requests for the same card.
