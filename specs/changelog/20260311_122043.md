# Changelog — 2026-03-11T12:20:43Z

## Update
- `server/extensions/automation/common/types.ts`: Added `label?`, `category?`, and `configSchema` fields to `ActionHandler` interface to support action-types discovery endpoint.
- `server/extensions/automation/engine/registry.ts`: Added `getAllActionHandlers()` export to mirror `getAllTriggerHandlers()`.
- `server/extensions/automation/api/index.ts`: Added side-effect import of `engine/actions/index` so all action handlers are registered at server startup.

## New
- `server/extensions/automation/engine/actions/variables.ts`: Variable substitution engine for `{cardName}`, `{boardName}`, `{listName}`, `{date}`, `{dueDate}`, `{triggerMember}` in action text fields. Unknown variables are left intact.
- `server/extensions/automation/engine/actions/card/moveToList.ts`: `card.move_to_list` — moves card to target list at top/bottom position.
- `server/extensions/automation/engine/actions/card/moveToTop.ts`: `card.move_to_top` — moves card to position 0 in current list.
- `server/extensions/automation/engine/actions/card/moveToBottom.ts`: `card.move_to_bottom` — moves card to last position in current list.
- `server/extensions/automation/engine/actions/card/addLabel.ts`: `card.add_label` — idempotently attaches a label.
- `server/extensions/automation/engine/actions/card/removeLabel.ts`: `card.remove_label` — idempotently detaches a label.
- `server/extensions/automation/engine/actions/card/addMember.ts`: `card.add_member` — idempotently assigns a member.
- `server/extensions/automation/engine/actions/card/removeMember.ts`: `card.remove_member` — idempotently unassigns a member.
- `server/extensions/automation/engine/actions/card/setDueDate.ts`: `card.set_due_date` — sets due date via absolute ISO string or relative offset in days.
- `server/extensions/automation/engine/actions/card/removeDueDate.ts`: `card.remove_due_date` — clears due date and resets due_complete flag.
- `server/extensions/automation/engine/actions/card/markDueComplete.ts`: `card.mark_due_complete` — marks due date as complete; errors if no due date set.
- `server/extensions/automation/engine/actions/card/addComment.ts`: `card.add_comment` — posts a comment with variable substitution support.
- `server/extensions/automation/engine/actions/card/archive.ts`: `card.archive` — archives the card.
- `server/extensions/automation/engine/actions/card/addChecklist.ts`: `card.add_checklist` — appends checklist items to a card (TODO: update when checklists table is added).
- `server/extensions/automation/engine/actions/card/mentionMembers.ts`: `card.mention_members` — posts a comment @-mentioning specified members.
- `server/extensions/automation/engine/actions/index.ts`: Registers all 14 card action handlers in the ACTION_REGISTRY.

## Technical Debt
- `card.add_checklist` stores items directly in `checklist_items` (flat table) since no `checklists` group table exists yet. The checklist name is not persisted as a group entity. Update when a checklists table is introduced.
