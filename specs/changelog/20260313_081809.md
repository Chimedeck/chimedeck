## Update
- Extended `POST /api/v1/admin/users` to accept an optional `autoVerifyEmail` boolean in the request body.
- When `autoVerifyEmail: true`, sets `email_verified` and `email_verified_at = NOW()` at insert time; no verification token or verification email is generated.
- When `autoVerifyEmail: false` or omitted (default), `email_verified_at` is `null` — existing behaviour is preserved.
- Response `data` now always includes `email_verified_at` (ISO string or `null`).

## New
- `db/migrations/0039_email_verified_at.ts` — adds `email_verified_at` timestamp column to `users`; backfills existing verified users.
- `server/extensions/auth/types.ts` — new file exporting `AdminCreateUserBody` interface (includes `autoVerifyEmail?: boolean`).
- `specs/tests/admin-auto-verify-email-api.md` — Playwright MCP test spec covering all four `autoVerifyEmail` scenarios.
- Added `autoVerifyEmail logic` unit-test group (3 tests) to `tests/integration/auth/adminCreateUser.test.ts`.

## Technical Debt
- `email_verified_at` backfill in migration uses `knex.fn.now()` as a proxy timestamp; real creation dates are not recoverable for pre-existing rows.
