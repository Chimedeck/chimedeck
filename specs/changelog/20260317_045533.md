# Changelog — 2026-03-17T04:55:33Z

## Update
- `src/extensions/Card/components/CardDescriptionTiptap.tsx`: Refactored to use the new shared `OneLineToolbar` component. Removed inline toolbar JSX, the `runToolbarCommand` helper, the overflow close effect, and the duplicate toolbar style constants. Behaviour is unchanged.

## New
- `src/extensions/Card/components/OneLineToolbar.tsx`: Extracted shared single-line rich-text toolbar for Tiptap editors. Accepts `editor`, `overflowOpen`, and `onToggleOverflow` props. Handles its own outside-click detection for the overflow menu. Renders Bold, Italic, Strikethrough, Bullet list as primary controls and Numbered list, Quote, Code block behind a **+** overflow dropdown. All buttons have `aria-label`/`title`; the toolbar container has `role="toolbar"`.
- `src/extensions/Comment/components/CommentEditor.tsx`: Replaced textarea/MentionInput with a Tiptap rich-text editor (StarterKit + Markdown extensions) plus the shared `OneLineToolbar`. Toolbar never wraps. Submit via button or Ctrl/Cmd+Enter; cancel via button or Escape. Content is read from `editor.getMarkdown()` on submit. @mention deferred to iteration 3 (+ overflow menu).
- `specs/tests/rich-text-toolbar-single-line-overflow.md`: Added five new Playwright MCP scenarios covering CommentEditor: single-line toolbar, overflow menu, primary controls, edit-comment flow, and keyboard navigation.

## Technical Debt
- CommentEditor previously supported @mention autocomplete via `MentionInput`. This is temporarily removed in this iteration and will be restored via the Tiptap Mention extension inside the + overflow menu (iteration 3).
