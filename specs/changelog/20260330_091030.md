## Update
- Fixed card reordering persistence for `PATCH /api/v1/cards/:id/move` when moving a card to index `0` in lists that already reached the lower fractional bound (e.g. first card at position `!`).
- Added strict position validation after `between(left, right)` so invalid non-between values are no longer written.

## New
- Added fallback rebalancing in move flow: when no valid lexicographic slot exists, the target list is re-spaced via `generatePositions` and the moved card is inserted at the requested index in a single transaction.
- Refactored move endpoint internals into helpers (`parseMoveBody`, `validateMoveLists`, `resolveInsertIndex`, `computeStrictPositionBetween`, `persistMove`) for clarity and lower complexity.

- Add focused unit/integration tests around fractional lower-bound handling to prevent regressions.
