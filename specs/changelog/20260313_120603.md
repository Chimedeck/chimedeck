# Update
- Switched API-facing avatar URL responses to use pre-signed S3 GET URLs when running against real AWS S3 (no custom S3 endpoint configured).
- Kept LocalStack/custom endpoint behavior unchanged to avoid breaking local/dev image rendering.
- Updated avatar key extraction logic used by avatar replace/remove flows so both path-style and virtual-hosted S3 URLs are handled safely.

# New
- Added shared helper module at server/common/avatar/resolveAvatarUrl.ts.
- Added utility extraction function for avatar S3 keys and a resolver that signs AWS S3 avatar URLs with a short TTL.

# Technical Debt
- Some SQL-aggregated payloads in board/card/notification APIs still return raw users.avatar_url values from DB rows and are not yet post-processed through the new resolver.
- Long-term, consider storing avatar S3 key in a dedicated column (e.g. avatar_s3_key) and generating URLs only at response time.
