# Changelog — 2026-03-19 10:11:50

## Update

- Migrated all hard-coded English UI strings in `src/extensions/TimelineView/` to `translations/en.json`.
- Replaced inline literals in `TimelineView.tsx`, `TimelineBar.tsx`, `TimelineZoomControl.tsx`, and `TimelineRow.tsx` with `translations['Key']` bracket notation.
- Added `specs/tests/sprint-93-timelineview-i18n.md` Playwright MCP test plan for TimelineView i18n verification.

## New

- `src/extensions/TimelineView/translations/en.json` — 13 translation keys covering: toolbar "Today" button, zoom level labels (Day/Week/Month/Quarter), zoom group aria-label, resize handle aria-labels, unscheduled section label, scheduled count suffix, empty state ("No cards with date ranges"), section title ("Timeline"), and "Add date range" action label.

## Technical Debt

- `TimelineView.zoomQuarter` and `TimelineView.addDateRange` keys are defined in `en.json` but not yet wired to rendered UI — Quarter zoom and the "Add date range" button are not yet implemented. Keys are pre-defined so future sprints can complete the feature without revisiting i18n.
- The `scheduledCount` key uses a split-string interpolation approach (`{n} {translations['key']}`). If locale word order differs from English (count-before-noun), a template-literal helper will be needed. Noted as future technical debt.
- `TimelineHeader.tsx` uses `MONTH_SHORT` and `DAY_SHORT` arrays (date abbreviations). These are calendar data rendered as part of date labels, not UI copy. Internationalising date formatting (Intl.DateTimeFormat) is deferred to a dedicated locale/date sprint.

## What Should Be Done Next

- Consider adding a locale-aware date rendering utility to replace the manual `MONTH_SHORT`/`DAY_SHORT` arrays across all view extensions (CalendarView, TimelineView, TableView).
- Wire `TimelineView.zoomQuarter` and `TimelineView.addDateRange` once those features are built.
