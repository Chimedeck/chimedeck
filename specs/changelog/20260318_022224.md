## Update
- Registered board-scoped search route `GET /api/v1/boards/:boardId/search` in the search API router.

## New
- `server/extensions/search/api/getBoardSearch.ts`: Handler for board-scoped search. Delegates access control to `applyBoardVisibility` and delegates query execution to `queryBoardSearch`.
- `server/extensions/search/mods/queryBoardSearch.ts`: Query module constrained strictly to a single board. Searches cards (non-archived) and lists, enforces default limit of 20 and max of 50, orders by ts_rank_cd relevance, and strips internal `rank` field from the response payload.
- `specs/tests/board-scoped-search.md`: Playwright MCP test scenarios covering auth required, board-not-found, access denied, query-too-short, query-invalid, board-only filtering, result shape, default/max limit, archived exclusion, stable ordering, empty query, and PUBLIC board access.

## Technical Debt
- `queryBoardSearch` does not yet support cursor-based pagination (matching the workspace search endpoint). Add cursor support when the board-search result volume grows large enough to require it.
