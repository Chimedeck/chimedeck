# Changelog — 2026-03-10T03:24:38Z

## Update
- `server/extensions/board/api/index.ts`: Registered three new board-level view routes:
  - `GET /api/v1/boards/:id/activity`
  - `GET /api/v1/boards/:id/comments`
  - `GET /api/v1/boards/:id/archived-cards`

## New
- `server/extensions/board/api/activity.ts`: Re-exports `handleBoardActivity` from the activity extension as `handleGetBoardActivity`, wiring the existing paginated board activity feed into the board router.
- `server/extensions/board/api/comments.ts`: `GET /api/v1/boards/:id/comments` — returns cursor-paginated comments across all cards in a board, with `author_name` and `card_title` includes. Default limit 50, max 100.
- `server/extensions/board/api/archived-cards.ts`: `GET /api/v1/boards/:id/archived-cards` — returns all archived cards for a board, with `list_title` include, ordered by `updated_at` desc.
- `specs/tests/board-views-api.test.md`: Playwright MCP test spec for all three new board-level view endpoints.

## Technical Debt
- `GET /api/v1/boards/:id/activity` was already accessible via the `activityRouter`. Now that it is also handled by the `boardRouter` (which is checked first), the route in `activityRouter` becomes a dead code path. A future cleanup should remove the duplicate route from `server/extensions/activity/api/index.ts`.
