# Comparison: Our Product vs Trello

This document highlights practical differences relevant for testing and product alignment.

| Feature / Area | Trello Behavior | Our Product Behavior | Difference (Same / Modified / Not Supported) | Notes / Reason (if different) |
|---|---|---:|---:|---|
| Boards | Boards are the primary container for lists/cards; support backgrounds, members, Power-Ups, and board settings. | Boards are core resources with lists/cards, settings, and plugin support. State is event-sourced and projections are rebuilt from the event stream. | Modified | Event-sourced model (auditability, replay) and deterministic projections — see [specs/architecture/event_sourcing.md](specs/architecture/event_sourcing.md). Real-time sync uses WebSocket + LWW conflict resolution: [specs/architecture/real_time_sync_protocol.md](specs/architecture/real_time_sync_protocol.md).
| Lists (columns) | Create, rename, reorder lists on a board; lists hold cards. | Same feature set: create, rename, reorder lists with fractional indexing. | Same | List ordering uses fractional index to compute positions (see sprint 07/08 specs). Fractional positions are validated server-side.
| Cards (core) | Cards support title, description, comments, attachments, labels, members, checklists, due dates, archive/copy/move. | Same core fields; cards stored with `position`, `due_date`, `archived` flags; detailed `GET /api/v1/cards/:id` includes labels, members, comments, attachments, activities. | Same | Card APIs and behaviors detailed in [specs/sprints/sprint-07.md](specs/sprints/sprint-07.md) and extended fields in [specs/sprints/sprint-08.md](specs/sprints/sprint-08.md).
| Card Detail / Modal (card back) | Rich card back with edit controls, attachments, checklist, members, labels and actions. Deep-linkable card URLs. | Card modal provides editable title/description (Markdown), checklist, labels, assignees, due date, actions menu. Modal is URL-driven (`?card=`) and supports optimistic updates. | Same | Modal behavior and URL sync are in [specs/sprints/sprint-19.md](specs/sprints/sprint-19.md).
| Drag & Drop | Drag cards between lists and reorder; keyboard alternatives exist. | Drag-and-drop implemented with `@dnd-kit`; keyboard alternatives provided (move via modal). Reordering uses fractional index. | Same | Keyboard accessibility and DnD details in sprint specs (Card Core & UI sprints).
| Comments & Activity | Threaded comments on cards; activity shows history of actions. | Threaded comments with versioning and soft-delete; immutable `Activity` audit log records all mutations and supports cursor pagination. | Modified | Comments are versioned and soft-deleted (`content: "[deleted]"`); activity table is append-only. See [specs/sprints/sprint-11.md](specs/sprints/sprint-11.md).
| Mentions (@user) | Autocomplete for members; triggers notifications. | Autocomplete restricted to board members; mentions saved inline and a `mentions` table is derived for notifications. | Same (narrower scope) | Mentions re-derived on save and only permit board members; see [specs/sprints/sprint-25.md](specs/sprints/sprint-25.md).
| Attachments (files & URLs) | Attach and preview files; integrate external services (Drive, Dropbox). | Supports file uploads via pre-signed S3 PUT, confirm step, virus scanning, signed read URLs; also supports external URLs with SSRF checks. | Modified | Secure upload flow with virus-scan and signed GET URLs; external integrations via plugin layer (no built-in Drive connector). See [specs/sprints/sprint-12.md](specs/sprints/sprint-12.md).
| Labels | Labels available to color-code cards (Trello: board-scoped labels). | Labels are workspace-scoped and reusable across boards; server enforces max 20 labels per card. | Modified | Workspace-scoped labels differ from Trello's board-scoped labels. See [specs/sprints/sprint-08.md](specs/sprints/sprint-08.md).
| Members & Assignments | Assign members to cards; add members to boards; guest invites supported. | Assign workspace members to cards; guest types and board guest permissions implemented (viewer/member). | Same (with role nuance) | Guests have `guestType` and client/server guards; see changelog and board members sprints (guests iteration). Example: [specs/changelog/20260318_075511.md](specs/changelog/20260318_075511.md).
| Checklists | Add checklist items, reorder, check/uncheck. | Same: checklist items stored with fractional position, progress shown on modal. | Same | Checklist model and APIs in [specs/sprints/sprint-08.md](specs/sprints/sprint-08.md) and Card Modal [specs/sprints/sprint-19.md](specs/sprints/sprint-19.md).
| Due Dates & Reminders | Set due dates and optional reminders; calendar view via Power-Up. | Cards have `due_date`; workspace-level due queries exist; full due-date reminder system is planned/partial. | Modified | `due_date` stored and queried; notification/alert system for due dates is noted as future work in sprint specs (partial). See [specs/sprints/sprint-08.md](specs/sprints/sprint-08.md).
| Notifications (in-app) | Real-time notifications and email digests for mentions/changes. | In-app notifications for mentions with personal WS channel, unread badge, panel, and APIs for mark-read/delete. | Same (narrower) | Notifications prototype focuses on mentions (sprint 26); email digests not documented. See [specs/sprints/sprint-26.md](specs/sprints/sprint-26.md).
| Search / Command Palette | Global search for cards/boards/members; Trello has search UI. | Command-palette (`⌘K`) with debounced search hitting `GET /api/v1/search` and quick navigation. | Same | Command-palette and search UI described in [specs/sprints/sprint-22.md](specs/sprints/sprint-22.md).
| Presence & Real-time Collaboration | Live cursors/presence and near-real-time updates across clients. | Real-time sync over WebSocket with presence, optimistic updates, and deterministic apply order. Polling fallback to HTTP events when WS unavailable. | Same (with explicit fallback) | Protocol and client behavior in [specs/architecture/real_time_sync_protocol.md](specs/architecture/real_time_sync_protocol.md) and polling fallback in [specs/sprints/sprint-51.md](specs/sprints/sprint-51.md).
| Offline / Polling Fallback | Trello relies on real-time; behavior varies by client. | Fallback polling `GET /api/v1/boards/:id/events?since=` every 5s after WS failures; UI indicates fallback. | Modified | Explicit polling fallback and UX indicator implemented. See [specs/sprints/sprint-51.md](specs/sprints/sprint-51.md).
| Automation (Butler-like) | Trello Butler provides broad triggers/actions, scheduled rules, card/board buttons. | Automation engine (RULE, CARD_BUTTON, BOARD_BUTTON, SCHEDULED, DUE_DATE) with DB schema, trigger/action registry and execution engine. | Modified (core present) | Core engine, CRUD and run logging implemented (sprints 61+). Full parity of Butler actions depends on implemented action handlers; the system is extensible. See [specs/sprints/sprint-61.md](specs/sprints/sprint-61.md).
| Plugins / Power-Ups | Trello Power-Ups (3rd-party) via iframe SDK, marketplace, and connectors. | Plugin system with SDK (`jhInstance`) and iframe connector model compatible with Power-Up shape; data layer and SDK exist, marketplace UI not fully built. | Modified | SDK + iframe model implemented; admin/enable flows exist. Marketplace UI is planned later — see [specs/architecture/plugins.md](specs/architecture/plugins.md).
| Custom Fields | Trello supports custom fields as a Power-Up; UI for adding fields and showing on card. | Native custom fields support (TEXT, NUMBER, DATE, CHECKBOX, DROPDOWN) with card values and `show_on_card` badges. | Modified | Custom fields are built-in (not a Power-Up). See changelog and custom-fields implementation notes in specs changelog entries (e.g. [specs/changelog/20260310_164500.md](specs/changelog/20260310_164500.md)).
| Permissions & Guest Roles | Trello has workspace/board admins, members, guests; granular perms via Enterprise. | Workspace/board roles, guest types with write/read guards; server-side and client-side permission helpers included. | Same (with explicit guestType) | Guest invite flows and `canBoardGuestWrite` guards described in changelog and board specs (see [specs/changelog/20260318_075511.md](specs/changelog/20260318_075511.md)).
| Webhooks / External Webhooks | Trello provides webhook API for external integrations. | Not documented in specs; no built-in webhook consumer/producer described. | Not Supported | Integrations are primarily via plugin SDK and REST API; external webhook endpoints are not defined in the current specs.
| Data Import / Migration | Trello stores full board state including attachments, member linkages, and external file references; boards often act as long-term archives. | Migration covers lists, cards, comments, labels, checklists and activity; attachments and some external integrations are not migrated. The intended plan: keep Trello boards as-is (read-only archive), remove non-admin members and leave a single admin/PM or dedicated guest account to retain access/ownership. | Modified | Attachments are intentionally not migrated due to storage/virus-scan and access constraints. Action items for migration: generate a migration report (mapping of Trello IDs → migrated IDs), record which attachments/links were skipped, mark archived boards as read-only, and notify workspace owners. |
